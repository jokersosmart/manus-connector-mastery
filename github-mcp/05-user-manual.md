# Outlook Calendar 連結器使用者操作手冊

作者：Joker

---

## 簡介：讓您的日曆為您工作

歡迎使用「Outlook Calendar 連結器」！本手冊專為終端使用者設計，旨在向您展示如何透過簡單的自然語言指令，讓 Manus 成為您強大的日曆管理助理。您將學習如何將日常的排程需求，轉化為 Manus 可以理解並執行的任務，從而節省您的寶貴時間，並將您的工作流程提升到一個新的自動化水平。

本手冊以**任務為導向**，為您提供一系列真實世界的使用場景。在每個場景中，我們都會展示：

1.  **您的指令**：您需要對 Manus 說的自然語言指令。
2.  **Manus 的魔法**：對 Manus 如何解析您的指令並轉換為底層工具調用的簡單解釋。

我們的目標是讓您直觀地理解「說」與「做」之間的聯繫，無需深入了解複雜的技術細節。

---

### 任務一：快速檢視我的行程

這是最基礎也是最高頻的需求。您可以用各種方式詢問您的日程安排。

| 您的指令 (您可以這樣說) | Manus 的魔法 (它會這樣理解與執行) |
| :--- | :--- |
| 「**我今天下午有什麼事？**」 | Manus 會鎖定「今天下午」這個時間範圍，並使用 `graph_listCalendarView` 工具來抓取您在該時段的所有日曆事件。 |
| 「**幫我看看下週所有的會議。**」 | Manus 會將「下週」解析為一個具體的日期範圍，然後同樣使用 `graph_listCalendarView` 來列出所有事件。 |
| 「**我跟 Alex 的下一個會議是什麼時候？**」 | Manus 會使用 `graph_listEvents` 工具，並設定一個篩選條件 (filter)，找出主題或參與者中包含「Alex」的未來事件。 |

---

### 任務二：輕鬆安排會議

告別在 Outlook 介面中手動點選的繁瑣操作。現在，一句話就能搞定會議安排。

| 您的指令 (您可以這樣說) | Manus 的魔法 (它會這樣理解與執行) |
| :--- | :--- |
| 「**安排一個『Q4 預算審查』會議，時間是下週三上午 10 點，邀請我和財務部門。**」 | Manus 會從您的話中提取出會議的**主題**、**時間**和**參與者**，然後調用 `graph_createEvent` 工具，將這些資訊填入對應的參數中，為您建立事件。 |
| 「**幫我訂一個每週一早上 9 點的團隊站會，直到年底。**」 | Manus 會識別出「每週一」和「直到年底」是重複性事件的信號。它會在調用 `graph_createEvent` 時，額外配置 `recurrence` 參數，設定好重複的模式和結束日期。 |
| 「**把下午三點的會改成線上會議。**」 | Manus 會先找到下午三點的會議，然後使用 `graph_updateEvent` 工具，將 `isOnlineMeeting` 參數設定為 `true`，並可能指定 `onlineMeetingProvider` 為 `teamsForBusiness`，為您生成一個 Teams 會議連結。 |

---

### 任務三：優雅地協調多人時間

這是連結器最具顛覆性的功能之一，能將您從無盡的溝通協調中解放出來。

| 您的指令 (您可以這樣說) | Manus 的魔法 (它會這樣理解與執行) |
| :--- | :--- |
| 「**幫我找一個我和 Lisa、Tom 下週都有空的 30 分鐘時間。**」 | 這會觸發強大的 `graph_findMeetingTimes` 工具。Manus 會將 Lisa 和 Tom 的 Email 地址作為輸入，自動在後台比對所有人的日曆空閒時段，並向您推薦幾個最佳選項。 |
| 「**就約第一個建議的時間。**」 | 在您從 `graph_findMeetingTimes` 的結果中做出選擇後，Manus 會接著調用 `graph_createEvent`，使用您選定的時間和之前提供的參與者資訊，完成會議的建立。 |

---

### 任務四：自動化您的日曆管理規則

讓 Manus 成為您 24/7 的日曆管家，根據您設定的規則自動處理事務。

| 您的指令 (您可以這樣說) | Manus 的魔法 (它會這樣理解與執行) |
| :--- | :--- |
| 「**幫我自動接受所有來自我們團隊內部、且我剛好有空的會議邀請。**」 | Manus 會為您建立一個自動化工作流。這個工作流會定期檢查新會議邀請，確認邀請人是否在您的團隊內，並使用 `graph_getSchedule` 檢查您在該時段是否空閒。如果所有條件都滿足，它會自動調用 `graph_acceptEvent`。 |
| 「**如果一個會議即將在 15 分鐘後開始，就發一條 Slack 訊息到『會議提醒』頻道。**」 | 這是一個跨應用的工作流。Manus 會設定一個監控器，定期檢查您的日曆。一旦發現符合條件的事件，它就會觸發另一個連結器——Slack 連結器，並執行發送訊息的操作。這完美詮釋了將日曆作為「觸發器」的核心價值。 |
| 「**拒絕所有在週末的會議邀請。**」 | 一個簡單但實用的規則。Manus 會檢查新邀請的日期，如果是在週六或週日，就自動調用 `graph_declineEvent` 工具，為您守護寶貴的休息時間。 |

## 結語：開始對話吧！

您已經看到了如何透過簡單的對話來駕馭複雜的日曆管理任務。關鍵在於，不要把 Manus 看作一個需要學習指令的軟體，而應將其視為一個能夠理解您**意圖**的助理。

下次當您需要處理任何與日曆相關的事務時，試著直接把您的需求告訴 Manus。您會驚訝地發現，許多您習以為常的繁瑣工作，都可以在一句話之間輕鬆完成。

現在，就去探索更多可能性，讓您的日曆真正為您服務吧！
