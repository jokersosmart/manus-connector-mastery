

# Cloudflare 連結器學習手冊：快速上手指南

**作者：Joker**

---

## 1. 歡迎！15分鐘掌握 Cloudflare 自動化核心

本手冊專為希望快速掌握 Cloudflare 連結器核心功能的您而設計。我們將跳過所有不必要的理論，直接進入最關鍵的技能和實踐。在接下來的 15 分鐘裡，您將學會如何透過簡單的自然語言指令，自動化建立和管理您的 Cloudflare 基礎設施。

**學習目標：**

*   理解「意圖驅動」的雲端管理模式。
*   完成您的帳戶設定。
*   掌握三項核心資源（KV, R2, D1）的建立方法。
*   學會如何對 D1 資料庫執行基本的 SQL 操作。

準備好了嗎？計時開始！

## 2. 核心概念：說人話，讓 AI 做事

傳統方式：您需要登入 Cloudflare -> 找到 R2 頁面 -> 點擊「建立儲存桶」 -> 輸入名稱 -> 點擊確認。

**使用 Manus 的新方式：** 您只需要告訴 Manus：「幫我建一個叫 `my-backup-bucket` 的 R2 儲存桶。」

這就是 **意圖驅動 (Intent-Driven)**。您表達您的「意圖」（您想要什麼），Manus Agent 會利用 Cloudflare 連結器自動完成所有「執行」細節。您的角色從「操作員」轉變為「指揮官」。

## 3. 快速啟動 (5 分鐘)

### 步驟一：認識您的工作空間

在開始之前，您需要知道您的指令將在哪個 Cloudflare 帳戶上執行。

*   **學習目標：** 確認並設定您的操作帳戶。
*   **自然語言指令：** 「我的 Cloudflare 下有哪些帳戶？」
*   **實踐練習：**
    1.  向 Manus 提問，列出您的所有帳戶。
    2.  如果有多個帳戶，選擇一個並告訴 Manus：「接下來的操作都在『您的帳戶名稱』上進行。」

## 4. 關鍵技能培養 (8 分鐘)

掌握以下三類資源的建立與操作，您就掌握了 80% 的日常 Cloudflare 管理任務。

### 技能一：建立 KV 命名空間 (用於設定)

*   **學習目標：** 快速建立一個用於儲存應用程式設定的鍵值儲存空間。
*   **場景：** 您的新應用需要一個地方來存放功能開關 (feature flags)。
*   **自然語言指令：** 「幫我建立一個 KV 命名空間，叫做 `app-feature-flags`。」
*   **實踐練習：** 為您的下一個專案「Project X」建立一個 KV 命名空間。

### 技能二：建立 R2 儲存桶 (用於檔案)

*   **學習目標：** 建立一個零出口費用的物件儲存桶，用於存放圖片、影片或備份。
*   **場景：** 您的網站需要一個地方來儲存使用者上傳的圖片。
*   **自然語言指令：** 「我需要一個 R2 儲存桶來放使用者頭像，名字就叫 `website-avatars`。」
*   **實踐練習：** 建立一個名為 `daily-database-backups` 的 R2 儲存桶。

### 技能三：建立並操作 D1 資料庫 (用於資料)

這是最重要的核心技能。您不僅要會建立，還要會與它互動。

*   **學習目標：** 建立一個無伺服器 SQL 資料庫，並能在其中建立資料表、插入資料。
*   **場景：** 您正在開發一個簡單的留言板功能。
*   **自然語言指令 (組合)：**
    1.  「幫我建立一個 D1 資料庫，叫 `guestbook-db`。」
    2.  「在 `guestbook-db` 裡，幫我建一個 `messages` 資料表，要有 `id`, `author`, `text` 三個欄位。」
    3.  「在 `messages` 表裡加一條留言，作者是『Joker』，內容是『這太酷了！』。」
*   **實踐練習：**
    1.  建立一個名為 `product-catalog-db` 的 D1 資料庫。
    2.  在其中建立一個 `products` 資料表，應包含 `sku`, `name`, `price` 欄位。
    3.  新增至少一個產品到資料表中。

## 5. 融會貫通 (2 分鐘)

現在，讓我們把所有技能串聯起來，完成一個真實的專案啟動任務。

*   **學習目標：** 用一個指令完成多個資源的建立。
*   **場景：** 您要啟動一個全新的部落格平台。
*   **終極指令 (單一意圖)：**

    > 「我要啟動一個新的部落格專案。幫我建立一個叫 `my-blog-settings` 的 KV 空間，一個叫 `my-blog-media` 的 R2 儲存桶，和一個叫 `my-blog-content` 的 D1 資料庫。最後，在資料庫裡幫我建好 `articles` 和 `comments` 兩個資料表。」

*   **實踐練習：** 嘗試用您自己的話，向 Manus 下達一個類似的、用於啟動您想像中的專案的指令。
