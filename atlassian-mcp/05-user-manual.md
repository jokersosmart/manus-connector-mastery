# Atlassian 連接器使用者操作手冊

作者：Joker

## 簡介

本手冊專為終端使用者設計，旨在展示如何透過簡單的自然語言指令，利用 Manus Atlassian 連接器完成日常在 Jira 和 Confluence 上的各種任務。您不需要了解任何程式碼或複雜的技術細節，只需要像和一位聰明的助理對話一樣，告訴 Manus 您想做什麼。

## 核心理念：從「指令」到「行動」

您下達的每一個自然語言指令，Manus 都會進行解讀，並將其轉換為一個或多個具體的「工具調用 (Tool Call)」。這個過程就像一位人類助理聽懂了您的要求，然後去操作對應的軟體介面。以下我們將透過一系列的任務範例，來展示這個神奇的過程。

--- 

## 任務一：建立新的 Jira 任務

這是最常見的需求之一。當您腦中閃過一個待辦事項、發現一個 Bug，或需要規劃一個新功能時，無須離開當前思緒，直接告訴 Manus 即可。

### 場景：規劃新功能

*   **您的指令：**
    > 「嘿 Manus，幫我在『產品設計』(PD) 專案裡，建立一個新的 Story，標題是『使用者個人化頭像上傳功能』，然後把它指派給設計師 Emily。」

*   **Manus 的轉換過程：**
    1.  **識別意圖：** Manus 識別出您想要「建立」一個「Jira 議題」。
    2.  **提取參數：**
        *   `project_key`: 'PD'
        *   `issue_type`: 'Story'
        *   `summary`: '使用者個人化頭像上傳功能'
        *   `assignee`: 'Emily'
    3.  **選擇工具：** Manus 選擇了 `jira_create_issue` 工具。
    4.  **執行調用：** Manus 在背景執行了這個工具調用，相當於在 Jira 介面中幫您填寫了所有欄位並點擊了「建立」。

--- 

## 任務二：查詢 Jira 議題狀態

當您想了解某個任務的進度，或想看看自己還有哪些待辦事項時，不再需要打開 Jira 網站，層層點擊尋找。

### 場景：每日站會前準備

*   **您的指令：**
    > 「我想看看我目前在 Jira 上，所有狀態是『進行中』的任務有哪些？」

*   **Manus 的轉換過程：**
    1.  **識別意圖：** Manus 知道您想要「搜尋」Jira 議題。
    2.  **建構查詢：** Manus 根據您的指令，在內部建立了一段 JQL (Jira Query Language) 查詢：`assignee = currentUser() AND status = 'In Progress'`。
    3.  **提取參數：**
        *   `jql`: 'assignee = currentUser() AND status = 'In Progress''
    4.  **選擇工具：** Manus 選擇了 `jira_search_issues` 工具。
    5.  **執行調用：** Manus 執行搜尋，並將結果以清晰的列表形式呈現給您。

--- 

## 任務三：同步會議記錄到 Confluence

開完會後，常常需要將會議中的決議和待辦事項整理到 Confluence 頁面中。現在，這個過程可以極大簡化。

### 場景：會議結束後

*   **您的指令：**
    > 「幫我在 Confluence 的『產品團隊』空間 (代號 PROD) 裡，建立一個標題為『關於 V3 版本的腦力激盪會議記錄』的頁面，內容是：
    > 
    > **決議：**
    > *   核心功能將聚焦在 AI 整合。
    > 
    > **待辦事項：**
    > *   Joker：研究市場上主要的 AI 整合方案。
    > *   Emily：設計初步的 AI 功能使用者介面草圖。」

*   **Manus 的轉換過程：**
    1.  **識別意圖：** Manus 偵測到您要在「Confluence」中「建立頁面」。
    2.  **提取參數：**
        *   `space_key`: 'PROD'
        *   `title`: '關於 V3 版本的腦力激盪會議記錄'
        *   `content`: (您提供的完整 Markdown 內容)
    3.  **選擇工具：** Manus 選擇了 `confluence_create_page` 工具。
    4.  **執行調用：** Manus 將您的會議記錄直接發布到 Confluence，省去了您所有複製貼上的時間。

--- 

## 任務四：將 Jira 任務連結到 Confluence 文件

這是實現「知識關聯」的關鍵一步，將分散的任務與集中的文件連結起來。

### 場景：撰寫技術規格文件

*   **您的指令：**
    > 「我正在寫一份技術文件，頁面 ID 是 12345。請幫我把 Jira 任務『開發新的認證模組』(TASK-567) 的連結，加到這個頁面的最下方。」

*   **Manus 的轉換過程：**
    1.  **識別意圖：** Manus 理解到您需要「更新」一個「Confluence 頁面」。
    2.  **多步執行：** 這個指令稍微複雜，Manus 可能會分兩步完成：
        a. **取得 Jira 任務資訊：** 首先，它可能會先內部查詢 TASK-567 的 URL。
        b. **更新 Confluence 頁面：** 然後，它會執行 `confluence_update_page` 工具，將取得的 URL 作為內容的一部分，附加到 ID 為 12345 的頁面中。
    3.  **選擇工具：** `jira_get_issue_details` (內部使用) 和 `confluence_update_page`。
    4.  **執行調用：** Manus 像一位細心的助理，先找到您要的資料，再幫您貼到正確的地方。

## 結語

正如您所見，使用 Manus Atlassian 連接器，您只需要專注於「您想達成什麼」，而將「如何操作」這個繁瑣的過程交給 Manus。這不僅僅是節省時間，更是改變了您與工作軟體的互動方式。現在就開始，用您的語言，指揮您的數位世界吧！
