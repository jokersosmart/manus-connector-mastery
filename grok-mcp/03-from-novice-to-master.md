# Grok 從新手到精通指南

**作者：Joker**

## 歡迎來到 Grok 的世界

歡迎！本指南將帶您踏上一段從新手到精通 Grok MCP 連結器的旅程。Grok 不僅僅是一個能言善道的聊天機器人，它是一個強大的協作者，能夠透過「工具」與外部世界互動，完成遠超文字生成範圍的複雜任務。無論您是初次接觸大型語言模型（LLM）的新手，還是經驗豐富的開發者，本指南都將為您提供一條清晰的學習路徑，讓您能淋漓盡致地發揮 Grok 的潛力。

我們的旅程將分為五個階段，從基礎概念開始，逐步深入到進階技巧與實戰應用。

## 第一章：基礎入門 (新手)

在這個階段，我們將建立對 Grok 的基本認識。

### 什麼是 Grok？

Grok 是由 xAI 開發的大型語言模型，其設計目標是提供真實、富有洞察力的答案。與許多其他 LLM 不同，Grok 的一個顯著特點是它能夠透過 X 平台獲取即時資訊，使其回答更具時效性。此外，Grok 還帶有一點叛逆和幽默感，讓互動體驗更加生動有趣。

### 您的第一次互動

在 Manus 中使用 Grok 非常簡單。您只需要像與人對話一樣，用自然語言提出您的問題或請求即可。Grok 可以回答問題、腦力激盪、撰寫文章、翻譯語言等等。

> **練習 1：**
> 請嘗試向 Grok 提出以下問題：
> - 「向我解釋什麼是黑洞，用小學生能聽懂的語言。」
> - 「幫我為一個新的咖啡品牌想五個有創意的名字。」
> - 「將『to be or not to be, that is the question』翻譯成中文。」

## 第二章：解鎖超能力 — 使用內建工具 (初學者)

真正的魔法始於「工具」。工具是讓 Grok 能夠執行特定動作的超能力。

### 什麼是「工具」？

您可以將工具想像成 Grok 可以使用的應用程式。當您提出一個需要外部資訊或特定計算的請求時，Grok 會自動選擇並使用合適的工具來完成任務。Grok 內建了幾種非常強大的工具。

- **網頁搜尋 (`web_search`)**：這是 Grok 通往即時網路世界的窗口。當您詢問關於最新新聞、事件或任何需要當前資訊的問題時，Grok 會使用它來尋找答案。
- **X 搜尋 (`x_search`)**：讓 Grok 能夠即時探索全球的對話，找到特定人物的發言或熱門趨勢。
- **程式碼執行器 (`code_execution`)**：這是一個內建的 Python 沙箱。您可以要求 Grok 執行數學計算、資料分析或任何可以透過程式碼解決的問題。

> **練習 2：**
> 嘗試以下指令，觀察 Grok 如何使用工具：
> - 「最近的奧斯卡最佳影片是哪一部？」（觸發 `web_search`）
> - 「看看 xAI 的官方 X 帳號發了什麼新東西。」（觸發 `x_search`）
> - 「幫我計算 30 的階乘。」（觸發 `code_execution`）

## 第三章：打造您的專屬工具 — 函式調用 (中級)

這是最令人興奮的部分：您可以教 Grok 使用您自己創建的工具。

### 什麼是函式調用？

函式調用（Function Calling）允許您定義自己的函式（例如，查詢公司資料庫、呼叫另一個 API），並讓 Grok 在需要時「請求」執行它們。這開啟了無限的可能性，您可以將 Grok 與您現有的任何系統或工作流程整合。

其運作流程如下：

1.  **定義**：您在程式碼中定義一個函式，並用 JSON Schema 格式描述它的功能、參數。
2.  **請求**：您將問題和函式定義一起發送給 Grok。
3.  **判斷**：Grok 判斷是否需要調用您的函式來回答問題。
4.  **返回請求**：如果需要，Grok API 會返回一個「工具調用請求」，告訴您它想用什麼參數調用哪個函式。
5.  **執行**：您的應用程式執行該函式。
6.  **返回結果**：您將函式的執行結果再發送給 Grok。
7.  **最終回答**：Grok 根據函式結果，產生最終的自然語言回答。

> **練習 3：**
> 嘗試設計一個名為 `get_stock_price` 的函式，它接受一個股票代碼作為參數，並返回該股票的當前價格。然後，向 Grok 提問：「蘋果公司（AAPL）現在的股價是多少？」

## 第四章：進階知識 — 與您的資料對話 (進階)

透過「集合搜尋」，您可以讓 Grok 成為您個人或公司知識庫的專家。

### 什麼是檢索增強生成 (RAG)？

RAG (Retrieval-Augmented Generation) 是一種讓 LLM 在回答問題前，先從一個特定的知識庫中「檢索」相關資訊的技術。Grok 的「集合搜尋 (`collections_search`)」工具就是 RAG 的一種實現。

您可以上傳您的文件（例如 PDF、Markdown、純文字檔），將它們建立成一個「集合」。然後，當您提問時，Grok 會優先在這個集合中尋找答案，提供基於您私有資料的精準回覆。

> **練習 4：**
> 建立一個包含您公司產品介紹的集合。然後向 Grok 提問：「我們的『奧創』產品有哪些主要功能？」

## 第五章：提示的藝術與多工具協同 (大師)

在最高級別，您將學會如何像指揮家一樣，引導 Grok 協同使用多個工具來解決複雜問題。

### 為工具使用而設計的提示

如何提問，會直接影響 Grok 能否準確地理解您的意圖並選擇正確的工具。關鍵在於清晰、具體地描述您的目標，而不是指令的步驟。

- **反例**：「搜尋 A，然後用 B 計算，最後總結。」
- **正例**：「我想知道 A 公司的最新財報數據，請幫我計算其營收年增率，並分析主要的成長動能是什麼。」

### 多工具鏈

對於複雜的任務，Grok 能夠在一次查詢中，有邏輯地依序或平行調用多個工具。例如，它可能會先用 `web_search` 查找資料，然後用 `code_execution` 分析數據，最後生成總結報告。

> **練習 5：**
> 嘗試這個複雜指令，分析 Grok 的思考過程：
> 「研究一下從 2024 年至今，NVIDIA 和 AMD 的股價表現。用 Python 畫一張比較圖，並總結哪家公司的成長更為顯著及其可能的原因。」
