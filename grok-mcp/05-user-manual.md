# Grok 終端使用者操作手冊

**作者：Joker**

## 前言：從「指令」到「行動」

歡迎閱讀 Grok 終端使用者操作手冊。本手冊的核心目的，是揭示 Manus 如何將您日常使用的自然語言指令，巧妙地轉換為對 Grok 連結器背後強大「工具」的精確調用。您將會看到，您所下的每一個簡單指令，背後都可能觸發了一系列複雜但高效的自動化操作。

本手冊以任務為導向，透過具體範例，讓您直觀地理解從「意圖」到「行動」的轉換過程。

## 任務一：獲取即時資訊與新聞

**您的目標：** 您想了解關於一個熱門話題的最新動態。

| | |
| :--- | :--- |
| **您的自然語言指令** | 「關於最近的 AI 晶片大戰，有哪些最新的進展？」 |
| **Manus 的幕後工作** | 1.  **意圖識別**：Manus 分析您的指令，識別出關鍵詞「最新進展」，理解到這是一個需要即時資訊的請求，而非利用其內部知識庫就能回答的問題。<br>2.  **工具選擇**：Manus 決定使用 Grok 的 `web_search` 工具。<br>3.  **工具調用**：它向 Grok 發出指令，相當於在說：「`web_search(query='latest news on AI chip war')`」。<br>4.  **結果生成**：Grok 執行網路搜尋，閱讀數篇相關的新聞文章和分析報告，然後將所有資訊整合成一段流暢、準確的摘要回覆給您，並附上主要的資訊來源連結。 |

## 任務二：執行計算與數據分析

**您的目標：** 您需要快速進行一個數學計算或簡單的數據處理。

| | |
| :--- | :--- |
| **您的自然語言指令** | 「請幫我計算一下這幾個數字的平均值和中位數：11, 42, 23, 87, 51, 9。」 |
| **Manus 的幕後工作** | 1.  **意圖識別**：Manus 識別出「計算」、「平均值」、「中位數」等詞語，確認這是一個數學運算任務。<br>2.  **工具選擇**：Manus 選擇使用 Grok 的 `code_execution` 工具，因為這類任務透過程式碼處理最為精確高效。<br>3.  **程式碼生成與調用**：Manus 自動編寫一段 Python 程式碼來完成這個任務，然後將其傳遞給 Grok 執行。指令內容類似：「`code_execution(code='import numpy as np; data = [11, 42, 23, 87, 51, 9]; print(f"平均值: {np.mean(data)}, 中位數: {np.median(data)}")')`」。<br>4.  **結果生成**：Grok 在其安全的沙箱環境中執行這段程式碼，捕捉到列印出來的輸出，並將其作為最終結果呈現給您。 |

## 任務三：查詢您的私有知識庫

**您的目標：** 您需要從公司內部的大量文件中，快速找到一條具體資訊。

| | |
| :--- | :--- |
| **您的自然語言指令** | （假設您已上傳公司所有季度的財報）「在我們 2025 年第二季度的財報中，『亞太地區』的營收貢獻是多少？」 |
| **Manus 的幕後工作** | 1.  **意圖識別**：Manus 偵測到「在我們的財報中」這樣的限定詞，理解到這不是一個通用問題，而是需要查詢您的私有資料。<br>2.  **工具選擇**：Manus 選擇使用 Grok 的 `collections_search` 工具，這是一個專為檢索增強生成（RAG）設計的工具。<br>3.  **工具調用**：Manus 向 Grok 發出指令，在您指定的「公司財報」集合中進行語意搜尋。指令類似：「`collections_search(collection='company_financial_reports', query='Q2 2025 revenue Asia-Pacific')`」。<br>4.  **結果生成**：Grok 在向量資料庫中找到最相關的文件段落，提取出關於亞太地區營收的具體數字和上下文，並生成精準的回答。 |

## 任務四：與外部系統互動

**您的目標：** 您希望 Manus 能幫您操作另一個軟體系統，例如查詢訂單狀態或建立一個待辦事項。

| | |
| :--- | :--- |
| **您的自然語言指令** | 「幫我查一下 JIRA 上，代號為『MAN-123』的任務目前的狀態是什麼？」 |
| **Manus 的幕後工作** | 這是一個更進階的「函式調用」場景，涉及多個步驟：<br>1.  **意圖識別與工具選擇**：Manus 將您的請求發送給 Grok。Grok 根據您預先定義好的可用工具列表，發現有一個名為 `get_jira_ticket_status` 的工具最符合需求。<br>2.  **請求工具調用**：Grok 不會自己執行，而是向 Manus 返回一個請求，內容是：「請幫我調用 `get_jira_ticket_status` 函式，參數是 `ticket_id='MAN-123'`」。<br>3.  **執行本地函式**：Manus 的後端服務接收到這個請求，執行您自己編寫的、與 JIRA API 對接的 `get_jira_ticket_status('MAN-123')` 函式。<br>4.  **返回執行結果**：假設您的函式從 JIRA 獲取到狀態為「進行中」，Manus 會將這個結果「`'In Progress'`」回傳給 Grok。<br>5.  **生成最終回覆**：Grok 收到函式的執行結果後，將其翻譯成通順的自然語言：「JIRA 任務 MAN-123 目前的狀態是『進行中』。」 |

## 結論

正如您所見，Manus 與 Grok 的協作，核心在於一個強大的「翻譯」過程：將您以自然語言表達的「目標」，翻譯成機器可以精確執行的「工具調用」。理解了這個過程，您就能更有效地設計您的指令，更充分地發揮 Grok 作為一個強大行動代理的潛力。
