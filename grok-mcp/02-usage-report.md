# Grok MCP 連結器使用說明報告

**作者：Joker**

## 1. 簡介

本文件旨在為 Manus MCP 的「Grok」連結器提供一份詳盡的使用說明。Grok 連結器透過其強大的工具調用（Tool Calling）能力，讓 Manus 得以執行超越純文字生成的複雜任務，例如即時網頁搜尋、執行程式碼、查詢使用者自訂資料庫等。本報告將詳細介紹 Grok 連結器提供的所有可用工具、其參數、功能、使用範例及預期輸出，以協助開發者充分利用其強大功能。

根據 xAI 的官方文件 [1]，Grok 的工具分為兩大類：

- **內建工具 (Built-in Tools)**：由 xAI 伺服器端管理並自動執行的工具，使用者僅需進行配置即可調用。
- **函式調用 (Function Calling)**：允許使用者定義自己的工具，模型可以請求調用這些自訂函式，給予開發者完全的控制權。

本報告將依序介紹這些工具的具體用法。

## 2. 內建工具 (Built-in Tools)

Grok 提供了一系列強大的內建工具，可直接在 Manus 中透過自然語言指令進行調用。

### 2.1. 網頁搜尋 (Web Search)

| 項目 | 說明 |
| :--- | :--- |
| **功能說明** | 賦予 Grok 即時搜尋網際網路並瀏覽網頁內容的能力，以回答關於最新事件、資訊或需要外部知識的問題。 |
| **參數** | 無需顯式參數。Manus 會自動將使用者的自然語言查詢轉換為搜尋指令。 |
| **使用範例** | **使用者指令**：「xAI 最近發布了哪些新的大型語言模型？」<br>**Manus 轉換**：Manus 將此問題識別為需要即時資訊，並指示 Grok 調用 `web_search` 工具。Grok 會執行搜尋，閱讀相關新聞稿或文章，並綜合資訊後回答。 |
| **預期輸出** | 一段基於最新網路搜尋結果的綜合性回答，並可能附上引用來源連結，讓使用者可以追溯資訊來源。 |

### 2.2. X 搜尋 (X Search)

| 項目 | 說明 |
| :--- | :--- |
| **功能說明** | 專門用於在 X (前 Twitter) 平台上搜尋貼文、使用者和話題。這對於獲取即時的公眾輿論、特定人物的發言或熱門趨勢非常有用。 |
| **參數** | 無需顯式參數。查詢內容由自然語言指令提供。 |
| **使用範例** | **使用者指令**：「幫我找一下 Elon Musk 最近關於 Grok 的貼文。」<br>**Manus 轉換**：Manus 指示 Grok 調用 `x_search` 工具，並以「Elon Musk Grok」等作為關鍵字進行搜尋。 |
| **預期輸出** | 相關的 X 貼文列表或內容摘要，可能包含貼文連結、作者和發布時間。 |

### 2.3. 程式碼執行器 (Code Execution)

| 項目 | 說明 |
| :--- | :--- |
| **功能說明** | 在一個沙箱環境中執行 Python 程式碼。這使得 Grok 能夠執行計算、資料分析、演算法實現等任務。 |
| **參數** | `code`: 需要執行的 Python 程式碼字串。 |
| **使用範例** | **使用者指令**：「計算從 1 到 100 的所有奇數的和，並告訴我結果。」<br>**Manus 轉換**：Manus 理解這是一個計算任務，指示 Grok 調用 `code_execution` 工具，並傳入生成的 Python 程式碼：`sum(i for i in range(1, 101) if i % 2 != 0)`。 |
| **預期輸出** | 程式碼執行的結果。在此範例中，輸出將是數字 `2500`。 |

### 2.4. 集合搜尋 (Collections Search)

| 項目 | 說明 |
| :--- | :--- |
| **功能說明** | 在使用者預先上傳的檔案集合（Collections）中進行語意搜尋。這是一種強大的檢索增強生成 (RAG) 功能，讓 Grok 能基於特定領域的私有知識進行回答。 |
| **參數** | `query`: 在集合中搜尋的自然語言查詢。 |
| **使用範例** | **使用者指令**：（假設使用者已上傳公司內部產品文件）「我們的『星塵計畫』第四季度的主要目標是什麼？」<br>**Manus 轉換**：Manus 指示 Grok 調用 `collections_search` 工具，在指定的檔案集合中搜尋與「星塵計畫 第四季度 目標」相關的內容。 |
| **預期輸出** | 從上傳文件中提取的相關段落或摘要，精準回答使用者的問題。 |

## 3. 函式調用 (Function Calling)

函式調用是 Grok 最靈活的功能，它允許開發者將任何自訂的 Python 函式暴露給模型，讓模型可以在需要時「請求」執行這些函式。

| 項目 | 說明 |
| :--- | :--- |
| **功能說明** | 開發者定義一組工具（函式），並在請求中提供這些工具的定義。Grok 會根據使用者的問題，判斷是否需要以及如何調用這些工具。如果需要，API 不會直接執行函式，而是會返回一個包含函式名稱和參數的「工具調用請求」。開發者需要在自己的應用程式中捕獲此請求，執行對應的函式，然後將執行結果返回給模型，讓模型根據結果產生最終回覆。 |
| **參數** | `tools`: 一個包含工具定義的列表。每個工具定義包含函式名稱、描述和參數綱要 (JSON Schema)。 |
| **使用範例** | **使用者指令**：「查詢一下我們資料庫裡，訂單號為 ORD-20251107 的客戶是誰？」<br>**Manus 轉換**：
1.  開發者預先定義了一個名為 `get_customer_by_order` 的函式，並將其定義提供給 Grok。
2.  Manus 將使用者指令發送給 Grok。Grok 分析後認為需要調用 `get_customer_by_order` 函式。
3.  Grok API 返回一個工具調用請求，內容為 `{ "name": "get_customer_by_order", "arguments": { "order_id": "ORD-20251107" } }`。
4.  Manus 的後端接收此請求，執行本地的 `get_customer_by_order("ORD-20251107")` 函式，假設返回結果為 `{"customer_name": "Joker"}`。
5.  Manus 將此執行結果回傳給 Grok。
6.  Grok 根據函式返回的結果，產生最終的自然語言回覆。 |
| **預期輸出** | 「訂單 ORD-20251107 的客戶是 Joker。」 |
