# Grok 學習手冊

**作者：Joker**

## 前言：您的 AI 協作夥伴

歡迎使用本學習手冊！本手冊專為希望快速上手並掌握 Grok MCP 連結器核心技能的使用者設計。我們將摒棄繁雜的理論，透過清晰的學習目標和動手實踐，帶您一步步解鎖 Grok 的強大功能，讓它成為您日常工作與學習中不可或缺的智慧夥伴。

## 模組一：基礎對話 — 您的第一個 AI 朋友

**學習目標：** 學會如何與 Grok 進行基本的自然語言互動，將其作為一個知識淵博的問答專家。

### 核心概念

Grok 的基礎是理解並回應您的文字指令。您可以向它提問、請它解釋概念、尋求建議，或進行腦力激盪。把它當作一個反應迅速、知識廣泛的團隊成員即可。

### 實踐練習 1

請開啟 Manus，選擇 Grok 模型，並嘗試以下任一指令：

1.  **知識問答**：「向我解釋什麼是『第一性原理』，並舉一個伊隆·馬斯克應用它的例子。」
2.  **創意發想**：「我正在計畫一場關於『未來工作模式』的演講，請幫我想三個吸引人的開場白。」
3.  **文字處理**：「幫我潤飾這段文字，讓它聽起來更專業：『我們的東西很好，因為我們很努力做。』」

**驗收標準：** 您能夠順暢地與 Grok 進行一問一答，並獲得符合您預期的文字回覆。

## 模組二：內建工具 — 賦予 Grok 超能力

**學習目標：** 掌握透過自然語言指令，驅動 Grok 使用其內建工具（網頁搜尋、程式碼執行）來解決超越純文字範圍的問題。

### 核心概念

當您的問題需要即時資訊或數學計算時，Grok 會自動啟用它的「工具」。您無需學習複雜的語法，只需像平常一樣提問。

-   **網頁搜尋 (`web_search`)**：當 Grok 偵測到您的問題涉及最新事件或需要外部知識時，它會自動上網搜尋。
-   **程式碼執行器 (`code_execution`)**：當您提出數學計算、資料排序等需要邏輯運算的需求時，Grok 會在背後編寫並執行 Python 程式碼。

### 實踐練習 2

請嘗試以下指令，並留意 Grok 可能會如何利用工具來回答：

1.  **即時資訊查詢**：「2024 年夏季奧運會的主辦城市是哪裡？有哪些新增的比賽項目？」
2.  **金融數據查詢**：「幫我查一下蘋果公司（AAPL）和微軟公司（MSFT）目前的市值，並計算兩者的差距。」
3.  **邏輯與計算**：「如果一個披薩切成 8 片，分給 3 個人，每人最多能拿幾片？還剩下幾片？」

**驗收標準：** 您能夠提出需要外部資訊或計算的問題，並從 Grok 的回答中看到它整合了來自工具的結果（例如，引用了網頁來源，或給出了精確的計算數值）。

## 模組三：函式調用 — 打造您的專屬自動化流程

**學習目標：** 理解函式調用的核心概念，並能構思如何將 Grok 與您自己的工作流程或資料庫進行整合。

### 核心概念

這是 Grok 最強大的功能之一。您可以定義自己的工具（函式），讓 Grok 在需要時呼叫它們。這相當於給了 Grok 一把鑰匙，讓它能夠操作您授權的任何外部系統，例如：

-   查詢您公司的 CRM 系統客戶資料。
-   在您的專案管理工具中建立一個新任務。
-   讀取您內部資料庫的銷售數據。

您只需向 Grok 描述您想做什麼，它就會自動找出對應的工具並請求執行。

### 實踐練習 3 (概念設計)

假設您是一名電商運營人員，您希望 Grok 能幫助您查詢商品庫存。請思考並回答以下問題：

1.  您會將這個工具（函式）命名為什麼？（例如：`get_product_inventory`）
2.  這個函式需要接收什麼「參數」才能運作？（例如：商品 ID 或商品名稱）
3.  當函式成功執行後，它應該「返回」什麼資訊？（例如：庫存數量）
4.  有了這個工具後，您會如何用一句自然語言指令來使用它？（例如：「幫我查一下『智慧降噪耳機 Pro』還有多少庫存？」）

**驗收標準：** 您能夠清晰地描述一個自訂工具的輸入、輸出和使用場景，理解函式調用如何將自然語言與具體業務操作連結起來。

## 模組四：集合搜尋 — 讓 Grok 成為您的領域專家

**學習目標：** 學會如何上傳私有文件，並利用「集合搜尋」功能，讓 Grok 基於您的專屬知識庫來回答問題。

### 核心概念

您可以將您的文件（如公司政策手冊、專案報告、會議記錄等）上傳到 Manus，並將它們組成一個「集合」。之後，當您向 Grok 提問時，可以指定它在這個集合內尋找答案。這使得 Grok 的回答不僅準確，而且完全基於您提供的私有、可信的資料來源。

### 實踐練習 4

1.  **建立知識庫**：找到一份您自己的文件（例如，一份 PDF 格式的報告或一篇 Markdown 格式的文章），在 Manus 中將其上傳並建立一個名為「我的知識庫」的集合。
2.  **進行查詢**：向 Grok 提出一個只能從該文件中找到答案的具體問題。例如：「在我的『第三季度市場分析報告』中，提到的最大挑戰是什麼？」

**驗收標準：** Grok 能夠根據您上傳的文件內容，準確地回答您的問題，而不是提供來自網路的通用資訊。
