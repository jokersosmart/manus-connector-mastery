# Canva 連結器使用者操作手冊

作者：Joker

---

本手冊為終端使用者設計，旨在展示如何透過簡單的自然語言指令，驅動 Manus 的 Canva 連結器來完成各種設計任務。我們將以任務為導向，提供具體的指令範例，並揭示 Manus 在幕後如何將您的話語轉換為精確的工具調用。

## 核心理念：您說話，Manus 執行

您不需要學習複雜的程式碼或 API 語法。您只需要像平常與人溝通一樣，告訴 Manus 您想在 Canva 中完成什麼。Manus 會理解您的意圖，並選擇最合適的工具來執行任務。

--- 

## 任務一：從零開始建立一個設計

當您需要一個特定尺寸的畫布，例如用於部落格文章的橫幅或特定廣告平台的尺寸時，這個功能非常有用。

**您的指令：**

> 「嘿 Manus，幫我在 Canva 建立一個 1200x630 像素的設計，用來當作 Facebook 貼文的圖片。」

**Manus 的幕後工作：**

1.  **意圖識別**：Manus 識別出您想要「建立設計」。
2.  **參數提取**：它從您的指令中提取出關鍵資訊：`寬度=1200`，`高度=630`。
3.  **工具調用**：Manus 調用 `canva.create_design` 工具，並傳入以下參數：

    ```json
    {
      "width": 1200,
      "height": 630
    }
    ```

--- 

## 任務二：使用現有範本快速產生設計

這是 Canva 的核心優勢。當您想利用 Canva 精美的範本庫來快速啟動一個設計時，只需提供範本 ID。

**您的指令：**

> 「我想用 Canva 的範本 `dae-abc-123` 來製作一份新的簡報。」

**Manus 的幕後工作：**

1.  **意圖識別**：同樣是「建立設計」，但這次是基於範本。
2.  **參數提取**：Manus 找到了您指定的 `範本 ID`。
3.  **工具調用**：Manus 再次調用 `canva.create_design`，但這次傳入的是 `template_id`：

    ```json
    {
      "template_id": "dae-abc-123"
    }
    ```

--- 

## 任務三：將您的品牌 Logo 上傳到 Canva

在開始設計前，您通常需要將自己的品牌資產（如 Logo、字體、產品圖片）上傳到 Canva。

**您的指令：**

> 「把 `/home/ubuntu/my_project/assets/logo-dark.png` 這個檔案上傳到我的 Canva 帳號。」

**Manus 的幕後工作：**

1.  **意圖識別**：Manus 理解您要「上傳資產」。
2.  **參數提取**：它獲取了您提供的檔案路徑。
3.  **工具調用**：Manus 調用 `canva.upload_asset` 工具：

    ```json
    {
      "file_path": "/home/ubuntu/my_project/assets/logo-dark.png"
    }
    ```

--- 

## 任務四：批次產生客製化內容

這是 Canva 連結器最高效的應用場景。想像您需要為 50 位活動參與者製作個人化的邀請函。

**您的指令：**

> 「用我的 Canva 邀請函範本 `template-vip-invite`，產生一張新的邀請函，把『貴賓姓名』這個欄位換成『陳大文』。」

**Manus 的幕後工作：**

1.  **意圖識別**：Manus 偵測到您想要「自動填充」一個設計。
2.  **參數提取**：它識別出 `範本 ID` 和一個包含 `變數名稱`（貴賓姓名）與 `對應值`（陳大文）的資料對。
3.  **工具調用**：Manus 調用 `canva.autofill_design` 工具。這個工具的強大之處在於 `data` 參數可以包含多個鍵值對，一次替換範本中的所有變數。

    ```json
    {
      "template_id": "template-vip-invite",
      "data": {
        "貴賓姓名": "陳大文"
      }
    }
    ```

您可以結合 Manus 的腳本功能，讀取一個名單檔案，並在迴圈中重複執行此指令，實現全自動的批次生產。

--- 

## 任務五：完成並匯出您的設計

當您的設計大功告成，就需要將它變成一個實際的檔案。

**您的指令：**

> 「好了，把我剛剛完成的設計 `dae-xyz-789` 匯出成高品質的 JPG 檔案。」

**Manus 的幕後工作：**

1.  **意圖識別**：Manus 知道您要「匯出設計」。
2.  **參數提取**：它抓取到 `設計 ID` 和目標 `格式` (JPG)。
3.  **工具調用**：Manus 調用 `canva.export_design` 工具：

    ```json
    {
      "design_id": "dae-xyz-789",
      "format": "JPG",
      "quality": 100
    }
    ```

## 結論

如您所見，Canva 連結器將複雜的 API 操作簡化為日常對話。您只需專注於您的創意目標，將執行的細節交給 Manus。現在就開始嘗試用您的語言來指揮 Canva 吧！
