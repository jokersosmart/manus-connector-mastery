# Metabase 連結器學習手冊

**作者：Joker**

---

## 簡介：兩小時掌握 Metabase 自動化核心技能

本手冊專為希望在最短時間內掌握 Metabase 連結器核心功能的學習者設計。我們深知您的時間寶貴，因此，本手冊摒棄了冗長的理論，專注於實用、高效的學習路徑。透過一系列目標明確的學習單元和動手實踐的練習，您將能夠在兩小時內，建立起使用 Manus 自動化 Metabase 數據任務的堅實能力。

我們的學習理念是「做中學」。每一個單元都圍繞一個具體的、可操作的目標展開，並提供清晰的指令範例和預期結果。完成本手冊後，您將不僅僅是「知道」如何使用這些工具，而是真正「能夠」運用它們來解決實際問題。

準備好了嗎？讓我們開始這段高效的學習之旅。

## 第一單元：基礎探索 (預計用時：30分鐘)

**學習目標**：熟悉您的 Metabase 環境，並成功執行您的第一個唯讀 (read-only) 查詢。

### 核心技能 1：盤點您的數據資產

在操作任何數據之前，首要任務是了解您擁有什麼。在 Metabase 的世界裡，這意味著了解您有哪些「資料庫」和「問題」。

- **實踐練習 1.1**：列出所有可用的資料庫。
  - **指令**：「列出我所有的 Metabase 資料庫。」
  - **目標**：成功獲取一個包含資料庫名稱和 ID 的清單。請特別記下您感興趣的資料庫的 ID，我們稍後會用到。

- **實踐練習 1.2**：列出所有已儲存的問題。
  - **指令**：「顯示我在 Metabase 上的所有問題。」
  - **目標**：成功獲取一個包含問題名稱和 ID 的清單。從這個清單中，挑選一個您感興趣的、看似簡單的問題（例如，一個沒有複雜篩選條件的計數問題），並記下它的 ID。

### 核心技能 2：執行並獲取結果

現在您已經有了一個目標問題的 ID，讓我們來獲取它的數據。

- **實踐練習 2.1**：執行一個問題並查看結果。
  - **指令**：「執行 ID 為 [您在練習 1.2 中記下的 ID] 的 Metabase 問題，並告訴我結果。」
  - **目標**：成功看到 Manus 以表格形式回傳的數據結果。這證明了您已經打通了從指令到獲取數據的完整鏈路。

- **實踐練習 2.2**：將結果匯出為不同格式。
  - **指令**：「將 ID 為 [您在練習 1.2 中記下的 ID] 的 Metabase 問題的結果匯出成 CSV 格式。」
  - **目標**：成功獲取一段 CSV 格式的文字輸出。嘗試將其複製並貼到一個純文字檔案中，以 `.csv` 的副檔名儲存，然後用電子表格軟體（如 Excel, Google Sheets）打開，確認格式無誤。這項技能在需要將數據導入其他系統時非常有用。

**單元回顧**：恭喜！您已經完成了本手冊中最重要的一步。您現在已經掌握了如何探索 Metabase 資源並從中提取數據的核心技能。這是所有更複雜自動化任務的基礎。

## 第二單元：動態與管理 (預計用時：45分鐘)

**學習目標**：學會處理需要動態輸入的查詢，並掌握管理 Metabase 資源的基本操作。

### 核心技能 3：處理參數化查詢

靜態查詢很有用，但真正的威力在於能夠動態地改變查詢條件。這需要您與帶有「篩選器」或「參數」的問題進行互動。

- **實踐練習 3.1**：找到一個需要參數的問題。
  - **方法**：瀏覽您在練習 1.2 中獲得的問題清單，或者在 Metabase UI 中尋找一個帶有篩選器（例如，日期選擇器、下拉選單）的問題。記下它的 ID。
  - **目標**：識別出一個用於本練習的目標問題 ID。

- **實踐練習 3.2**：執行帶有參數的查詢。
  - **指令**：「執行 ID 為 [您在練習 3.1 中記下的 ID] 的問題，並將 [參數名稱] 設定為 [您想查詢的值]。」（提示：如果您不確定參數的確切名稱，可以先嘗試不帶參數執行，Manus 通常會提示您需要哪些參數。）
  - **目標**：成功獲取篩選後的數據結果。例如，如果您查詢的是特定日期的銷售額，您應該只看到該日期的數據。這項技能是建立客製化、自動化報告的關鍵。

### 核心技能 4：管理您的數據源

除了查詢，您還可以透過 Manus 管理 Metabase 的設定。

- **實踐練習 4.1**：深入了解一個資料庫的結構。
  - **指令**：「顯示 ID 為 [您在練習 1.1 中記下的 ID] 的資料庫的所有資料表和欄位。」
  - **目標**：成功獲取一個描述該資料庫結構的詳細清單。這在您需要了解可以查詢哪些數據時非常有用，無需離開 Manus 的對話介面。

- **實踐練習 4.2 (選做)**：新增一個資料庫。
  - **警告**：這是一個寫入 (write) 操作，它會真實地在您的 Metabase 實例中新增一個資料庫。請確保您有相應的權限，並使用一個測試或非關鍵的數據源進行練習。
  - **指令**：「幫我在 Metabase 中新增一個資料庫。」
  - **目標**：跟隨 Manus 的引導，提供必要的連接資訊（如名稱、類型、主機、用戶名等），並成功收到資料庫已建立的確認訊息。這個練習將讓您體驗到透過 Manus 進行基礎設施管理的便捷性。

**單元回顧**：您現在已經從一個數據的「消費者」，進階為一個數據的「互動者」和「管理者」。您不僅能獲取數據，還能動態地塑造您的查詢，並對 Metabase 的底層資源進行操作。

## 第三單元：綜合實戰 (預計用時：45分鐘)

**學習目標**：將所學技能融會貫通，建立一個簡單但完整的自動化報告流程。

### 核心技能 5：設計並執行一個迷你工作流

現在，讓我們將所有碎片化的技能串聯起來，完成一個有實際意義的任務。

- **實戰專案**：建立一個「本日關鍵指標」的即時報告。

- **實踐練習 5.1**：定義您的報告內容。
  - **任務**：選擇 1 到 2 個您最關心的、可以代表「今天」業務狀況的 Metabase 問題。例如，「今日註冊用戶數」或「今日訂單金額」。記下它們的 ID。

- **實踐練習 5.2**：手動執行工作流。
  - **任務**：透過一系列連續的指令，手動模擬您的自動化報告流程。
  1.  **第一步**：執行第一個問題。「執行 ID 為 [問題A的ID] 的問題，將日期參數設為今天。」
  2.  **第二步**：記錄結果。將 Manus 回傳的關鍵數值（例如，用戶數 50 人）記錄下來。
  3.  **第三步**：執行第二個問題。「執行 ID 為 [問題B的ID] 的問題，將日期參數設為今天。」
  4.  **第四步**：記錄結果。同樣地，記錄下關鍵數值（例如，訂單金額 $1,200）。
  5.  **第五步**：匯總報告。向 Manus 發出最後一個指令：「幫我整理一下，今日註訪用戶數是 50 人，訂單金額是 $1,200。」
  - **目標**：成功地透過一系列指令，完成一次從數據獲取到資訊匯總的完整流程。這個手動執行的過程，是您後續將其完全自動化的藍圖。

- **實踐練習 5.3 (進階)**：構想您的自動化任務。
  - **任務**：思考一下，如何將練習 5.2 中的手動步驟，轉化為一個可以由 Manus 自動執行的任務？您需要哪些元素？
    -   一個**觸發器**：可能是定時的（例如，每天下午五點），也可能是由某個事件觸發的。
    -   一系列**動作**：對應您在 5.2 中執行的每一步指令。
    -   一個**輸出**：最終的報告應該發送到哪裡？是 Slack、電子郵件，還是儲存到一個文件中？
  - **目標**：在腦中或紙上，勾勒出這個自動化任務的結構。這將為您在真實工作中使用 Manus 的任務排程功能，打下堅實的思維基礎。

**單元回顧與總結**：您已經成功地將單點的技能，串聯成一個有價值的流程。這正是自動化的精髓所在。您不僅學會了 Metabase 連結器的各個工具，更重要的是，您學會了如何像一個自動化專家一樣思考：將複雜的任務拆解為一系列由機器執行的、可靠的步驟。

您已經完成了本手冊的所有核心內容。現在，是時候將這些新技能應用到您的日常工作中，去發現和解決那些一直困擾您的、重複性的數據任務了。祝您在 Metabase 自動化的道路上一帆風順！
