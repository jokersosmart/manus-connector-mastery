# 「LINE」連結器使用者操作手冊

作者：Joker

## 簡介

本手冊為終端使用者設計，旨在展示如何透過簡單的自然語言指令，利用 Manus Agent 和「LINE」連結器來執行各種自動化任務。您不需要了解程式碼或複雜的 API，只需像與真人助理對話一樣，下達您的指令即可。

## 核心概念：從指令到行動

當您給出一個與 LINE 相關的指令時，Manus Agent 會在背後進行智慧的解析，並將其轉換為對「LINE」連結器中特定工具的調用。以下是一些範例，說明這個過程是如何運作的。

--- 

### 任務一：發送一則簡單的訊息

**您的指令：**

> 「嘿 Manus，幫我傳個 LINE 給 `U123456...`，跟他說『專案文件已經更新，請查收』。」

**Manus 的內部轉換：**

Manus Agent 識別出您的意圖是「發送訊息」。它會解析出三個關鍵資訊：

1.  **目標服務：** LINE
2.  **接收者：** `U123456...`
3.  **訊息內容：** 「專案文件已經更新，請查收」

接著，它會調用 `line.send_message` 工具，並將這些資訊作為參數傳入：

```json
{
  "tool_name": "line.send_message",
  "parameters": {
    "recipient_id": "U123456...",
    "message": "專案文件已經更新，請查收"
  }
}
```

**結果：** 您的 LINE 聯絡人會立即收到這則訊息。

--- 

### 任務二：發送群組通知

**您的指令：**

> 「提醒一下『產品開發』群組，我們下午三點的會議地點改到 301 會議室。」

**Manus 的內部轉換：**

Manus Agent 知道「產品開發」是一個 LINE 群組。它可能會預先儲存了這個群組的 ID (`C987654...`)，或者透過名稱進行查找。然後，它會建構出相似的工具調用：

```json
{
  "tool_name": "line.send_message",
  "parameters": {
    "recipient_id": "C987654...",
    "message": "提醒：下午三點的會議地點改到 301 會議室。"
  }
}
```

**結果：** 所有在「產品開發」群組的成員都會看到這則更新通知。

--- 

### 任務三：結合其他工具的跨系統工作流程

**您的指令：**

> 「Manus，幫我看一下今天 Google 日曆上『客戶拜訪』這個行程的地址，然後傳 LINE 告訴司機 Peter。」

**Manus 的內部轉換：**

這個任務比較複雜，Manus 會將其拆解為多個步驟：

1.  **步驟一：查詢日曆**
    -   調用 `google_calendar.find_event` 工具，搜尋標題為「客戶拜訪」的今日活動。
    -   從回傳的活動資訊中，提取出 `location` (地址) 欄位的值，例如「台北市信義區市府路 1 號」。

2.  **步驟二：發送 LINE 訊息**
    -   查找聯絡人「司機 Peter」對應的 LINE 使用者 ID (`U...`)。
    -   將上一步獲取的地址，組合成一則完整的訊息。
    -   調用 `line.send_message` 工具：

    ```json
    {
      "tool_name": "line.send_message",
      "parameters": {
        "recipient_id": "U(司機Peter的ID)",
        "message": "Peter 您好，今天客戶拜訪的地址是：台北市信義區市府路 1 號。"
      }
    }
    ```

**結果：** 司機 Peter 會收到一則包含準確地址的 LINE 訊息，整個過程無需您手動查詢、複製和貼上。

--- 

## 結論

正如您所見，「LINE」連結器的強大之處在於它能將您的自然語言指令，無縫轉換為實際的 LINE 操作。透過與 Manus Agent 的互動，您可以輕鬆實現訊息的自動化發送與整合，大幅提升您的溝通效率。請盡情發揮您的想像力，探索更多有趣且實用的指令組合！
