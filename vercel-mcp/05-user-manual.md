# Notion 連結器終端使用者操作手冊

**作者：** Joker

---

歡迎使用 Manus Notion 連結器！本手冊專為終端使用者設計，旨在向您展示如何透過簡單的自然語言指令，讓 Manus 成為您強大的 Notion 助理。您無需關心複雜的程式碼或 API 細節，只需像與真人同事對話一樣，告訴 Manus 您想做什麼。

本手冊將以一個個具體的「任務場景」為導向，為您提供豐富的指令範例，並揭示在這些指令背後，Manus 是如何聰明地調用其工具箱來為您完成工作的。讓我們開始探索，看看您能如何將重複的工作交給 Manus，專注於更有創造力的事情。

## 任務一：快速建立筆記與頁面

**場景描述：**
當您靈感湧現，或需要快速記錄一些資訊時，您希望能立刻建立一個 Notion 頁面來存放它們，而不想被繁瑣的點擊操作打斷思路。

--- 

### 指令範例 1：建立一個獨立的筆記頁面

> **您說：**
> 「幫我開一個新的 Notion 頁面，標題是『關於下個季度的產品想法』，然後在裡面寫下：『核心思路是提升使用者體驗』。」

**Manus 的內心戲：**

1.  **分析意圖：** 使用者想要建立一個新的、獨立的 Notion 頁面，並且已經指定了標題和初始內容。

2.  **選用工具：** 這個任務非常直接，顯然應該使用 `notion-create-pages` 工具。

3.  **組合參數：** 我需要將使用者的指令轉換為 `create-pages` 工具能聽懂的 JSON 格式。
    *   `properties` 物件裡的 `title` 應該是「關於下個季度的產品想法」。
    *   `content` 參數應該是「核心思路是提升使用者體驗」。
    *   因為使用者沒有指定要放在哪個資料庫或頁面下，所以 `parent` 參數可以省略，這樣頁面就會建立在使用者的私人空間中。

4.  **執行調用：**
    ```json
    {
      "pages": [
        {
          "properties": {
            "title": "關於下個季度的產品想法"
          },
          "content": "核心思路是提升使用者體驗"
        }
      ]
    }
    ```

**最終效果：**
一個全新的、符合您要求的 Notion 頁面瞬間被建立好了，您可以立即投入到更深入的思考和寫作中。

--- 

### 指令範例 2：在指定的資料庫中新增一筆記錄

> **您說：**
> 「在我的『讀書清單』資料庫裡，新增一本書，書名叫《思考，快與慢》。」

**Manus 的內心戲：**

1.  **分析意圖：** 這次使用者不是要建立獨立頁面，而是要在一個名為「讀書清單」的**資料庫**中新增一個項目。這需要更精確的操作。

2.  **選用工具：** 建立頁面依然是 `notion-create-pages` 的職責，但在這之前，我需要先了解「讀書清單」這個資料庫的結構。所以，我需要先用 `notion-fetch` 工具。

3.  **組合參數與執行（第一步）：**
    *   我需要先找到「讀書清單」資料庫。我會先用 `notion-search` 搜尋這個名稱，找到它的 ID。
    *   然後，我會用 `notion-fetch` 工具，傳入這個 ID，來獲取它的詳細結構，特別是那個關鍵的 `data_source_id`，以及它的標題屬性到底叫什麼（很可能就叫「書名」）。

4.  **組合參數與執行（第二步）：**
    *   現在我知道了「門牌號碼」(`data_source_id`) 和屬性名稱。
    *   我會再次調用 `notion-create-pages` 工具。
    *   這次，`parent` 參數會被設定為我剛剛獲取的 `data_source_id`。
    *   `properties` 物件裡，我會將使用者提供的「《思考，快與慢》」填入到正確的標題屬性（例如「書名」）中。

5.  **執行調用（第二步）：**
    ```json
    {
      "parent": {
        "data_source_id": "從第一步fetch獲取的ID"
      },
      "pages": [
        {
          "properties": {
            "書名": "思考，快與慢"
          }
        }
      ]
    }
    ```

**最終效果：**
您無需關心 `data_source_id` 這些技術細節。在您下達指令後，您的「讀書清單」資料庫中就神奇地多出了一本名為《思考，快與慢》的新書記錄。

## 任務二：輕鬆修改現有內容

**場景描述：**
您的 Notion 工作區是一個動態的系統，資訊總是在不斷變化。您需要一種快速的方法來更新任務狀態、修改筆記內容或更正資訊，而無需在頁面之間來回尋找和點擊。

---

### 指令範例 1：更新資料庫項目的屬性

> **您說：**
> 「把我『任務列表』裡，關於『撰寫產品部落格文章』的任務，狀態改成『已完成』，並把截止日期設定為今天。」

**Manus 的內心戲：**

1.  **分析意圖：** 使用者想要更新一個特定資料庫項目中的兩個屬性：「狀態」和「截止日期」。

2.  **選用工具：** 這是典型的更新操作，我會使用 `notion-update-page` 工具。但首先，我需要精準地找到使用者說的那個任務頁面。

3.  **組合參數與執行（第一步）：**
    *   我會使用 `notion-search` 工具，在「任務列表」資料庫中搜尋標題包含「撰寫產品部落格文章」的頁面，從而獲得其 `page_id`。

4.  **組合參數與執行（第二步）：**
    *   現在我有了目標頁面的 `page_id`。
    *   我會調用 `notion-update-page` 工具。
    *   `page_id` 參數就是我剛找到的 ID。
    *   `command` 參數是 `update_properties`。
    *   `properties` 物件將包含兩個鍵值對：`"狀態": "已完成"` 和 `"截止日期": "2026-02-24"` (今天的日期)。

5.  **執行調用（第二步）：**
    ```json
    {
      "page_id": "從第一步search獲取的頁面ID",
      "command": "update_properties",
      "properties": {
        "狀態": "已完成",
        "截止日期": "2026-02-24"
      }
    }
    ```

**最終效果：**
您指令中提到的那個任務，其狀態和截止日期兩個欄位被同時、準確地更新了，整個過程流暢無比。

---

### 指令範例 2：修改頁面中的特定文字

> **您說：**
> 「在我昨天的會議記錄裡，把『預計下週上線』那句話，改成『預計下週三之前上線』。」

**Manus 的內心戲：**

1.  **分析意圖：** 使用者想要修改的不是屬性，而是頁面正文裡的一段具體文字。這需要更精細的定位。

2.  **選用工具：** 依然是 `notion-update-page` 工具，但這次的關鍵在於如何定位那段文字。

3.  **組合參數與執行：**
    *   首先，我會用 `notion-search` 找到「昨天的會議記錄」頁面，獲取其 `page_id`。
    *   接著，我會調用 `notion-update-page`。
    *   `page_id` 是我剛找到的 ID。
    *   `command` 參數將是 `replace_content_range`，因為使用者想要「替換」文字。
    *   最關鍵的是 `selection_with_ellipsis` 參數。我會將使用者的描述轉換為定位字串，例如 `"預計下週上...計下週上線"`，以確保能夠獨一無二地匹配到目標文字。
    *   `new_str` 參數則是要替換成的新內容：「預計下週三之前上線」。

4.  **執行調用：**
    ```json
    {
      "page_id": "會議記錄的頁面ID",
      "command": "replace_content_range",
      "selection_with_ellipsis": "預計下週上...計下週上線",
      "new_str": "預計下週三之前上線"
    }
    ```

**最終效果：**
您無需手動尋找那句話並逐字修改。指令下達後，會議記錄中的相關描述就被精準地更新了，確保了資訊的時效性。

## 任務三：結構化您的資料庫

**場景描述：**
當您需要一個全新的系統來管理專案、追蹤習慣或整理資訊時，您希望能快速建立一個結構清晰的資料庫，並在需要時對其進行調整，而無需手動進行繁瑣的欄位設定。

---

### 指令範例 1：從零到一建立一個新資料庫

> **您說：**
> 「幫我建立一個『新專案追蹤』資料庫，放在『我的工作區』頁面下面。它需要有這幾個欄位：『專案名稱』，這是標題；『專案經理』，可以選人；『狀態』，是單選，選項有『規劃中』、『進行中』和『已完成』；還有一個『截止日期』。」

**Manus 的內心戲：**

1.  **分析意圖：** 使用者想要建立一個全新的、結構明確的資料庫，並指定了它的父頁面和所有屬性（欄位）的細節。

2.  **選用工具：** 這是 `notion-create-database` 工具的完美應用場景。

3.  **組合參數與執行：**
    *   首先，我需要找到「我的工作區」頁面的 ID，以便將新資料庫放在它下面。這會用到 `notion-search`。
    *   接著，我會調用 `notion-create-database`。
    *   `parent` 物件會包含我剛找到的頁面 ID。
    *   `title` 參數是「新專案追蹤」。
    *   最核心的是 `schema` 參數。我會將使用者描述的欄位轉換為 SQL DDL 風格的語句：
        ```sql
        CREATE TABLE (
          "專案名稱" TITLE,
          "專案經理" PEOPLE,
          "狀態" SELECT('規劃中':gray, '進行中':blue, '已完成':green),
          "截止日期" DATE
        )
        ```

4.  **執行調用：**
    ```json
    {
      "parent": {
        "page_id": "我的工作區頁面ID"
      },
      "title": "新專案追蹤",
      "schema": "CREATE TABLE (\"專案名稱\" TITLE, \"專案經理\" PEOPLE, \"狀態\" SELECT('規劃中':gray, '進行中':blue, '已完成':green), \"截止日期\" DATE)"
    }
    ```

**最終效果：**
一個結構完整、屬性齊全的新資料庫在您指定的位置被建立。您和您的團隊可以立即開始使用它來管理專案，無需任何手動設定。

---

### 指令範例 2：為現有資料庫增加新功能

> **您說：**
> 「在我剛才建立的『新專案追蹤』資料庫裡，再幫我加一個『預算』欄位，要用美元格式的數字。」

**Manus 的內心戲：**

1.  **分析意圖：** 使用者想要修改一個現有資料庫的結構，為它新增一個欄位。

2.  **選用工具：** 這是 `notion-update-data-source` 工具的職責。

3.  **組合參數與執行：**
    *   首先，我需要獲取「新專案追蹤」資料庫的 `data_source_id`。這可能需要一次 `notion-search` 和 `notion-fetch`。
    *   然後，我會調用 `notion-update-data-source`。
    *   `data_source_id` 參數就是我剛獲取的 ID。
    *   `statements` 參數將包含一個 `ADD COLUMN` 語句，用於定義新的「預算」欄位及其類型。

4.  **執行調用：**
    ```json
    {
      "data_source_id": "新專案追蹤資料庫的data_source_id",
      "statements": "ADD COLUMN \"預算\" NUMBER FORMAT 'dollar'"
    }
    ```

**最終效果：**
「新專案追蹤」資料庫無縫地增加了一個「預算」欄位，您現有的所有專案記錄都擁有了這個新屬性，可以開始填寫預算資訊了。整個過程對現有資料毫無影響。

---

通過這些範例，您可以看到，Manus 不僅僅是您的執行者，更是您的 Notion 架構師。您可以透過自然語言，輕鬆地設計、建立和維護複雜的資訊系統，將 Notion 的潛力發揮到極致。
