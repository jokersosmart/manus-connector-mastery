**作者：** Joker

---

本學習手冊旨在幫助您快速上手 Manus Notion 連結器，並掌握其核心技能。我們將透過一系列清晰的學習目標和實踐練習，引導您從基礎操作開始，逐步建立自動化思維，最終能夠獨立解決實際工作中的重複性問題。

與長篇大論的指南不同，本手冊更注重「做中學」。每個章節都將圍繞一個核心技能展開，並提供具體的練習題，讓您在動手操作中鞏固所學知識。準備好開始您的 Notion 自動化之旅了嗎？讓我們從第一個關鍵技能開始。

## 單元一：建立您的第一個自動化頁面

**學習目標：**
- 了解 `create-pages` 工具的基本用法。
- 能夠使用 Manus 指令在 Notion 中建立一個新的、帶有標題和內容的頁面。

**核心技能：** `notion-create-pages`

### 技能解析

`notion-create-pages` 是您將要學習的第一個、也是最基本的一個工具。它的作用正如其名：在您的 Notion 工作區中建立新頁面。您可以指定頁面的標題 (`title`) 和頁面的內容 (`content`)。

想像一下，您正在對 Manus 下達指令，這個指令會被轉換成一個 JSON 物件，看起來像這樣：

```json
{
  "pages": [
    {
      "properties": {
        "title": "我的第一個自動化頁面"
      },
      "content": "Hello, Automation World!"
    }
  ]
}
```

Manus 會將這個結構化的指令發送給 Notion，Notion 就會為您建立一個標題為「我的第一個自動化頁面」，內容為「Hello, Automation World!」的新頁面。

### 實踐練習

現在，輪到您來嘗試了。請嘗試向 Manus 下達一個指令，來完成以下任務。

**練習題 1.1：建立一個購物清單**

> **任務：** 請 Manus 幫您建立一個新的 Notion 頁面，標題為「本週購物清單」，並在頁面內容中包含以下三個待辦事項：
> - [ ] 牛奶
> - [ ] 雞蛋
> - [ ] 麵包

**思考提示：**
- 您需要告訴 Manus 頁面的「標題」是什麼。
- 您需要告訴 Manus 頁面的「內容」是什麼，注意待辦事項的 Markdown 格式。

**您的指令可能看起來像這樣：**

> 「幫我建立一個 Notion 頁面，標題是『本週購物清單』，內容是：\n- [ ] 牛奶\n- [ ] 雞蛋\n- [ ] 麵包」

**預期結果：**
在您的 Notion 私人頁面區域，將會出現一個名為「本週購物清單」的新頁面，點開後，您會看到三個已經列好的待辦事項。

恭喜您！您已經掌握了最基本的 Notion 自動化技能。在下一個單元中，我們將學習如何將頁面建立在指定的資料庫中，讓您的自動化變得更加井然有序。

## 單元二：將想法歸入資料庫

**學習目標：**
- 了解在資料庫中建立頁面與建立獨立頁面的區別。
- 學習如何獲取並使用 `data_source_id` 將新頁面新增至指定的資料庫。
- 掌握如何為新頁面設定屬性 (Properties)。

**核心技能：** `notion-fetch`, `notion-create-pages`

### 技能解析

在 Notion 中，資料庫裡的每一個「項目」(Item) 本質上也是一個頁面，但它比獨立頁面多了「屬性」。要將一個新頁面準確地放入特定資料庫，您需要先告訴 Manus 兩件事：

1.  **目標資料庫是哪一個？**
2.  **這個新頁面的各項屬性值是什麼？**

這就需要 `notion-fetch` 和 `notion-create-pages` 的協同工作。

1.  **使用 `notion-fetch` 獲取「門牌號碼」**
    在您對一個資料庫進行任何操作前，必須先用 `fetch` 工具「看」一下這個資料庫的結構。您可以把資料庫的 URL 或 ID 交給 `fetch`，它會返回一個包含 `data_source_id` 的詳細報告。這個 `data_source_id` 就像是資料庫的精確「門牌號碼」，後續的 `create-pages` 工具就需要用它來定位。

2.  **使用 `create-pages` 帶上「門牌號碼」和「屬性」**
    當您使用 `create-pages` 時，除了頁面內容，您還需要提供 `parent` 物件來指定「門牌號碼」，並在 `properties` 物件中設定好符合該資料庫結構的各項屬性。

    ```json
    {
      "parent": {
        "data_source_id": "f336d0bc-b841-465b-8045-024475c079dd" // 這就是從 fetch 獲取的門牌號碼
      },
      "pages": [
        {
          "properties": {
            "任務名稱": "學習 Notion 自動化", // 屬性名稱需與資料庫定義一致
            "狀態": "進行中"
          }
        }
      ]
    }
    ```

### 實踐練習

假設您有一個「任務列表」資料庫，它有兩個屬性：「任務名稱」(標題) 和「狀態」(單選)。

**練習題 2.1：新增一個學習任務**

> **任務：** 請 Manus 在您的「任務列表」資料庫中，新增一個任務，任務名稱為「完成學習手冊第二單元」，並將其狀態設定為「進行中」。

**思考提示：**
-   在新增任務前，您首先需要知道「任務列表」資料庫的 `data_source_id`。所以第一步應該是什麼？
-   您的指令需要包含目標資料庫、任務的名稱以及任務的狀態。

**您的指令可能看起來像這樣（分兩步）：**

> **第一步：** 「幫我看一下我的『任務列表』資料庫的結構。」 (觸發 `fetch`)

> **第二步 (在得到 `data_source_id` 後)：** 「在我的『任務列表』資料庫裡新增一個任務，名稱是『完成學習手冊第二單元』，狀態設定為『進行中』。」 (觸發 `create-pages`)

**預期結果：**
打開您的「任務列表」資料庫，您會看到一個新的項目被建立，其「任務名稱」和「狀態」兩個欄位已經被準確地填充好了。

做得很好！您現在已經學會了如何與結構化的資料庫進行互動。接下來的單元，我們將學習如何修改已經存在的頁面或任務，讓您的自動化能力更加完整閉環更加完整。

## 單元三：動態更新您的世界

**學習目標：**
- 了解 `update-page` 工具的基本用法。
- 能夠使用 Manus 指令修改一個現有頁面的屬性。
- 學習如何使用 `selection_with_ellipsis` 來定位並修改頁面中的特定內容。

**核心技能：** `notion-update-page`

### 技能解析

世界是變化的，您的 Notion 工作區也一樣。`notion-update-page` 工具賦予您動態修改已存在頁面的能力，讓您的自動化系統能夠響應變化。

更新操作主要分為兩類：**更新屬性**和**更新內容**。

1.  **更新屬性 (`update_properties`)**
    這是最常見的更新操作。例如，將一個任務的狀態從「待辦」改為「完成」，或者修改一個專案的截止日期。您只需要指定要更新的頁面 ID，以及要修改的屬性名稱和新值。

    ```json
    {
      "page_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "command": "update_properties",
      "properties": {
        "狀態": "已完成"
      }
    }
    ```

2.  **更新內容 (`replace_content_range`, `insert_content_after`)**
    有時候，您需要修改的不是屬性，而是頁面正文裡的某段話。這時，您需要一個方法來告訴 Manus 您想修改的到底是「哪一段話」。`selection_with_ellipsis` 參數就是為此而生的。

    它像一個「文字定位器」，您只需要提供想定位文字的**開頭部分**和**結尾部分**，中間用 `...` 連接。Manus 就能精準地找到這段內容，然後進行替換或在它後面插入新內容。

    ```json
    {
      "page_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "command": "insert_content_after",
      "selection_with_ellipsis": "這是會議的總結...行動計劃如下：",
      "new_str": "\n\n**補充說明：** 會議氣氛良好。"
    }
    ```

### 實踐練習

**練習題 3.1：完成一個任務**

> **任務：** 在您的「任務列表」資料庫中，找到名為「完成學習手冊第二單元」的任務，並將其「狀態」屬性修改為「已完成」。

**思考提示：**
-   您需要先知道這個任務頁面的 ID。您可以透過 `search` 工具找到它。
-   您的指令需要明確指出要更新哪個頁面的哪個屬性，以及新的值是什麼。

**您的指令可能看起來像這樣：**

> 「幫我把『任務列表』裡，那個叫做『完成學習手冊第二單元』的任務，狀態改成『已完成』。」

**練習題 3.2：為購物清單添加備註**

> **任務：** 找到您在單元一建立的「本週購物清單」頁面。在「雞蛋」那個待辦事項的後面，加上「(要買有機的)」。

**思考提示：**
-   您需要用 `selection_with_ellipsis` 來定位到「- [ ] 雞蛋」這一行。
-   您是想替換它，還是在它後面追加內容？這決定了您要使用的 `command`。

**您的指令可能看起來像這樣：**

> 「在我的『本週購物清單』頁面裡，找到『- [ ] 雞蛋』這一行，把它改成『- [ ] 雞蛋 (要買有機的)』。」 (這會觸發 `replace_content_range`)

**預期結果：**
打開您的「任務列表」，「完成學習手冊第二單元」的狀態變成了「已完成」。打開「本週購物清單」，「雞蛋」後面出現了新的備註。

## 單元四：綜合挑戰 —— 打造您的每日日誌系統

**學習目標：**
- 融會貫通 `create-pages` 和 `update-page` 等核心技能。
- 學習設計一個簡單但實用的自動化工作流。
- 體驗自動化如何簡化日常重複性任務。

**核心技能：** 綜合運用 `notion-create-pages`, `notion-update-page`

### 挑戰描述

我們每天都會產生新的想法、完成新的任務、遇到新的問題。一個「每日日誌 (Daily Journal)」是記錄和反思這一切的絕佳工具。在這個綜合挑戰中，您將利用 Manus Notion 連結器，從零開始，打造一個半自動化的每日日誌系統。

**您的目標是建立一個流程，包含以下兩個核心動作：**

1.  **開啟新的一天：** 一個指令就能在您的「日誌」資料庫中，建立一個以當天日期為標題的新頁面，並自動包含「今日完成的任務」、「新的想法」、「遇到的問題」三個章節標題。
2.  **記錄當日小結：** 一個指令就能將您在其他地方（例如一個名為「靈感收集箱」的頁面）隨手記錄的想法，自動抓取並追加到當天日誌的「新的想法」章節中。

### 實踐練習

**準備工作：**

1.  **手動建立資料庫：** 請您先在 Notion 中手動建立一個名為「我的日誌」的資料庫，它只需要一個預設的「名稱」(標題) 屬性即可。
2.  **手動建立頁面：** 請您再手動建立一個名為「靈感收集箱」的普通頁面。

**練習題 4.1：自動化「開啟新的一天」**

> **任務：** 請向 Manus 下達一個指令，在「我的日誌」資料庫中，建立一個以今天日期（例如「2026-02-24」）為標題的新頁面，並讓頁面內容自動包含以下結構：
> ```markdown
> ## 今日完成的任務
> 
> 
> ## 新的想法
> 
> 
> ## 遇到的問題
> 
> ```

**思考提示：**
-   您需要使用 `create-pages` 工具。
-   您需要先 `fetch`「我的日誌」資料庫以獲取 `data_source_id`。
-   頁面的標題是什麼？頁面的內容 (`content`) 又是什麼？

**練習題 4.2：隨手記錄想法**

> **任務：** 現在，請想像您在一天中靈光一閃，需要快速記錄。請向 Manus 下達指令，在「靈感收集箱」頁面裡，追加一行文字：「或許可以開發一個自動整理 Notion 筆記的工具」。

**思考提示：**
-   您需要使用 `update-page` 工具的 `insert_content_after` 或 `replace_content` (如果頁面是空的) 命令。
-   您需要先知道「靈感收集箱」頁面的 ID。

**練習題 4.3：自動化「記錄當日小結」**

> **任務：** 在一天結束時，您需要整理思緒。請向 Manus 下達一個指令，將「靈感收集箱」裡的所有內容，移動到您今天建立的日誌頁面的「新的想法」章節下方。

**思考提示：**
-   這是一個多步驟的流程，需要 Manus 協調多個工具：
    1.  用 `fetch` 讀取「靈感收集箱」的內容。
    2.  用 `update-page` 和 `insert_content_after`，將讀取到的內容追加到日誌頁面中。
    3.  (可選) 用 `update-page` 和 `replace_content`，清空「靈感收集箱」，為第二天做準備。
-   您需要用 `selection_with_ellipsis` 來精準定位到「## 新的想法」這個標題。

**您的指令可能看起來像這樣：**

> 「幫我把『靈感收集箱』裡的內容，全部追加到我今天的日誌頁面裡，放在『新的想法』標題下面。然後清空靈感收集箱。」

**預期結果：**
您成功地用幾個指令，完成了一套每日日誌的記錄與整理流程。您今天建立的日誌頁面中，成功地從別處同步過來了您的想法筆記。
