# 「Prisma Postgres」連結器使用者操作手冊

**作者：Joker**

本手冊為您——終端使用者——而設計。您不需要學習複雜的程式碼或 API，只需透過自然的日常對話，就能讓 Manus 為您管理強大的 Prisma Postgres 資料庫。本手冊將以一個個具體的任務為導向，向您展示如何下達指令，並揭示 Manus 在背後為您做了什麼。

---

## 核心理念：您說話，Manus 工作

您與 Manus 的互動就像與一位技術高超的資料庫管理員 (DBA) 對話。您提出需求，Manus 會理解您的意圖，並自動選擇最合適的工具來完成任務。您唯一需要關心的，就是「您想做什麼」。

---

## 任務導覽

### 任務一：我想知道我有多少個資料庫

- **您的目標**：快速盤點一下您在 Prisma 帳戶下所有的資料庫實例。
- **您的指令**：「嘿 Manus，幫我列出我在 Prisma 上的所有資料庫。」
- **背後原理 (Manus 如何工作)**：
  1.  Manus 聽到「列出」、「所有資料庫」等關鍵字，判斷出您的意圖是「盤點資源」。
  2.  它會選擇 `ListDatabasesTool` 這個最符合您需求的工具。
  3.  由於這個工具不需要任何額外資訊，Manus 會直接執行它。
  4.  最後，Manus 將工具返回的資料庫列表，以清晰的卡片形式呈現給您。

### 任務二：我想了解一個陌生資料庫的內部結構

- **您的目標**：接手一個舊專案，需要快速看懂其資料庫裡有哪些資料表和欄位。
- **您的指令**：「幫我看看 ID 是 `db_clxm8p4a1000008l9g1j2h3k4` 的這個資料庫，它的結構長什麼樣子？」
- **背後原理 (Manus 如何工作)**：
  1.  Manus 識別出「看...結構」、「分析...結構」等指令，並在您的話中找到了關鍵資訊 `db_clxm8p4a1000008l9g1j2h3k4`。
  2.  它選擇了 `IntrospectSchemaTool`，因為這個工具專門用於探測資料庫結構。
  3.  它將您提供的 ID `db_clxm8p4a1000008l9g1j2h3k4` 填入 `IntrospectSchemaTool` 所需的 `database_id` 參數中。
  4.  執行工具後，Manus 將返回的 Prisma Schema 程式碼呈現給您。

### 任務三：我想從資料庫裡快速查個資料

- **您的目標**：不需要打開任何資料庫軟體，直接透過聊天查詢業務數據。
- **您的指令**：「幫我在 `production-database` 上查一下，`User` 資料表裡總共有幾筆資料？」
- **背後原理 (Manus 如何工作)**：
  1.  Manus 偵測到「查一下」、「...表裡有多少資料」等查詢意圖。
  2.  它會先在您的 Prisma 帳戶中尋找名為 `production-database` 的資料庫，以獲取其 ID。
  3.  接著，它將您的自然語言查詢「`User` 資料表裡總共有幾筆資料」轉換為一句標準的 SQL 語句：`SELECT COUNT(*) FROM "User";`。
  4.  然後，Manus 呼叫 `ExecuteSqlQueryTool`，並將資料庫 ID 和轉換後的 SQL 語句分別填入 `database_id` 和 `query` 參數。
  5.  最後，將查詢結果以易於理解的方式回覆給您。

### 任務四：我需要為我的新應用程式取得資料庫連線資訊

- **您的目標**：安全地為您的應用程式取得一個可以用來連接資料庫的密碼（連接字串）。
- **您的指令**：「我需要一個 `staging-database` 的連接字串來做測試。」
- **背後原理 (Manus 如何工作)**：
  1.  Manus 理解到「連接字串」是您需要的核心資源。
  2.  它會先找到 `staging-database` 的 ID。
  3.  然後呼叫 `CreateConnectionStringTool`，並傳入該 ID。
  4.  工具會生成一個新的、安全的連接字串，Manus 隨後將其展示給您。出於安全考量，這個字串只會顯示一次。

### 任務五：我準備要發布新版了，得先做個備份以防萬一

- **您的目標**：在進行高風險操作（如上線、資料遷移）之前，為您的重要資料庫建立一個即時的時間點備份。
- **您的指令**：「在發布之前，先幫我把 `production-database` 備份一下。」
- **背後原理 (Manus 如何工作)**：
  1.  「備份」是個非常明確的指令。Manus 會立即鎖定 `CreateBackupTool`。
  2.  它找到 `production-database` 的 ID，並將其作為 `database_id` 參數傳遞給工具。
  3.  執行後，Manus 會告訴您備份任務已經開始，並在完成後通知您，同時提供該備份的唯一 ID。

### 任務六：糟糕！線上出問題了，快用昨天的備份還原！

- **您的目標**：當災難發生時，快速從一個已知的良好備份中，恢復出一個全新的、乾淨的資料庫實例，以盡快恢復服務。
- **您的指令**：「快！用 ID 是 `backup_clx...` 的這個備份，幫我還原一個叫做 `hotfix-db` 的新資料庫！」
- **背後原理 (Manus 如何工作)**：
  1.  「還原」、「從備份」是強烈的意圖信號。Manus 會選擇 `CreateRecoveryTool`。
  2.  它從您的指令中提取了 `backup_id` (`backup_clx...`) 和新資料庫的名稱 `hotfix-db`。
  3.  Manus 呼叫工具，並填入相應的參數。
  4.  一個全新的資料庫實例將開始建立，其資料狀態與您指定的備份完全一致。Manus 會全程向您報告進度。

---

透過以上任務，您可以看到，「Prisma Postgres」連結器將強大的資料庫管理能力，轉化為您與 Manus 之間的簡單對話。您只需專注於您的目標，剩下的交給 Manus 就好。
