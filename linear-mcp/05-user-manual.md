# 「Linear」連結器使用者操作手冊

**作者：Joker**

---

本手冊專為終端使用者設計，旨在展示如何透過直觀的自然語言指令，驅動 Manus Agent 在 Linear 平台上執行各種專案管理任務。我們將以任務為導向，提供具體的指令範例，並解釋 Manus 如何將這些指令轉換為對應的工具調用。

## 核心概念：從「意圖」到「行動」

Manus Agent 的核心是理解您的「意圖」(Intent)，並將其轉化為一系列具體的「行動」(Actions)，也就是工具調用。您不需要學習複雜的程式語法，只需要像與同事對話一樣，清楚地表達您的需求即可。

---

## 任務一：快速回報一個 Bug

作為一名開發者或測試人員，快速地將發現的 Bug 記錄到 Linear 中是每天的例行公事。讓我們看看如何透過 Manus 來簡化這個流程。

### 情境

您在測試公司網站時，發現點擊登入按鈕後，頁面會重新導向到一個錯誤的頁面。您需要立即在「Frontend」團隊中建立一個高優先度的 Bug 報告，並指派給自己。

### 您的指令

> "嘿 Manus，幫我在 Linear 的 Frontend 團隊回報一個 Bug。標題是「登入按鈕重新導向錯誤」，優先度設為「緊急」，然後指派給我自己。"

### Manus 的執行過程

Manus Agent 在收到您的指令後，會進行以下拆解與執行：

1.  **意圖識別:** Agent 識別出您的意圖是「建立一個新的 Linear 議題」。
2.  **實體提取:** Agent 從您的指令中提取出關鍵資訊：
    *   團隊 (Team): "Frontend"
    *   議題類型 (Type): "Bug" (雖然 Linear API 中沒有直接的 `type` 欄位，但這可以作為標籤或標題的一部分)
    *   標題 (Title): "登入按鈕重新導向錯誤"
    *   優先度 (Priority): "緊急" (Urgent)
    *   指派對象 (Assignee): "我自己" (@me)
3.  **工具選擇:** Agent 選擇 `linear.create_issue` 工具來執行此任務。
4.  **參數對應:** Agent 將提取出的實體對應到工具所需的參數。

**最終的工具調用:**

```json
{
  "tool": "linear.create_issue",
  "team_id": "FE", // Agent 會查詢到 Frontend 團隊的 Key 是 FE
  "title": "[Bug] 登入按鈕重新導向錯誤",
  "priority": "Urgent",
  "assignee_id": "USER-001" // Agent 會查詢到您自己的 User ID
}
```

透過這樣的方式，您僅用一句話就完成了過去需要多次點擊和輸入才能完成的操作。
