# 「Linear」連結器從新手到精通指南

**作者：Joker**

---

歡迎來到「Linear」連結器的學習之旅！本指南將帶領您從一位對 Linear 與 Manus 整合一無所知的新手，逐步成長為能夠靈活運用此連結器解決複雜專案管理問題的專家。我們將學習路徑分為四個階段：**基礎入門**、**核心功能**、**進階技巧**與**實戰應用**。

## 第一階段：基礎入門 (Novice)

在這個階段，您的目標是了解 Linear 的基本概念，並學會如何透過 Manus Agent 執行最基礎的查詢操作。

### 1. 認識 Linear 核心概念

在開始使用連結器之前，您必須先熟悉 Linear 的幾個核心物件。這將幫助您理解工具的參數與回傳值所代表的意義。

*   **議題 (Issue):** 專案中的一個工作單元，可以是一個 bug、一個新功能、或一項任務。
*   **專案 (Project):** 一系列議題的集合，通常用來追蹤一個較大的目標或產品發布。
*   **週期 (Cycle):** 一個固定時間長度的開發衝刺 (Sprint)，團隊會在此期間內完成特定數量的議題。
*   **團隊 (Team):** 在 Linear 中一起協作的一群使用者。

### 2. 您的第一個指令：查詢團隊資訊

讓我們從一個最簡單的指令開始。您不需要提供任何參數，就可以獲取您在 Linear 中所屬的所有團隊列表。

**自然語言指令:**
> "列出我在 Linear 上的所有團隊。"

**Manus Agent 執行:**
```json
{
  "tool": "linear.get_teams"
}
```

藉由這個簡單的指令，您可以驗證 Manus Agent 是否已成功連接到您的 Linear 帳號，並為接下來的學習打下基礎。

### 3. 查詢您的任務

接下來，嘗試查詢指派給您的所有待辦事項。這是一個非常實用的指令，可以幫助您快速掌握目前的工作負載。

**自然語言指令:**
> "我在 Linear 上有哪些待辦事項？"

**Manus Agent 執行:**
```json
{
  "tool": "linear.get_issues",
  "assignee_id": "@me", // @me 是一個特殊的識別符，代表當前使用者
  "status": "Todo"
}
```

## 第二階段：核心功能 (Adept)

熟悉基礎查詢後，您將學習如何使用變更工具 (Mutation Tools) 來實際影響 Linear 中的資料，並掌握更複雜的查詢技巧。

### 1. 建立您的第一個議題

這是您第一次使用連結器在 Linear 中建立資料。您需要提供議題的標題、所屬團隊，以及可選的描述。

**自然語言指令:**
> "在 Frontend 團隊建立一個標題為「修復首頁跑版問題」的議題。"

**Manus Agent 執行:**
```json
{
  "tool": "linear.create_issue",
  "team_id": "FE", // 假設 Frontend 團隊的 key 是 FE
  "title": "修復首頁跑版問題",
  "description": "在 Safari 瀏覽器中，首頁的 header 會蓋住部分內容。"
}
```

### 2. 更新議題狀態

當您開始處理一個議題時，您可以使用 `update_issue` 工具來變更它的狀態。

**自然語言指令:**
> "將議題 FE-123 的狀態更新為「進行中」。"

**Manus Agent 執行:**
```json
{
  "tool": "linear.update_issue",
  "issue_id": "FE-123",
  "status": "In Progress"
}
```

### 3. 為議題加上評論

與團隊成員的溝通至關重要。您可以使用 `create_comment` 工具在議題上留下您的想法。

**自然語言指令:**
> "在議題 FE-123 上留言：「我已經重現這個問題，正在著手修復。」"

**Manus Agent 執行:**
```json
{
  "tool": "linear.create_comment",
  "issue_id": "FE-123",
  "body": "我已經重現這個問題，正在著手修復。"
}
```
