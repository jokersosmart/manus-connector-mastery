# Wrike 連接器學習路徑：從新手到精通

**作者: Joker**

---

## 前言：釋放 Wrike 的自動化潛能

Wrike 不僅僅是一個專案管理工具，它是一個強大的工作作業系統。當與 Manus 的自動化能力結合時，其潛力將被無限放大。本指南旨在為您提供一條清晰的學習路徑，無論您是初次接觸 Wrike 自動化的新手，還是尋求進階技巧的專家，都能從中獲益，最終將 Wrike 連接器運用自如，打造高效、智慧的自動化工作流程。

---

## 第一階段：基礎概念與入門 (Novice)

**目標：** 理解 Wrike 連接器的核心價值，並能執行基本的讀取與創建操作。

### 1. 核心概念

- **什麼是 Wrike 連接器？** 它是 Manus 與 Wrike 之間的橋樑，讓您能用自然語言指令來操作 Wrike 裡的資料。
- **核心資源：** 了解 Wrike 的基本組成單位：
  - **任務 (Tasks):** 工作的最小執行單元。
  - **專案 (Projects):** 由多個任務組成的集合，有明確的目標和時間表。
  - **資料夾 (Folders):** 用於組織和分類任務與專案的容器。
  - **空間 (Spaces):** 團隊或部門的專屬工作區，包含相關的資料夾和專案。
- **ID 的重要性：** 在 Wrike 中，每一個資源（任務、專案、使用者等）都有一個獨一無二的 ID。這是您與連接器互動時，精確指定操作對象的關鍵。

### 2. 首次探索：查詢資訊

- **學習 `wrike.tasks.search`:** 這是您最先需要掌握的工具。嘗試用它來：
  - 找出「我負責的未完成任務」。
  - 列出「特定專案中的所有任務」。
  - 搜尋標題包含「報告」的任務。
- **學習 `wrike.folders.list`:** 了解您的 Wrike 工作區的結構。
  - 列出根目錄下的所有「空間」。
  - 檢視某個專案下的所有子資料夾。

### 3. 首次互動：創建任務

- **學習 `wrike.tasks.create`:** 這是最有成就感的一步。嘗試：
  - 在您的個人待辦事項資料夾中，為自己創建一個任務。
  - 在一個共享的專案中，創建一個任務並指派給同事。

**階段性成果：** 您能夠使用 Manus 指令查詢 Wrike 中的任務和專案，並能成功創建新的任務。

## 第二階段：核心功能與流程自動化 (Intermediate)

**目標：** 掌握修改與關聯操作，並能將多個步驟串聯成一個簡單的自動化流程。

### 1. 修改現有工作

- **學習 `wrike.tasks.update`:** 任務的狀態是動態變化的。學習如何：
  - 將一個任務的狀態從「進行中」改為「已完成」。
  - 為一個現有任務新增或移除負責人。
  - 修改任務的標題或描述。
- **學習 `wrike.comments.create`:** 溝通是協作的關鍵。嘗試：
  - 在一個任務下方新增評論，並用 `@` 提及同事。

### 2. 建立你的第一個自動化流程

- **場景：每日站會報告**
  - **步驟 1:** 使用 `wrike.tasks.search` 找出團隊成員昨日完成的所有任務。
  - **步驟 2:** 使用 `wrike.tasks.search` 找出他們今天計畫要做的任務。
  - **步驟 3:** 將這兩份清單整理後，使用 `wrike.comments.create` 發佈到團隊的每日站會任務中。

- **場景：新客戶專案啟動**
  - **步驟 1:** 使用 `wrike.folders.create` 根據範本創建一個新的客戶專案。
  - **步驟 2:** 使用 `wrike.tasks.create` 在新專案中，自動生成一系列標準的啟動任務（如：召開啟動會議、簽署合約、建立共享資料夾）。
  - **步驟 3:** 將這些任務指派給對應的團隊成員。

**階段性成果：** 您能夠編寫一個包含 2-3 個步驟的 Manus 腳本，自動化一個重複性的 Wrike 工作流程。

## 第三階段：進階技巧與整合 (Advanced)

**目標：** 靈活運用所有 Wrike 工具，並能將其與其他 MCP 連接器結合，實現跨系統的複雜自動化。

### 1. 精通附件與結構操作

- **學習 `wrike.attachments.upload`:** 自動將產出的文件（如報告、圖片）附加到相關的 Wrike 任務中。
- **學習 `wrike.folders.create` 的進階用法:** 創建包含完整結構的專案，包含子專案和資料夾。

### 2. 跨系統整合

- **場景：從客服系統到產品開發**
  - **觸發:** 當 `Zendesk` 連接器偵測到一個新的「Bug 回報」工單。
  - **步驟 1:** 使用 `Zendesk` 工具獲取工單的詳細資訊。
  - **步驟 2:** 使用 `wrike.tasks.create` 在工程團隊的「Bug 修復」專案中，自動創建一個新的 Bug 任務，並將 Zendesk 工單的描述和連結帶入。
  - **步驟 3:** 使用 `wrike.comments.create` 在 Wrike 任務下方留言，通知產品經理。

- **場景：從 GitHub 到專案追蹤**
  - **觸發:** 當 `GitHub` 連接器偵測到一個新的 `Pull Request` 被合併到主分支。
  - **步驟 1:** 獲取 PR 的標題和關聯的 Issue 編號。
  - **步驟 2:** 使用 `wrike.tasks.search` 找到對應 Wrike 中開發任務。
  - **步驟 3:** 使用 `wrike.tasks.update` 自動將該任務的狀態更新為「待測試」，並將 PR 的連結附加到任務中。

**階段性成果：** 您能夠設計並實現一個跨越至少兩個不同服務（如 GitHub, Zendesk, Google Drive）的自動化工作流程，並以 Wrike 作為任務管理的核心樞紐。

## 第四階段：專家級應用與策略 (Master)

**目標：** 不僅是工具的使用者，更是自動化策略的設計者和推動者。

### 1. 動態與條件式流程

- **設計包含邏輯判斷的流程：** 例如，根據任務的優先級（高、中、低），自動將其指派給不同的團隊或成員。
- **設計排程與監控流程：** 例如，設定一個每日執行的 Manus 腳本，檢查所有「高優先級」但超過 3 天未更新的任務，並自動在任務下留言提醒負責人。

### 2. 成為組織內的自動化顧問

- **分析團隊痛點：** 主動觀察並訪談團隊成員，找出可以被 Wrike 自動化解決的瓶頸。
- **設計與分享範本：** 將您創建的優秀自動化流程打包成可重用的 Manus 腳本或技能，分享給整個組織。
- **推廣與賦能：** 舉辦內部工作坊，教授其他人如何使用 Wrike 連接器，將自動化的思維模式傳遞開去。

**階段性成果：** 您能夠為您的團隊或組織，規劃整體的 Wrike 自動化策略，並透過賦能他人，將效率提升擴散到整個組織，成為名副其實的「Wrike 自動化大師」。
