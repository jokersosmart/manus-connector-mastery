# PostHog 連結器使用者操作手冊

作者：Joker

---

本手冊旨在為終端使用者提供一份清晰、易懂的操作指南，說明如何透過自然語言指令與 Manus 互動，以利用 PostHog 連結器完成常見的產品分析任務。我們將展示具體的指令範例，並解釋 Manus 如何將這些指令轉換為對應的工具調用，讓您直觀地理解「所說即所得」的強大能力。

## 任務一：監控每日核心指標

**您的目標**：您想知道您的產品昨天有多少新使用者註冊，以及他們主要來自哪些管道。

**您對 Manus 說**：

> 「幫我查詢昨天 `user_signup` 事件的總數，並按『註冊來源』這個屬性進行分組統計。」

**Manus 如何理解與執行**：

Manus 會解析您的指令，並意識到這是一個查詢分析洞察的任務。它會構建一個即時的趨勢分析查詢，並調用 `get_insights` 工具。

- **工具調用**：`get_insights`
- **參數 (示意)**：
  ```json
  {
    "insight_query": {
      "kind": "Trends",
      "series": [{"event": "user_signup"}],
      "date_from": "-1d",
      "date_to": "-1d",
      "breakdown": "signup_source"
    }
  }
  ```

**您得到的結果**：

Manus 會以一個清晰的表格或圖表向您展示結果，例如：

| 註冊來源 | 註冊數量 |
| :--- | :--- |
| Google | 150 |
| Facebook | 80 |
| 直接訪問 | 45 |

## 任務二：分析使用者流失點

**您的目標**：您發現很多使用者開始了購買流程，但最終沒有完成支付。您想知道他們在哪一步放棄了。

**您對 Manus 說**：

> 「幫我建立一個使用者購買漏斗，分析從『查看商品』到『加入購物車』，再到『完成購買』的轉化率。」

**Manus 如何理解與執行**：

Manus 識別出「漏斗」和「轉化率」等關鍵詞，確定這是一個漏斗分析任務。它會將您提到的步驟轉換為事件名稱，並調用 `get_insights` 工具。

- **工具調用**：`get_insights`
- **參數 (示意)**：
  ```json
  {
    "insight_query": {
      "kind": "Funnels",
      "series": [
        {"event": "viewed_product"},
        {"event": "added_to_cart"},
        {"event": "completed_purchase"}
      ]
    }
  }
  ```

**您得到的結果**：

Manus 會返回一個漏斗圖，清晰地顯示每一步的轉化率和流失率，讓您一眼就看出問題所在。

## 任務三：安全地測試新功能

**您的目標**：您團隊開發了一個新的首頁設計，但不確定它是否比舊版更好。您想先讓一小部分使用者體驗新版，收集回饋。

**您對 Manus 說**：

> 「我想為 `new_homepage_design` 這個功能旗標設定一個規則，只對 10% 的使用者啟用它。」

**Manus 如何理解與執行**：

Manus 理解您想要修改一個功能旗標的發布規則。它會調用對應的工具（可能是 `update_feature_flag`，此處為示意）來更新該旗標的設定。

- **工具調用**：`update_feature_flag` (示意)
- **參數 (示意)**：
  ```json
  {
    "flag_key": "new_homepage_design",
    "rollout_percentage": 10
  }
  ```

**您得到的結果**：

Manus 會確認功能旗標已更新。之後，您可以進一步詢問：

> 「幫我看看使用者 `user-beta-tester` 是否能看到 `new_homepage_design`？」

Manus 則會調用 `evaluate_feature_flag` 工具來給您一個確切的「是」或「否」的答案。

## 任務四：深入了解特定使用者群體

**您的目標**：您想對產品的「超級使用者」進行一次訪談，需要先找出他們是誰。

**您對 Manus 說**：

> 「幫我建立一個名為『超級使用者』的群組，條件是過去 30 天內登入超過 20 次，並且發布過至少 5 篇內容。」

**Manus 如何理解與執行**：

Manus 識別出「群組」和「條件」，知道您要建立一個使用者分群 (Cohort)。它會將您的自然語言條件轉換為 PostHog 的群組規則，並調用 `create_cohort` 工具。

- **工具調用**：`create_cohort`
- **參數 (示意)**：
  ```json
  {
    "name": "超級使用者",
    "groups": [
      {
        "properties": [
          {
            "key": "login_count_30d",
            "operator": "gt",
            "value": 20
          },
          {
            "key": "posts_published_30d",
            "operator": "gte",
            "value": 5
          }
        ]
      }
    ]
  }
  ```

**您得到的結果**：

Manus 會告訴您群組已成功建立，並返回該群組的 ID。之後，您可以基於這個群組進行更深入的分析，例如：

> 「幫我看看『超級使用者』這個群組最常使用的三大功能是什麼？」

本手冊展示了 PostHog 連結器的一些核心應用場景。我們鼓勵您用自己的話來描述您的分析需求，Manus 將盡力理解並為您執行。
