# Slack 連結器使用者操作手冊

作者：Joker

## 歡迎使用 Manus for Slack！

本手冊是為您——終端使用者——所設計。在這裡，您不需要了解任何程式碼或複雜的技術細節。您只需要學習如何像與真人同事對話一樣，透過簡單的自然語言指令，讓 Manus 成為您在 Slack 中的得力助手。我們將透過一系列貼近日常工作的任務範例，向您展示如何輕鬆駕馭這個強大的工具。

---

## 核心理念：您說話，Manus 工作

您與 Manus 互動的核心方式，就是在 Slack 的任何對話串 (thread) 中，用 `@manus` 標記它，然後用一句話告訴它您需要什麼。Manus 會讀取您前後的對話內容，理解您的意圖，並將其轉換為具體的工具調用來完成任務。

--- 

## 任務導向範例

### 任務一：發送通知與公告

**情境**：您需要通知專案團隊，今天下午三點有一個臨時的緊急會議。

**您的指令**：

> @manus 請在 `#project-hydra` 頻道通知大家，今天下午三點在 301 會議室有緊急會議，討論伺服器遷移問題。

**Manus 如何理解與執行**：

Manus 會識別出幾個關鍵要素：

-   **目的地**：`#project-hydra` 頻道
-   **內容**：「今天下午三點在 301 會議室有緊急會議，討論伺服器遷移問題。」

然後，它會調用內部的 `send_message` 工具，就像這樣：

```
工具調用: send_message(destination="#project-hydra", content="通知大家，今天下午三點在 301 會議室有緊急會議，討論伺服器遷移問題。")
```

**結果**：指定的訊息會立刻出現在 `#project-hydra` 頻道中。

---

### 任務二：快速建立協作空間

**情境**：您和幾位同事臨時需要一個地方討論下個季度的腦力激盪，不希望打擾主要頻道。

**您的指令**：

> @manus 幫我開一個私密的頻道，叫做 `q3-brainstorming`，然後把 @danny 和 @emily 拉進來。

**Manus 如何理解與執行**：

Manus 會將這個任務拆解成兩個步驟：

1.  **建立頻道**：識別出頻道名 `q3-brainstorming` 和私密屬性。
2.  **邀請成員**：識別出需要被邀請的成員 `@danny` 和 `@emily`。

它會依序調用 `create_channel` 和 `invite_to_channel` (假設的工具)：

```
// 第一步
工具調用: create_channel(name="q3-brainstorming", is_private=true)

// 第二步 (使用上一步的結果)
工具調用: invite_to_channel(channel="CABC123_ID_FROM_STEP_1", users=["@danny", "@emily"])
```

**結果**：一個新的私密頻道被建立，且指定的成員會被自動邀請加入。

---

### 任務三：查詢您需要的資訊

**情境**：您需要聯絡一位不常接觸的同事，但忘記了他的職位和聯絡方式。

**您的指令**：

> @manus 我想找一下 `@frank` 的資料，他的職位是什麼？

**Manus 如何理解與執行**：

Manus 知道您的意圖是獲取使用者資訊。

-   **目標使用者**：`@frank`

它會調用 `get_user_info` 工具，並從回傳的資料中篩選出您要的「職位」資訊。

```
工具調用: get_user_info(user_id="UFRANK_ID")
// Manus 會處理回傳的 JSON，並以自然語言回覆您
```

**結果**：Manus 會在對話串中直接回覆您：「Frank 的職位是『資深產品設計師』。」

---

### 任務四：跨工具的超級任務

**情境**：團隊在 Slack 熱烈討論完一個新功能後，您希望將結論記錄下來，並安排一個後續的追蹤會議。

**您的指令**：

> @manus 總結一下我們剛才關於「使用者標籤系統」的討論，把重點記錄到 Notion 的「產品會議」資料庫。然後，幫我預約下週三下午兩點，跟 @susan 和 @tom 開一個 30 分鐘的會議，討論開發時程。

**Manus 如何理解與執行**：

這是一個複雜的跨平台任務，Manus 會像一個真正的專案經理一樣，按部就班地執行：

1.  **閱讀與總結**：首先，它會閱讀當前的 Slack 對話串，提煉出關於「使用者標籤系統」的關鍵決策和想法。
2.  **記錄到 Notion**：接著，它會切換到 `notion` 連結器，調用 `create_page` 工具，將總結的內容寫入指定的資料庫。
3.  **安排會議**：最後，它會切換到 `google_calendar` (或類似的日曆) 連結器，檢查 `@susan` 和 `@tom` 的行事曆，找到空檔並建立會議邀請。

**結果**：您不需要離開 Slack，一個想法就從討論變成了結構化的筆記和具體的行動排程。

## 結語

本手冊中的範例僅僅是冰山一角。Manus 的強大之處在於其靈活性和組合性。請大膽嘗試用您自己的話，向 Manus 提出您在工作上遇到的各種瑣碎、重複或需要跨工具協調的任務。您會驚訝地發現，許多過去需要手動操作半小時的工作，現在可能只需要您的一句話就能完成。

**您的任務**：現在就去一個測試頻道，試著用 `@manus` 標記它，並下達您的第一個指令吧！
