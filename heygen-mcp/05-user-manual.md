# HeyGen 終端使用者操作手冊

**作者：Joker**

---

本手冊旨在為終端使用者提供一份清晰、以任務為導向的操作指南。您將學會如何使用自然的語言與 Manus 溝通，來完成從簡單到複雜的 AI 影片創作任務。對於每個任務，我們都會展示您的指令以及 Manus 在幕後如何將其轉換為精確的 `HeyGen` 工具調用。

## 任務一：我想要快速把一段文字變成影片

這是最常見、最直接的需求。您有一個想法，一段腳本，您希望立刻看到影片結果，不想被任何技術細節所困擾。

#### 您的指令

您只需要告訴 Manus 您想要說的話即可。

> **您說**：「嘿 Manus，幫我用 HeyGen 做個影片，內容是：『AI 正在改變我們創造內容的方式，這是一個激動人心的時代！』」

#### Manus 的行動

當 Manus 收到這個指令時，它會識別出您的核心意圖是「從文本生成影片」，並提取出關鍵的「腳本」內容。然後，它會調用 `HeyGen` 連結器中為此設計的最簡單工具：`generate_simple_video`。

> **Manus 調用**：
> ```json
> {
>   "tool": "heygen.generate_simple_video",
>   "script": "AI 正在改變我們創造內容的方式，這是一個激動人心的時代！"
> }
> ```

**說明**：您的自然語言被精確地映射到了 `script` 參數上。因為您沒有指定任何虛擬人或聲音，Manus 選擇了最高效的「簡易模式」，讓 HeyGen 自動為您搭配最合適的預設選項。

## 任務二：我想要影片裡的人物和聲音更特別一點

當您熟悉了基本操作後，您會希望影片能更好地反映您的品牌形象或個人風格。這時，您需要對影片中的「演員」和「聲音」有更多的控制權。

#### 您的指令

您的指令會變得更具體，包含對角色特徵的描述。

> **您說**：「我想用 HeyGen 做個影片。找一個亞洲女性當主角，用聽起來專業的美式英文男聲。腳本是：『歡迎來到我們的新品發布會。』背景用深灰色。」

#### Manus 的行動

Manus 會將您的指令拆解為幾個步驟：

1.  **尋找資源**：它會先在內部查詢符合「亞洲女性」和「美式英文男聲」的可用資源 ID。
    *   `heygen.list_avatars(ethnicity='asian', gender='female')` -> 得到 `avatar-f-asian-01`
    *   `heygen.list_voices(language='en-US', gender='male')` -> 得到 `voice-en-us-male-01`
2.  **執行生成**：接著，它會將這些 ID 和您的其他要求（腳本、背景色）組裝起來，調用 `generate_custom_video` 工具。

> **Manus 調用**：
> ```json
> {
>   "tool": "heygen.generate_custom_video",
>   "script": "歡迎來到我們的新品發布會。",
>   "avatar_id": "avatar-f-asian-01",
>   "voice_id": "voice-en-us-male-01",
>   "background_color": "#333333"
> }
> ```

**說明**：您指令中的描述詞（`亞洲女性`、`美式英文男聲`、`深灰色`）被 Manus 成功地轉換為了工具所能理解的具體參數（`avatar_id`, `voice_id`, `background_color`）。

## 任務三：我想要做一個有開頭、有結尾的完整故事

對於更複雜的專案，例如產品教學或微電影，您需要將故事拆分成不同的部分或「場景」，並將它們串連起來。

#### 您的指令

您的指令會更像是在對一位製片助理下達一系列的指令，描述一個完整的工作流程。

> **您說**：
> 「我們來用 HeyGen 拍個短片。標題是『我的創作之旅』。
> 1.  **第一幕**：先創建一個場景，讓一個叫 `David` 的虛擬人說：『創作始於一個簡單的想法。』
> 2.  **第二幕**：再創建一個場景，這次讓 `Anna` 說：『但將想法變為現實，需要工具的幫助。』
> 3.  **第三幕**：最後一個場景，讓他們倆一起出現，說：『HeyGen 和 Manus，讓創意無限。』
> 都好了之後，把這三幕按順序組合起來，生成最終影片。」

#### Manus 的行動

Manus 會理解這是一個多步驟的「工作流」。它會按順序、有條不紊地執行一系列工具調用：

1.  **創建場景**：調用 `heygen.create_scene` 三次，獲得三個獨立的場景 ID（例如 `scene-01`, `scene-02`, `scene-03`）。
2.  **填充場景**：依次對每個場景調用 `heygen.add_to_scene`，將您指定的虛擬人和腳本添加進去。
3.  **渲染影片**：最後，調用 `heygen.render_video`，將這三個場景 ID 按您指定的順序傳入，完成最終的影片合成。

> **Manus 調用 (簡化示意)**：
> ```json
> // 步驟 1: 創建場景
> {"tool": "heygen.create_scene"} -> "scene-01"
> {"tool": "heygen.create_scene"} -> "scene-02"
> {"tool": "heygen.create_scene"} -> "scene-03"

> // 步驟 2: 填充場景
> {"tool": "heygen.add_to_scene", "scene_id": "scene-01", ...}
> {"tool": "heygen.add_to_scene", "scene_id": "scene-02", ...}
> {"tool": "heygen.add_to_scene", "scene_id": "scene-03", ...}

> // 步驟 3: 渲染影片
> {
>   "tool": "heygen.render_video",
>   "scene_ids": ["scene-01", "scene-02", "scene-03"],
>   "title": "我的創作之旅"
> }
> ```

**說明**：Manus 不僅能理解單一指令，更能理解一個包含多個步驟、有先後順序的複雜任務。它能將您的敘事性描述，拆解為一個由多個工具調用組成的「工具鏈」，並按邏輯順序執行，最終完成您的創作意圖。
