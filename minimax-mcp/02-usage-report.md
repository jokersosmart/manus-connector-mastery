# MiniMax 連結器使用說明報告

作者：Joker

## 1. 總覽

MiniMax 連結器提供了一套強大的工具，讓使用者可以直接在 Manus 環境中利用 MiniMax 的大型語言模型能力。這些工具涵蓋了從文本生成、對話、翻譯到程式碼編寫等多種應用場景，旨在無縫融入您的工作流程，提升生產力。

本報告將詳細介紹每個可用工具的參數、功能、使用範例及預期輸出。

## 2. 可用工具 (Tools)

### 2.1. `generate_text` - 文本生成

此工具為通用的文本生成器，能夠根據提供的提示詞 (Prompt) 創造出連貫、有邏輯的段落、文章、摘要或任何形式的文本內容。

#### 參數

| 參數 | 類型 | 是否必須 | 描述 |
| :--- | :--- | :--- | :--- |
| `prompt` | `string` | 是 | 用於指導模型生成內容的詳細指令。 |
| `max_tokens` | `integer` | 否 | 指定生成文本的最大長度（Token 數量）。預設值為 1024。 |
| `temperature` | `float` | 否 | 控制輸出的隨機性。較高的值（如 0.8）會產生更多樣的輸出，較低的值（如 0.2）則會產生更具確定性的輸出。預設值為 0.7。 |
| `model` | `string` | 否 | 指定使用的 MiniMax 模型版本，例如 `abab6.5` 或 `abab6.5s`。預設為 `abab6.5`。 |

#### 功能說明

`generate_text` 是 MiniMax 連結器的核心功能，適用於需要從零開始創造內容的場景。您可以透過精心設計的 `prompt` 來控制生成內容的風格、語氣、格式和主題。

#### 使用範例

**指令：**

> 使用 MiniMax 的 `generate_text` 工具，為一個新的 AI 咖啡廳撰寫一段吸引人的開幕宣傳文案，強調其「由 AI 精準調配的個人化咖啡」。

**Manus 調用：**

```json
{
  "tool": "minimax.generate_text",
  "args": {
    "prompt": "撰寫一段約 150 字的宣傳文案，為一家名為『位元滴濾 (Bit & Drip)』的 AI 咖啡廳開幕做宣傳。文案需強調我們的特色：每一杯咖啡都由 AI 根據顧客的口味偏好進行即時分析與精準調配，創造獨一無二的風味體驗。風格需具備科技感與溫度感。",
    "max_tokens": 300
  }
}
```

#### 預期輸出

```text
歡迎來到「位元滴濾 (Bit & Drip)」，在這裡，每一口咖啡都是為您量身打造的專屬體驗。我們告別傳統的固定菜單，透過先進的 AI 風味分析引擎，即時了解您的口味偏好，從數千種咖啡豆與沖煮參數的組合中，為您調配出那一杯完美的咖啡。這不僅僅是一杯咖啡，更是一段由程式碼與溫度交織而成的味覺探索之旅。即刻蒞臨，喚醒您沉睡的味蕾，體驗科技為您帶來的溫暖與驚喜。
```

### 2.2. `chat` - 多輪對話

此工具專為需要上下文理解的對話場景設計，能夠進行多輪問答、角色扮演或作為一個虛擬助手來完成連續性任務。

#### 參數

| 參數 | 類型 | 是否必須 | 描述 |
| :--- | :--- | :--- | :--- |
| `messages` | `array` | 是 | 一個包含對話歷史的陣列。每個物件包含 `role` (`user` 或 `assistant`) 和 `content`。 |
| `temperature` | `float` | 否 | 控制輸出的隨機性。預設值為 0.7。 |
| `model` | `string` | 否 | 指定使用的 MiniMax 模型版本。預設為 `abab6.5`。 |

#### 功能說明

`chat` 工具的核心能力在於其記憶和理解對話上下文的能力。這使得它非常適合用於需要來回溝通的任務，例如 brainstorming、問題排查或作為一個隨時待命的知識夥伴。

#### 使用範例

**指令：**

> 我正在規劃一個關於「未來城市」的演講。首先，幫我 brainstorm 一些可能的子主題。然後，針對我選擇的主題，提供更詳細的大綱。

**Manus 調用 (第一輪)：**

```json
{
  "tool": "minimax.chat",
  "args": {
    "messages": [
      {
        "role": "user",
        "content": "我正在規劃一個關於『未來城市』的演講，請幫我 brainstorm 一些可能的子主題。"
      }
    ]
  }
}
```

**預期輸出 (第一輪)：**

```text
當然！這裡有幾個關於「未來城市」的子主題方向：
1.  **智慧交通與無人駕駛**：探討自駕車、飛行計程車、以及 AI 驅動的交通流量管理系統如何改變我們的移動方式。
2.  **永續建築與垂直農業**：專注於綠色建築、能源自給自足的社區，以及在城市內部進行大規模糧食生產的可能性。
3.  **數位治理與公民參與**：討論區塊鏈、大數據和 AI 如何讓市政服務更有效率，並促進市民的直接參與。
4.  **混合實境與都市空間**：探索 AR/VR 如何與物理城市結合，創造新的工作、娛樂和社交體驗。

您對哪個主題比較感興趣？
```

**Manus 調用 (第二輪)：**

```json
{
  "tool": "minimax.chat",
  "args": {
    "messages": [
      {
        "role": "user",
        "content": "我正在規劃一個關於『未來城市』的演講，請幫我 brainstorm 一些可能的子主題。"
      },
      {
        "role": "assistant",
        "content": "當然！這裡有幾個關於「未來城市」的子主題方向...您對哪個主題比較感興趣？"
      },
      {
        "role": "user",
        "content": "我對第二個主題『永續建築與垂直農業』很感興趣，請為此提供一個詳細的演講大綱。"
      }
    ]
  }
}
```

#### 預期輸出 (第二輪)

```text
好的，這是一個關於「永續建築與垂直農業」的演講大綱：

**主題：城市綠洲：打造自給自足的未來都會**

*   **開場 (5分鐘)**
    *   引言：從當前都市化的挑戰（污染、資源消耗）切入。
    *   提出核心問題：城市能否像森林一樣，成為一個能自我調節、自我供給的生態系統？
*   **第一部分：會呼吸的建築 (10分鐘)**
    *   介紹綠色建築的最新發展：從被動式節能設計到能產生能源的正能源建築。
    *   案例分析：新加坡的皮克林賓樂雅臻選酒店 (PARKROYAL COLLECTION Pickering)。
*   **第二部分：天空中的農場 (10分鐘)**
    *   解釋垂直農業的概念及其對糧食安全的重要性。
    *   技術展示：LED 照明、水耕/氣耕技術、自動化監控系統。
*   **第三部分：從藍圖到現實 (5分鐘)**
    *   探討實現這些願景所需面臨的技術、經濟和政策挑戰。
*   **結語 (5分鐘)**
    *   總結永續建築與垂直農業如何共同塑造一個更具韌性、更宜居的未來城市。
    *   呼籲行動：鼓勵聽眾從自身社區開始思考改變。
```
