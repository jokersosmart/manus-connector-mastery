## 1. 歡迎使用 Dify 連結器：賦予 Manus 無限可能

歡迎！本手冊將引導你了解如何透過自然語言，與 Manus 中強大的 **Dify 連結器** 進行互動。你不需要了解任何程式碼，只需要像平常一樣下達指令。

**Dify 連結器是什麼？**

簡單來說，它是一個「技能工廠」。它讓開發者或 AI 愛好者在一個名為 Dify 的視覺化平台上，為 Manus 打造各種功能強大的新「技能」（在 Dify 中稱為「應用」或「工作流」）。

當你向 Manus 下達一個複雜的指令時，如果 Manus 認為某個在 Dify 中預先建好的「技能」最適合完成這個任務，它就會自動調用 Dify 連結器來執行。

本手冊將透過具體的任務範例，向你展示如何使用這些由 Dify 賦能的強大技能，並揭示其背後簡單的運作原理。

## 2. 任務導向範例：你可以如何使用它

以下是幾個常見的任務場景，以及你該如何透過自然語言指令，讓 Manus 調用 Dify 連結器來完成它們。

### 範例一：深度網路研究與摘要

**你的目標**：你需要研究一個主題，並快速獲取某個特定網頁的核心內容。

**你的自然語言指令**：
> 「幫我研究一下『量子計算的商業應用』，並總結 `https://www.tech-journal.com/quantum-business-case` 這篇文章的重點。」

**Manus 如何工作 (幕後揭秘)**：

1.  **意圖識別**：Manus 識別出這是一個包含「研究」和「網頁摘要」的複合任務。
2.  **選擇工具**：Manus 知道 Dify 中有一個預設的「研究助理」技能，該技能整合了網路搜尋和網頁抓取工具。
3.  **工具調用**：Manus 將你的指令轉換為對 Dify 連結器的調用。這個調用會觸發 Dify 內部的一個工作流，該工作流可能包含以下步驟：
    -   **第一步 (搜尋)**：使用 **Google Search** 工具執行搜尋「量子計算的商業應用」。
    -   **第二步 (抓取)**：使用 **Firecrawl** 工具，專門訪問你指定的網址 `https://www.tech-journal.com/quantum-business-case`。
    -   **第三步 (摘要)**：將抓取到的網頁內容交給一個 **LLM** 節點，並指示它「總結這篇文章的重點」。
4.  **返回結果**：Dify 將最終生成的摘要結果返回給 Manus，再由 Manus呈現給你。

### 範例二：從程式碼儲存庫中獲取資訊

**你的目標**：你想快速查看某個公開的 GitHub 專案中的特定文件內容，而不想親自打開網站尋找。

**你的自然語言指令**：
> 「我想看看 `langgenius/dify` 這個 GitHub 專案的 `README.md` 檔案裡寫了什麼。」

**Manus 如何工作 (幕後揭秘)**：

1.  **意圖識別**：Manus 從你的指令中識別出 `GitHub`、專案名稱 `langgenius/dify` 和檔案路徑 `README.md`。
2.  **選擇工具**：它匹配到 Dify 中一個名為「GitHub 工具」的技能。
3.  **工具調用**：Manus 將你的請求轉換為一個結構化的工具調用，看起來像這樣 (簡化版)：
    ```json
    {
      "tool_name": "dify.github_read_file",
      "parameters": {
        "repository": "langgenius/dify",
        "path": "README.md"
      }
    }
    ```
4.  **返回結果**：Dify 的 GitHub 工具根據這些參數，透過 API 讀取到對應的檔案內容，並將其返回給 Manus。

### 範例三：從非結構化文字中提取關鍵資訊

**你的目標**：你有一段文字，只想從中快速抽取出特定格式的資訊，例如電子郵件或電話號碼。

**你的自然語言指令**：
> 「從這段文字『會議記錄：主要聯絡人是 David，他的郵箱是 david.lee@example.com，請大家盡快聯繫他』中，幫我把電子郵件地址找出來。」

**Manus 如何工作 (幕後揭秘)**：

1.  **意圖識別**：Manus 偵測到「找出電子郵件地址」這個指令，這是一個典型的模式匹配任務。
2.  **選擇工具**：它選擇了 Dify 中內建的「正則表達式提取」工具，這個工具專門用來尋找符合特定樣式的文字。
3.  **工具調用**：Manus 將你的文字和一個預設好的「電子郵件正則表達式」一起發送給 Dify 工具。
    ```json
    {
      "tool_name": "dify.regex_extract",
      "parameters": {
        "text": "會議記錄：主要聯絡人是 David...",
        "regex": "[\\w\\.-]+@[\\w\\.-]+\\.[a-zA-Z]{2,}" // 這是一個匹配郵箱的規則
      }
    }
    ```
4.  **返回結果**：Dify 的工具執行匹配，並將找到的 `david.lee@example.com` 返回給你。

## 3. 結論：你的指令，無限的技能

Dify 連結器的真正威力在於其**無限的可擴展性**。今天你看到的範例只是冰山一角。開發者可以持續在 Dify 平台上創建更複雜、更貼近你業務需求的「技能」，例如：

-   自動分析銷售數據並生成週報的「銷售分析師」。
-   監控社群媒體並報告輿情的「公關助理」。
-   根據你的旅遊偏好自動規劃行程的「旅遊規劃師」。

而你，作為使用者，**永遠只需要透過最自然的語言下達指令**。你無需關心背後是哪個 Dify 應用在運作，只需享受 Manus 因 Dify 而日益強大的服務能力。

現在就試試看，向 Manus 提出一個挑戰性的任務吧！
