# Dify 連結器使用說明報告

**作者：Joker**

---

## 1. 總覽

Dify 連結器將 Dify 平台強大的 AI 應用建構能力，整合至 Manus MCP 生態系統。它允許使用者透過自然語言指令，調用在 Dify 中預先建構或從 Marketplace 安裝的各種工具。這些工具涵蓋了從網路搜尋、開發者工具、數據處理到企業協作等多種場景。

核心機制是 Dify 的「工具節點 (Tools node)」，它將外部服務的 API 封裝成結構化的、可配置的模組。當 Manus 調用 Dify 連結器時，實際上是在觸發一個包含特定工具節點的 Dify 工作流 (Workflow)。

## 2. 代表性內建工具詳解

以下將詳細介紹幾個 Dify 平台提供的核心內建工具，這些工具最能體現 Dify 連結器的功能與應用潛力。

### 2.1 Google 搜尋 (Google Search)

- **功能說明**：此工具允許 AI 應用執行 Google 搜尋，並獲取搜尋結果頁 (SERP) 的摘要和網頁內容。它是實現 RAG (檢索增強生成)、進行事實查核和獲取即時資訊的基礎。
- **核心參數**：
    - `query` (字串，必填): 要執行的搜尋關鍵字。
- **使用範例**：
    - **自然語言指令**：「幫我用 Google 搜尋一下『Dify 的最新功能』。」
    - **Manus 調用 (示意)**：
      ```json
      {
        "tool_name": "dify.google_search",
        "parameters": {
          "query": "Dify 的最新功能"
        }
      }
      ```
- **預期輸出**：返回一個包含多個搜尋結果的列表，每個結果包含標題、連結和摘要。部分進階設定下，還可能包含網頁的純文字內容。

### 2.2 GitHub 工具

- **功能說明**：提供與 GitHub 儲存庫互動的能力，例如讀取檔案、查詢議題 (Issues) 或提交程式碼。這使得 AI 應用能夠參與到軟體開發的流程中。
- **核心參數** (以讀取檔案為例)：
    - `repository` (字串，必填): 儲存庫的完整名稱，格式為 `owner/repo`。
    - `path` (字串，必填): 儲存庫中檔案的路徑。
    - `branch` (字串，選填): 要讀取的分支名稱，預設為 `main`。
- **使用範例**：
    - **自然語言指令**：「讀取 `langgenius/dify` 專案中 `README.md` 檔案的內容。」
    - **Manus 調用 (示意)**：
      ```json
      {
        "tool_name": "dify.github_read_file",
        "parameters": {
          "repository": "langgenius/dify",
          "path": "README.md"
        }
      }
      ```
- **預期輸出**：返回指定檔案的內容 (通常是 Base64 編碼的字串) 以及檔案的元數據。

### 2.3 網頁抓取 (Firecrawl)

- **功能說明**：Firecrawl 是一個強大的網頁抓取和內容提取工具。它能將任何網頁轉換為乾淨的、結構化的 Markdown 或純文字，非常適合用於知識庫建構和數據採集。
- **核心參數**：
    - `url` (字串，必填): 要抓取的網頁連結。
    - `mode` (字串，選填): 抓取模式，可選 `scrape` (抓取並清理) 或 `crawl` (深度爬取)，預設為 `scrape`。
- **使用範例**：
    - **自然語言指令**：「幫我抓取 `https://docs.dify.ai/` 這個網站的內容，並轉換成 Markdown。」
    - **Manus 調用 (示意)**：
      ```json
      {
        "tool_name": "dify.firecrawl",
        "parameters": {
          "url": "https://docs.dify.ai/",
          "mode": "scrape"
        }
      }
      ```
- **預期輸出**：返回一個包含網頁標題、純文字內容和 Markdown 格式內容的 JSON 物件。

### 2.4 正則表達式提取 (Regex)

- **功能說明**：一個強大的文字處理工具，可以從一段文字中，根據指定的正則表達式規則，提取出符合模式的資訊。
- **核心參數**：
    - `text` (字串，必填): 要進行提取的原始文字內容。
    - `regex` (字串，必填): 用於匹配的 PCRE 正則表達式。
- **使用範例**：
    - **自然語言指令**：「從這段文字『我的信箱是 example@dify.ai』中，幫我把電子郵件地址提出來。」
    - **Manus 調用 (示意)**：
      ```json
      {
        "tool_name": "dify.regex_extract",
        "parameters": {
          "text": "我的信箱是 example@dify.ai",
          "regex": "[\\w\\.-]+@[\\w\\.-]+\\.[a-zA-Z]{2,}"
        }
      }
      ```
- **預期輸出**：返回一個包含所有匹配結果的字串列表。

## 3. 結論

Dify 連結器為 Manus 帶來了前所未有的擴展性。透過其視覺化的工作流和豐富的工具市場，使用者可以快速建構功能強大的 AI 應用，並將其作為一個個獨立的「技能」供 Manus 調用。本報告所列僅為冰山一角，我們鼓勵使用者深入探索 Dify Marketplace，發掘更多能與自身工作流結合的工具，創造出真正個人化的 AI 助手。
