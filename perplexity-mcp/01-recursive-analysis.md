### B. 收斂 (五何法)

**元素 1: 對 AI 答案準確性的疑慮 (過去 - 別人 - Input data)**

*   **為什麼使用者會對 AI 答案的準確性有疑慮？**
    因為他們過去可能遇過 AI「一本正經地胡說八道」(Hallucination) 的情況，導致對產出內容的不信任。
*   **為什麼 AI 會產生幻覺？**
    因為大型語言模型的核心是基於機率預測下一個詞，而非真正的理解。當訓練資料不足或存在偏見時，模型可能會產生看似合理但與事實不符的內容。
*   **為什麼訓練資料會不足或有偏見？**
    因為網路上的資訊本身就充滿了錯誤、過時和帶有偏見的內容，而模型在訓練時會全盤吸收。
*   **為什麼我們不能只用「乾淨」的資料訓練模型？**
    因為定義和篩選「乾淨」的資料本身就是一個極其龐大且複雜的工程，且何謂「乾淨」也存在主觀性。
*   **為什麼這對 Perplexity 連接器很重要？**
    **根本原因：** 使用者的信任是連接器成功的基石。如果不能有效解決或緩解使用者對準確性的疑慮，即使功能再強大，也難以被廣泛採納。因此，連接器的設計必須將「提供可驗證的來源」和「突顯答案的引用」作為核心功能。

**元素 2: 如何處理 API 回傳的錯誤 (現在 - 自己 - Input control)**

*   **為什麼處理 API 錯誤很重要？**
    因為網路不穩定、API 金鑰錯誤或 Perplexity 服務本身的問題都可能導致請求失敗。若不妥善處理，會嚴重影響使用者體驗。
*   **為什麼不直接把錯誤訊息顯示給使用者？**
    因為原始的 API 錯誤訊息通常很技術性，對非開發者使用者來說難以理解，甚至可能引發不必要的焦慮。
*   **為什麼需要設計一個對使用者友善的錯誤處理機制？**
    為了讓使用者在遇到問題時，能清楚地知道發生了什麼，以及他們可以採取什麼行動 (例如：重試、檢查網路、聯絡客服)。
*   **為什麼不只是簡單地顯示「發生錯誤，請重試」？**
    因為不同的錯誤類型需要不同的應對策略。例如，Rate Limit 錯誤需要提示使用者稍後再試，而認證失敗則需要引導他們檢查設定。
*   **為什麼這對 Perplexity 連接器很重要？**
    **根本原因：** 一個穩健的連接器不僅要在成功時提供價值，更要在失敗時提供清晰的引導。優雅地處理錯誤是建立使用者信任和提升產品專業形象的關鍵環節。

### 本輪根本原因清單

1.  連接器的設計必須將「提供可驗證的來源」和「突顯答案的引用」作為核心功能，以解決使用者對 AI 答案準確性的疑慮。
2.  連接器必須實作一個對使用者友善且具體的錯誤處理機制，以提升產品的穩健性和專業形象。

### B. 收斂 (五何法)

**元素 1: 如何在不干擾閱讀的情況下顯示引用 (現在 - 自己 - Input control)**

*   **為什麼這是一個挑戰？**
    因為過多的視覺元素會分散使用者的注意力，打斷他們的閱讀流暢性，降低核心資訊的獲取效率。
*   **為什麼不能簡單地把連結放在文末？**
    因為這樣會切斷內文與證據之間的直接關聯，使用者需要頻繁地在文末和內文之間來回跳轉，體驗不佳。
*   **為什麼彈出式視窗 (Popover) 是一個可能的選項？**
    因為它可以在使用者需要時才顯示詳細資訊 (來源標題、URL)，而在不需要時保持介面簡潔。
*   **為什麼不直接在文字旁邊顯示一個小圖示？**
    這也是一個好方法，但需要設計一個直觀且不突兀的圖示。數字標號 `[1]` 是學術界和維基百科已經建立使用者心智模型的符號，學習成本較低。
*   **為什麼最終選擇「數字標號 + 彈出式視窗」的組合？**
    **根本原因：** 這種組合在「資訊密度」與「介面簡潔」之間取得了最佳平衡。它既能透過熟悉的符號 `[1]` 明確標示出處，又不會用過多的資訊淹沒使用者，同時提供了「按需索取」的深入了解路徑，滿足了從快速瀏覽到深度驗證的不同使用者需求。

**元素 2: 針對不同錯誤類型設計不同的提示訊息 (現在 - 自己 - Input control)**

*   **為什麼不能用統一的錯誤訊息？**
    因為「網路中斷」和「查詢過於模糊」是兩種性質完全不同的問題，需要使用者採取截然不同的行動。
*   **為什麼需要分類？**
    為了提供更精準的引導。我們可以將錯誤分為「使用者可解決的」(如輸入問題)、「環境問題」(如網路) 和「服務端問題」(如 API 故障)。
*   **為什麼要引導使用者？**
    因為一個好的產品應該賦予使用者解決問題的能力，而不是讓他們感到無助。清晰的引導可以將一次失敗的互動轉化為一次學習機會。
*   **為什麼這個引導應該是具體的？**
    模糊的建議 (如「請稍後再試」) 會讓使用者無所適從。具體的建議 (如「您的查詢可能太廣泛了，試試加入更多細節？」) 則能直接幫助他們成功。
*   **為什麼這對建立信任很重要？**
    **根本原因：** 精細化的錯誤處理向使用者傳達了一個訊息：開發者認真對待他們的每一次互動，並預見了可能出現的障礙。這種對細節的關注和對使用者處境的共情，是建立長期信任和產品忠誠度的關鍵，它將產品從一個單純的「工具」提升為一個可靠的「夥伴」。

### 終止條件已滿足

在第二輪循環中，`Input data` 和 `Input control` 欄位產出的內容，如「UI 元件庫」、「API 資料結構」、「使用者回饋數據」等，已進入具體的開發實作細節，而非更高層次的戰略或原則問題。這表示關於「為什麼要這麼做」和「核心挑戰是什麼」的根本原因已經探討得相對清晰，遞迴分析可以在此停止。

### 核心洞察摘要

1.  **信任是核心資產**：使用者對 AI 準確性的疑慮是最大的使用障礙。Perplexity 連接器的首要任務是透過**提供可驗證的引用來源**來建立信任。這不是一個附加功能，而是核心價值主張。
2.  **平衡是設計關鍵**：引用的呈現方式必須在**資訊密度與介面簡潔之間取得平衡**。採用「數字標號 + 彈出式視窗」的設計，可以在不干擾閱讀流的同時，滿足使用者按需驗證來源的深度需求。
3.  **失敗是體驗一環**：穩健的產品不僅要在成功時表現出色，更要在失敗時提供**清晰、具體、友善的引導**。精細化的錯誤處理能將挫折轉化為信任，將工具提升為可靠的夥伴。
4.  **從工具到夥伴**：最終目標是讓使用者將 Perplexity 連接器視為一個可靠的研究夥伴。這需要產品在功能（準確、可信）和體驗（流暢、有引導）上都達到極高標準，最終將使用者從資訊的被動消費者，轉變為主動的知識創造者。
