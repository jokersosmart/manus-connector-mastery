# HeyGen API 終端使用者操作手冊

作者：Joker

## 簡介

本手冊專為希望透過自然語言指令，利用 Manus 的強大能力來操作 HeyGen API 的使用者而設計。您不需要了解複雜的程式碼，只需下達清晰的指令，Manus 就會為您完成影片的生成與管理。

## 核心概念：從指令到行動

當您向 Manus 下達關於影片生成的指令時，Manus 會在背後將您的自然語言轉換為對 HeyGen API 的具體工具調用。理解這個對應關係，能幫助您更精準地表達您的需求。

| 您的指令 (範例) | Manus 的理解 | HeyGen API 工具調用 |
| :--- | :--- | :--- |
| 「幫我用 HeyGen 做個影片，內容是『哈囉世界』」 | 使用者想快速生成一個簡單影片 | `POST /v1/video_agent/generate` |
| 「我想做個影片，指定要用 Mark 當頭像，聲音要用 Jenny」 | 使用者需要精確控制影片元素 | `POST /v2/video/generate` |
| 「我 5 分鐘前提交的影片好了嗎？」 | 使用者想查詢影片狀態 | `GET /v1/video_status.get` |
| 「HeyGen 有哪些頭像可以用？」 | 使用者需要查詢可用資源 | `GET /v2/avatars` |

## 任務導向指令範例

以下是一些常見的任務以及您可以如何向 Manus 下達指令。

### 任務一：快速製作一支短片

如果您只需要將一段文字快速變成影片，而不關心具體的頭像或背景。

*   **您的指令：**
    > 「Manus，請用 HeyGen API 幫我生成一支影片，標題是『每日提醒』，內容是：『今天也要記得保持微笑喔！』」

*   **Manus 的執行流程：**
    1.  解析您的指令，識別出標題和內容。
    2.  選擇使用最直接的 `video_agent/generate` 工具。
    3.  建構出如下的 API 請求內容：
        ```json
        {
          "title": "每日提醒",
          "script": [ {"text": "今天也要記得保持微笑喔！"} ]
        }
        ```
    4.  提交請求給 HeyGen API。
    5.  自動開始輪詢影片狀態，並在影片完成後，將下載連結提供給您。

### 任務二：製作一支指定角色的客製化影片

當您對影片的視覺和聽覺有特定要求時。

*   **您的指令：**
    > 「Manus，我想用 HeyGen 做一支教學影片。請幫我列出所有可用的女性英文聲音。我選好後，要用 ID 為 `avatar-id-123` 的頭像，搭配我選的聲音，背景用白色的。影片內容是：『歡迎來到我們的線上課程。』」

*   **Manus 的執行流程：**
    1.  收到您的第一個請求，執行 `GET /v2/voices`，並篩選出語言為英文、性別為女性的聲音列表呈現給您。
    2.  您從列表中選擇了一個聲音 ID，例如 `voice-id-456`。
    3.  Manus 整合您的所有要求（頭像 ID、聲音 ID、背景顏色、腳本內容）。
    4.  選擇使用 `video/generate` 工具，因為這涉及到精準的元素控制。
    5.  建構出如下的 API 請求內容：
        ```json
        {
          "video_inputs": [
            {
              "avatar": {"avatar_id": "avatar-id-123"},
              "voice": {
                "voice_id": "voice-id-456",
                "text": "歡迎來到我們的線上課程。"
              },
              "background": {"type": "color", "value": "#FFFFFF"}
            }
          ]
        }
        ```
    6.  提交請求，並在完成後回報結果。

### 任務三：製作多場景影片

如果您的影片包含多個部分，由不同角色或在不同場景下講述。

*   **您的指令：**
    > 「Manus，幫我做一個雙人對話影片。第一段，讓 Luna 說：『專案進度如何？』；第二段，讓 Mark 回答：『一切順利，預計明天能完成。』」

*   **Manus 的執行流程：**
    1.  識別出這是一個多場景的影片需求。
    2.  根據您提供的角色和台詞，建構一個包含多個場景物件的腳本。
    3.  根據場景的複雜度，可能會選擇 `video_agent/generate` 或 `video/generate` 工具。
    4.  建構出類似以下的 API 請求內容：
        ```json
        {
          "script": [
            {
              "actor": "Luna",
              "text": "專案進度如何？"
            },
            {
              "actor": "Mark",
              "text": "一切順利，預計明天能完成。"
            }
          ]
        }
        ```
    5.  提交請求並追蹤後續狀態。

透過這些範例，您可以看到，您只需要專注於「您想要什麼樣的影片」，而將「如何實現」的技術細節交給 Manus。只要您的指令越清晰、越具體，Manus 就越能準確地為您完成任務。
