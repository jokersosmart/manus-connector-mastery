# Dify 從新手到精通：一份完整的學習路徑指南

**作者：Joker**

---

## 前言：你的 AI 應用開發新起點

歡迎來到 Dify 的世界。Dify 是一個功能強大且直觀的 LLM 應用開發平台，它將複雜的 AI 技術轉化為人人皆可上手的視覺化工作流。本指南將帶你從零開始，逐步深入，最終掌握使用 Dify 建構複雜、實用的 AI 應用的能力。無論你是開發者、產品經理，還是對 AI 充滿熱情的探索者，這份指南都將是你從新手走向精通的可靠路徑。

## 第一階段：新手上路 (The Novice) - 理解基礎

在這個階段，我們的目標是建立對 Dify 核心概念的理解，並成功運行你的第一個 AI 應用。

### 1. 認識 Dify 的核心積木

- **Studio**：這是你的主要工作區，一個視覺化的畫布，你將在這裡透過拖拽和連接節點來「繪製」你的應用邏輯。
- **應用類型 (App Types)**：Dify 提供兩種主要的應用類型：
    - **Workflow (工作流)**：專為處理單輪、自動化任務而設計，例如從文章中提取摘要、格式化數據等。
    - **Chatflow (對話流)**：專為建構多輪對話機器人而設計，能夠處理上下文記憶和連續互動。
- **節點 (Nodes)**：節點是構成工作流的最小功能單元。每個節點都執行一個特定的任務，例如調用語言模型、從知識庫檢索資訊或執行程式碼。

### 2. 你的第一個 Dify 應用：一個簡單的問候機器人

1.  **創建應用**：在 Dify Studio 中，點擊「創建新應用」，選擇「Chatflow」類型，並給它一個名字，例如「MyFirstBot」。
2.  **認識畫布**：你會看到一個預設的畫布，包含 `Start` (開始)、`Answer` (回答) 等節點。
3.  **配置 `Start` 節點**：`Start` 節點定義了對話開始時的輸入。你可以暫時使用預設設定。
4.  **加入 `LLM` 節點**：從左側節點列表中，拖拽一個 `LLM` (大型語言模型) 節點到畫布上。
5.  **連接節點**：將 `Start` 節點的輸出連接到 `LLM` 節點的輸入，再將 `LLM` 節點的輸出連接到 `Answer` 節點的輸入。
6.  **配置 `LLM` 節點**：在 `LLM` 節點的設定中，選擇一個可用的模型 (如 GPT-4)，並在提示 (Prompt) 中輸入：「`用戶說：{{#start.query#}}。請你作為一個熱情的助手，向用戶問好。`」。這裡的 `{{#start.query#}}` 是一個變數，代表用戶的輸入。
7.  **測試與發布**：點擊右上角的「調試與預覽」，在聊天窗口中輸入「你好」，看看機器人是否會熱情地回應你。恭喜，你已經成功創建了第一個 AI 應用！

## 第二階段：學徒進階 (The Apprentice) - 掌握核心功能

熟悉了基本操作後，我們將探索 Dify 更強大的功能，讓你的應用變得更「聰明」。

### 1. 打造你的專屬知識庫：RAG 應用

- **概念**：RAG (檢索增強生成) 允許你的 AI 應用基於你提供的特定文件或數據來回答問題，而不是僅僅依賴模型的通用知識。
- **實踐**：
    1.  在 Dify 的「知識庫」區塊，創建一個新的知識庫，並上傳一些文件 (如 PDF, TXT, Markdown)。
    2.  在你的 Chatflow 中，加入 `Knowledge Retrieval` (知識檢索) 節點。
    3.  將 `Start` 節點的輸出連接到 `Knowledge Retrieval` 節點，並在後者中選擇你剛創建的知識庫。
    4.  將檢索到的上下文傳遞給 `LLM` 節點的提示，讓模型基於這些上下文來生成答案。

### 2. 連接外部世界：使用工具 (Tools)

- **概念**：工具節點讓你的應用能夠與外部 API 和服務互動，例如執行 Google 搜尋、查詢天氣、讀取 GitHub 儲存庫等。
- **實踐**：
    1.  在工作流中加入 `Tools` 節點。
    2.  在節點設定中，選擇一個內建工具，例如 `Google Search`。
    3.  將用戶的輸入作為搜尋查詢，並將搜尋結果用於後續的處理或生成回答。

## 第三階段：工匠之路 (The Journeyman) - 探索進階技巧

現在，你已經準備好挑戰更複雜的邏輯和更具自主性的代理 (Agent) 應用。

### 1. 創建自主代理：Agentic Workflow

- **概念**：與按部就班執行指令的傳統工作流不同，代理工作流能夠自主規劃、推理，並決定使用哪些工具來完成一個更宏觀的目標。
- **實踐**：
    1.  使用 `Agent` 節點代替 `LLM` 節點。
    2.  為 Agent 設定一個目標 (Goal)，例如「成為一個能夠回答 Dify 相關問題的專家」。
    3.  為 Agent 配備工具，例如 `Knowledge Retrieval` (關聯 Dify 官方文件) 和 `Google Search`。
    4.  現在，當你向這個 Agent 提問時，它會自主決定是從知識庫中尋找答案，還是上網搜尋最新資訊。

### 2. 掌握控制流：If-Else 與迭代

- **概念**：透過 `If-Else` (條件判斷)、`Question Classifier` (問題分類器) 和 `Iteration` (迭代) 等節點，你可以建構出具備複雜判斷和循環能力的應用。
- **實踐**：創建一個可以根據用戶問題類型，分發到不同處理路徑的工作流。例如，如果是技術問題，則查詢知識庫；如果是閒聊，則直接由 LLM 回答。

## 第四階段：大師境界 (The Master) - 實戰與整合

達到這個階段，你已經具備了獨立設計和開發複雜 AI 應用的能力。是時候將你的作品部署到真實世界，並將其整合到 Manus 的生態中了。

### 1. 部署與發布

- **多種方式**：Dify 允許你將應用發布為：
    - **獨立 WebApp**：一個可公開訪問的網頁應用。
    - **後端 API**：供其他程式調用的標準 API 介面。
    - **Manus MCP 伺服器**：無縫整合到 Manus 生態，讓 Manus 能夠透過自然語言調用你的 Dify 應用。

### 2. 實戰案例：建立一個研究助理 Agent

- **目標**：創建一個能夠自動上網搜尋、閱讀多個網頁、綜合資訊並生成研究報告的 Agent。
- **實現**：這需要你綜合運用前面學到的所有技能：Agent 節點、多個工具 (Google Search, Firecrawl)、迭代 (處理多個搜尋結果) 以及變數管理 (匯總資訊)。

### 3. Dify MCP 連結器：能力的終極放大

- **概念**：當你將 Dify 應用作為 MCP 伺服器發布時，你在 Dify 中創建的每一個複雜工作流，都變成了 Manus 的一個新「技能」。
- **價值**：這意味著你可以為自己、為團隊，甚至為整個社群，創造出高度客製化、功能強大的 AI 工具，並透過 Manus 的統一介面來使用它們。

## 結語

從新手到精通的旅程，是對 Dify 功能的探索，更是對 AI 應用邊界的拓展。你現在掌握的不僅僅是一個工具，而是一種全新的創造方式。繼續探索，不斷實踐，將你的想像力透過 Dify 變為現實吧！
