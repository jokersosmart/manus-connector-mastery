# 「Apollo」連結器終端使用者操作手冊

**作者：Joker**

本手冊專為「Apollo」連結器的終端使用者設計。您不需要了解 GraphQL 或任何程式設計細節，只需學習如何透過自然語言，讓 Manus AI 為您完成與後端系統互動的各種任務。

---

## 核心理念：用對話完成工作

Apollo 連結器的魔力在於，它讓 Manus AI 能夠理解並使用您公司內部的各種軟體系統。您可以像和一位聰明的同事對話一樣，要求 Manus AI 幫您查詢資料、更新狀態，甚至執行複雜的業務流程。

關鍵在於：**清晰地表達您的「意圖」**。

您不需要說出確切的工具名稱或參數，Manus AI 會根據您的描述，自動選擇最合適的工具並填入對應的資訊。

---

## 任務導向範例

以下將以一個常見的「電子商務平台」為例，展示您如何透過自然語言指令，讓 Manus AI 調用 Apollo 連結器提供的工具來完成任務。

### 任務一：查詢產品資訊

當您需要快速了解某個產品的詳細情況時，您可以這樣說：

> **您的指令**：「幫我查一下 ID 為 `prod_12345` 的無線降噪耳機現在還有多少庫存？」

**Manus AI 的內心戲（工具調用過程）**：

1.  **意圖識別**：使用者想要查詢特定產品 (`prod_12345`) 的資訊，特別是「庫存 (`stock`)」。
2.  **工具選擇**：掃描所有可用的 Apollo 工具，發現 `getProduct(id: String!)` 這個查詢最符合需求，因為它可以根據 ID 返回產品的詳細資訊，其中包含 `stock` 欄位。
3.  **參數填充**：從使用者的指令中提取出 `id` 的值是 `prod_12345`。
4.  **執行調用**：向 Apollo 連結器發出請求，執行 `getProduct` 工具，並傳入參數 `{ "id": "prod_12345" }`。
5.  **結果呈現**：收到後端 API 返回的產品資料後，從中提取 `stock` 的值，並用自然語言回答您：

> **Manus AI 的回覆**：「好的，ID 為 `prod_12345` 的無線降噪耳機目前還有 150 個庫存。」

--- 

### 任務二：篩選符合條件的客戶

當您需要從海量資料中，找到一批符合特定條件的目標時，您可以更複雜地描述您的需求：

> **您的指令**：「幫我列出所有在過去一個月內下單超過 3 次，並且住在加州的客戶名單。」

**Manus AI 的內心戲（工具調用過程）**：

1.  **意圖識別**：使用者需要一個客戶列表，篩選條件有三個：
    - 時間範圍：過去一個月
    - 行為：下單次數 > 3
    - 地區：加州
2.  **工具選擇**：Manus AI 發現 `listCustomers` 這個工具最為合適。它接受一個 `filter` 物件作為參數，可以進行複雜的條件組合。
3.  **參數填充**：Manus AI 會將您的自然語言條件，轉換為結構化的 `filter` 物件：
    ```json
    {
      "filter": {
        "lastOrderDate": { "gte": "2026-01-24" }, // 假設今天是 2026-02-24
        "orderCount": { "gt": 3 },
        "state": "CA"
      }
    }
    ```
4.  **執行調用**：執行 `listCustomers` 工具，並傳入上述參數。
5.  **結果呈現**：將返回的客戶列表整理成清晰的表格或清單呈現給您。

> **Manus AI 的回覆**：「好的，這是在過去一個月內下單超過 3 次且居住在加州的客戶名單：
> - John Doe (john.doe@example.com)
> - Jane Smith (jane.smith@example.com)
> ...」

### 任務三：執行一個業務操作

當您需要更新系統狀態時，例如批准一筆訂單或修改一個設定，您也可以直接下達指令。

> **您的指令**：「請批准訂單 `order_xyz789`，並在備註中加入『客戶要求快速出貨』。」

**Manus AI 的內心戲（工具調用過程）**：

1.  **意圖識別**：使用者要執行一個「批准訂單」的動作，目標是 `order_xyz789`，並且需要附加一個備註。
2.  **工具選擇**：Manus AI 找到了 `approveOrder(orderId: String!, notes: String)` 這個操作 (Mutation) 工具。
3.  **參數填充**：從指令中提取 `orderId` 為 `order_xyz789`，`notes` 為「客戶要求快速出貨」。
4.  **執行調用**：執行 `approveOrder` 工具。
5.  **結果驗證與回覆**：操作成功後，Manus AI 可能會接著調用 `getOrderStatus` 工具來確認訂單狀態是否已變為「已批准」，然後向您回報。

> **Manus AI 的回覆**：「訂單 `order_xyz789` 已成功批准，並已添加備註。目前的訂單狀態為：已批准，等待出貨。」

## 使用技巧

- **提供上下文**：如果您連續下達多個相關指令，Manus AI 會記得之前的對話內容。例如，在查詢完一個產品後，您可以直接說「把它加入我的購物車」。
- **明確指定 ID**：當涉及到具體的項目（如產品、訂單、客戶）時，盡量提供它們的唯一識別碼 (ID)，這能幫助 Manus AI 最快、最準確地定位目標。
- **逐步修正**：如果 Manus AI 第一次沒有完全理解您的意思，別擔心。您可以像和真人溝通一樣，補充更多資訊或修正您的要求，例如「不對，我是指上個月，不是這個月」。
