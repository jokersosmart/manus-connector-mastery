# 「Mailchimp Marketing」連結器使用者操作手冊

**作者：Joker**

---

## 導覽

本手冊為您——終端使用者——而設計。在這裡，我們不談論複雜的 API 或參數，只專注於一件事：**如何透過簡單的對話，讓 Manus 為您完成 Mailchimp 的工作**。

我們將以任務為導向，展示一系列您在日常工作中可能遇到的場景。對於每個場景，我們都會提供一個您可以直接使用的自然語言指令範例，並揭示 Manus 在背後是如何聰明地將您的話語轉換為精確的工具調用。

---

## 任務一：盤點我的聯絡人列表

**情境**：您剛開始使用，想知道您的 Mailchimp 帳戶裡到底有哪些聯絡人列表（在 Mailchimp 中稱為「受眾」）。

### 您可以這樣說：

> 「幫我看看 Mailchimp 裡有哪些受眾列表。」

#### Manus 如何理解與執行：

Manus 聽到「看看」、「列表」、「受眾」這些關鍵詞，就知道您想要的是一個清單。它會選擇最適合的工具來完成這個任務。

-   **工具調用**：
    ```json
    {
      "tool": "mailchimp.list_audiences"
    }
    ```
-   **轉換解析**：您的簡單請求被直接對應到 `list_audiences` 這個無需任何參數的工具，以獲取您帳戶中所有受眾的概覽。

## 任務二：新增一位訂閱者

**情境**：您在線下活動中手動收集到一個新的潛在客戶 email，需要立即將他加入您的主要電子報列表，以便他能收到下一期的內容。

### 您可以這樣說：

> 「把 `new.lead@email.com` 加到我的『電子報訂閱者』列表裡。」

#### Manus 如何理解與執行：

Mantus 會識別出幾個關鍵資訊：「新增聯絡人」、「電子郵件地址」和「目標列表名稱」。如果它不確定「電子報訂閱者」對應的確切 ID，它可能會先執行 `list_audiences` 來查找，或者直接詢問您。假設它找到了 ID 為 `a1b2c3d4e5`。

-   **工具調用**：
    ```json
    {
      "tool": "mailchimp.add_contact",
      "audience_id": "a1b2c3d4e5",
      "email_address": "new.lead@email.com",
      "status": "subscribed"
    }
    ```
-   **轉換解析**：
    -   「把...加到」觸發了 `add_contact` 工具。
    -   `new.lead@email.com` 被識別為 `email_address` 參數。
    -   「『電子報訂閱者』列表」被 Manus 解析並映射到其對應的 `audience_id`。
    -   在沒有特別指明的情況下，Manus 會預設一個合理的訂閱狀態 `subscribed`。

## 任務三：檢查特定郵件的成效

**情境**：上週您發送了一次重要的產品更新郵件，現在您想快速知道它的表現如何。

### 您可以這樣說：

> 「上週發的『產品更新 V2.0』那封郵件，成效報告是怎麼樣的？」

#### Manus 如何理解與執行：

Mantus 需要先找到您說的這封郵件，然後才能獲取它的報告。這是一個典型的兩步驟過程，但對您來說只是一句話。

1.  **第一步：尋找郵件**。Manus 會先搜尋符合條件的行銷活動。
    -   **工具調用 (可能)**：
        ```json
        {
          "tool": "mailchimp.list_campaigns",
          "status": "sent"
        }
        ```
    -   然後在結果中尋找標題最接近「產品更新 V2.0」的活動，並提取其 `id`，例如 `c1d2e3f4g5`。

2.  **第二步：獲取報告**。使用上一步找到的 ID，獲取詳細報告。
    -   **工具調用**：
        ```json
        {
          "tool": "mailchimp.get_campaign_report",
          "campaign_id": "c1d2e3f4g5"
        }
        ```
-   **轉換解析**：您的單一指令被 Manus 拆解成一個「先查找，後獲取」的內部工作流程。它自動處理了 ID 的傳遞，最終只將您最關心的成效報告呈現給您。

## 任務四：快速建立一個活動框架

**情境**：您有一個新的行銷點子，需要立刻建立一個活動草稿，以便您的設計同事可以開始填充內容。

### 您可以這樣說：

> 「幫我為『夏季特賣』活動建立一個新的 Mailchimp 行銷活動草稿，目標是我的『VIP 客戶』列表。」

#### Manus 如何理解與執行：

Mantus 捕捉到「建立」、「行銷活動草稿」、「活動名稱」和「目標列表」等關鍵意圖。

-   **工具調用**：
    ```json
    {
      "tool": "mailchimp.create_campaign",
      "type": "regular",
      "recipients": {
        "list_id": "k6l7m8n9p0" // 假設 'VIP 客戶' 列表的 ID
      },
      "settings": {
        "subject_line": "夏季特賣",
        "from_name": "[您的預設寄件人名稱]",
        "reply_to": "[您的預設回覆信箱]"
      }
    }
    ```
-   **轉換解析**：
    -   「建立...行銷活動草稿」觸發了 `create_campaign` 工具。
    -   「『夏季特賣』活動」被用作 `subject_line`。
    -   「『VIP 客戶』列表」被解析為 `recipients` 物件中的 `list_id`。
    -   其他如 `from_name` 等必要資訊，Manus 可能會使用您帳戶的預設值或向您詢問。

## 結語

正如您所見，您無需成為 Mailchimp 專家，也無需記住任何工具名稱或參數。您只需要用最自然的方式，告訴 Manus 您想**完成什麼任務**。

理解 Manus 在背後如何思考和工作，可以幫助您更精準地提出您的需求。現在，就去嘗試將您自己的 Mailchimp 日常工作，交給 Manus 來處理吧！
