# Jam MCP 連結器使用者操作手冊

**作者：Joker**

## 歡迎！這是一本為您而寫的手冊

您好！本手冊將向您展示如何透過自然的對話，讓 Manus AI Agent 幫您處理來自 Jam 的 Bug 報告。您不需要學習複雜的程式碼或工具參數，只需要像和同事交談一樣，告訴 Manus 您的目標即可。

本手冊的核心是「任務導向」。我們將展示一系列常見的偵錯任務，提供您可以直接使用的「自然語言指令」，並揭示 Manus 在背後是如何聰明地將您的話語轉換為精確的「工具調用」。

---

## 任務一：快速了解一個新的 Bug 報告

當您收到一個新的 Jam 連結時，最先想做的一定是快速了解「這大概是個什麼問題？」。

### 您的指令

> 「嘿 Manus，幫我看看這個 Jam 報告，告訴我它的重點是什麼。連結是：https://jam.dev/j/xxxxxxxx」

### Manus 的思考與行動

收到這個指令後，Manus 會識別出您的意圖是「獲取摘要」。它知道 `jam` 連結器中有一個完美的工具可以完成這件事。

*   **單一步驟：獲取摘要**
    *   **動作**：調用 `getSummary` 工具。
    *   **目的**：快速生成一份包含作者、時間、問題描述和下一步建議的報告。

#### 背後的工具調用

```json
{
  "tool_name": "getSummary",
  "parameters": {}
}
```

**結果**：Manus 會直接用一份清晰的摘要來回覆您，讓您在 30 秒內掌握問題核心。

## 任務二：深入調查一個後端伺服器錯誤

摘要告訴您這可能是一個「伺服器錯誤」。現在，您需要深入挖掘，找到是哪個 API 呼叫失敗了。

### 您的指令

> 「看來是個 500 錯誤。幫我深入查一下這個 Jam，找出是哪個網路請求失敗了，並看看當時的瀏覽器有沒有什麼異常。」

### Manus 的思考與行動

Manus 會將您的指令拆解成一個調查計畫：

1.  **定位伺服器錯誤**：您的關鍵字「網路請求失敗」和「500 錯誤」會讓 Manus 鎖定 `getNetworkRequests` 工具，並使用 `statusCode` 過濾器。
2.  **檢查前端線索**：您的指令「看看瀏覽器有沒有異常」會讓 Manus 接著使用 `getConsoleLogs` 工具，尋找相關的前端錯誤訊息。
3.  **綜合報告**：最後，Manus 會將兩個工具的發現結合起來，給您一個完整的答案。

#### 背後的工具調用

```json
// 步驟一：尋找伺服器錯誤
{
  "tool_name": "getNetworkRequests",
  "parameters": {
    "statusCode": "5xx"
  }
}

// 步驟二：檢查相關的前端日誌
{
  "tool_name": "getConsoleLogs",
  "parameters": {
    "limit": 20 // 限制數量以聚焦在錯誤發生前後
  }
}
```

**結果**：Manus 會告訴您：「失敗的請求是 `POST /api/checkout`，伺服器回應了 `500 Internal Server Error`。與此同時，瀏覽器主控台顯示了一個 `API call failed` 的錯誤訊息。」

## 任務三：重現使用者的確切操作步驟

有時候，Bug 的觸發方式非常詭異，您需要知道使用者到底做了什麼。

### 您的指令

> 「我還是不明白使用者是怎麼觸發這個問題的。把這個 Jam 裡他的每一步操作都列出來給我。」

### Manus 的思考與行動

這個指令非常明確。「每一步操作」直接對應到 `getUserEvents` 工具。

*   **單一步驟：獲取使用者事件**
    *   **動作**：調用 `getUserEvents` 工具。
    *   **目的**：獲取一份包含所有點擊、輸入和頁面跳轉的詳細清單。

#### 背後的工具調用

```json
{
  "tool_name": "getUserEvents",
  "parameters": {}
}
```

**結果**：Manus 會以一個有序列表的形式，向您展示使用者的完整操作流程，例如：「1. 前往首頁 -> 2. 點擊『登入』按鈕 -> 3. 在『使用者名稱』欄位輸入『Joker』...」

## 任務四：評估一個問題的影響範圍

一個 Bug 出現了，但它到底影響了多少人？是不是只有某個特定頁面有問題？

### 您的指令

> 「幫我搜一下，過去一週內，所有跟『結帳頁面』相關的 Jam 報告有哪些？」

### Manus 的思考與行動

這個任務需要「搜尋」和「過濾」。

1.  **執行搜尋**：您的關鍵字「搜一下」、「結帳頁面」和「過去一週」會讓 Manus 使用 `searchJams` 工具，並設定 `url` 和時間範圍作為過濾條件。
2.  **呈現結果**：Manus 會將搜尋到的 Jam 清單整理後呈現給您。

#### 背後的工具調用

```json
{
  "tool_name": "searchJams",
  "parameters": {
    "url": "example.com/checkout",
    "date_since": "2026-02-17"
  }
}
```

**結果**：Manus 會給您一個列表，包含所有在過去一週內回報的、與結帳頁面相關的 Jam 連結和標題，讓您能快速評估問題的嚴重性和普遍性。

## 任務五：記錄結論並整理工作區

在您完成分析後，需要記錄下您的發現並將任務歸檔。

### 您的指令

> 「好了，這個問題我搞清楚了。幫我在這個 Jam 上加個註解：『問題根源是支付 API 超時，已轉給後端團隊』。然後把它移到『已分析』資料夾。」

### Manus 的思考與行動

這個指令包含兩個獨立但連續的動作：

1.  **添加註解**：指令中的「加個註解」和引號裡的內容，會讓 Manus 使用 `addComment` 工具。
2.  **移動檔案**：指令中的「把它移到『已分析』資料夾」，會讓 Manus 接著使用 `moveJam` 工具。

#### 背後的工具調用

```json
// 步驟一：添加註解
{
  "tool_name": "addComment",
  "parameters": {
    "comment": "問題根源是支付 API 超時，已轉給後端團隊"
  }
}

// 步驟二：移動 Jam
{
  "tool_name": "moveJam",
  "parameters": {
    "folder": "已分析"
  }
}
```

**結果**：Manus 會回覆您：「操作完成。註解已添加，並且該 Jam 已被移動到『已分析』資料夾。」 您無需手動操作，即可完成記錄和整理工作。

## 結語

如您所見，與 Manus 協作就像與一位聰明且高效的助理對話。您只需要專注於您的「目標」，而 Manus 會負責處理所有底層的工具調用和執行細節。希望本手冊能幫助您更好地利用 Manus 和 Jam，讓您的偵錯工作變得前所未有的輕鬆和愉快。
