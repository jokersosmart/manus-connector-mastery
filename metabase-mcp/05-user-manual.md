**作者：Joker**

## 引言：用對話驅動您的數據洞察

歡迎使用 Metabase 連結器！本手冊旨在為終端使用者提供一份清晰、易懂的操作指南。在這裡，您將學習如何透過簡單的自然語言指令，與 Manus 進行對話，從而輕鬆地完成各種數據查詢與管理任務。您無需關心複雜的技術細節，只需像與一位數據分析師同事交談一樣，提出您的需求即可。

本手冊以「任務」為導向，為您展示一系列常見的應用場景。在每個場景下，我們都會提供具體的「您可以說」的指令範例，並解釋 Manus 在幕後是如何理解您的意圖，並將其轉換為對 Metabase 的具體操作的。

## 1. 查詢與探索數據

這是 Metabase 連結器最核心、最常用的功能。您可以隨時向 Manus 詢問您關心的數據。

### 場景 1.1：執行一個已儲存的查詢

在您的 Metabase 中，團隊成員已經建立了許多有用的「問題」(Questions)。您只需知道問題的 ID，就可以隨時獲取最新的數據。

**當您想...**

查看一個特定報告的最新數據時。

**您可以說：**

> 「幫我執行 Metabase ID 為 568 的問題，然後告訴我結果。」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 識別出您想要「執行」一個 Metabase 的「問題」，並提取出關鍵實體 `ID: 568`。
2.  **工具選擇**：基於您的意圖，Manus 選擇了 `metabase.get_question_results` 這個工具。
3.  **參數對應**：Manus 將您提供的 `568` 作為 `question_id` 參數的值。
4.  **工具調用**：Manus 執行工具調用 `metabase.get_question_results(question_id=568)`。
5.  **結果呈現**：在收到 Metabase 回傳的數據後，Manus 會將其格式化為一個清晰易讀的表格，呈現給您。

### 場景 1.2：匯出查詢結果

有時您需要將數據用於其他地方，例如在 Excel 中製作圖表。

**當您想...**

將一個查詢的結果儲存為檔案時。

**您可以說：**

> 「把 Metabase ID 為 568 的問題的結果，匯出成 CSV 格式。」

**Manus 如何理解與執行：**

1.  **意圖識別**：除了識別出您要執行問題 `568` 外，Manus 還捕捉到了您對「格式」的特定需求 `CSV`。
2.  **工具選擇**：同樣是 `metabase.get_question_results` 工具。
3.  **參數對應**：這次，Manus 會填充兩個參數：`question_id=568` 和 `export_format='csv'`。
4.  **工具調用**：Manus 執行 `metabase.get_question_results(question_id=568, export_format='csv')`。
5.  **結果呈現**：您將會直接收到一段 CSV 格式的純文字，您可以方便地複製和儲存。

### 場景 1.3：執行帶有篩選條件的查詢

對於那些需要動態輸入（例如，選擇日期或客戶）的查詢，您也可以在對話中直接提供。

**當您想...**

查詢特定時間段或特定條件下的數據時。

**您可以說：**

> 「執行 Metabase ID 為 789 的問題，把日期篩選器設定為『上個月』。」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 識別出您不僅要執行問題，還要提供一個「篩選器」或「參數」。
2.  **工具選擇**：依然是 `metabase.get_question_results` 工具。
3.  **參數對應**：Manus 會將您的指令對應到 `question_id=789` 和一個 `parameters` 物件，例如 `parameters=[{'type': 'date/month', 'value': 'previous_month'}]`。
4.  **工具調用**：Manus 執行帶有 `parameters` 參數的工具調用。
5.  **結果呈現**：您將看到經過篩選後的、符合您特定條件的數據結果。

## 2. 盤點與管理資源

除了查詢數據，您還可以使用 Manus 來了解和管理您的 Metabase 中有哪些可用的數據資產。

### 場景 2.1：查看所有可用的資料庫

**當您想...**

快速了解公司都接入了哪些數據源時。

**您可以說：**

> 「列出我們在 Metabase 上的所有資料庫。」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 識別出您想要「列出」或「獲取多個」(`get many`) `資料庫` (`databases`) 資源。
2.  **工具選擇**：Manus 選擇 `metabase.get_many_databases` 工具。
3.  **工具調用**：由於此工具無需參數，Manus 直接執行 `metabase.get_many_databases()`。
4.  **結果呈現**：您會得到一個清單，包含所有資料庫的名稱和 ID。

### 場景 2.2：探索特定資料庫的內容

**當您想...**

在不離開對話的情況下，了解某個資料庫裡有哪些數據表和欄位時。

**您可以說：**

> 「ID 為 23 的資料庫裡都有哪些表？」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 理解您想要獲取一個特定 `資料庫` 的 `欄位` 或 `結構`。
2.  **工具選擇**：Manus 選擇 `metabase.get_database_fields` 工具。
3.  **參數對應**：Manus 將 `ID: 23` 對應到 `database_id=23`。
4.  **工具調用**：Manus 執行 `metabase.get_database_fields(database_id=23)`。
5.  **結果呈現**：您會看到一個結構化的清單，詳細列出了該資料庫下的所有數據表及其包含的欄位。

## 3. 監控與警報

對於數據驅動的團隊來說，及時發現問題與機遇至關重要。Metabase 的警報功能可以幫助您自動監控關鍵指標，而 Manus 則讓您能隨時隨地管理這些警報。

### 場景 3.1：查看一個警報的設定

**當您想...**

確認某個警報的觸發條件或通知對象時。

**您可以說：**

> 「幫我看看 ID 是 12345 的 Metabase 警報是怎麼設定的？」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 識別出您想要「獲取單一」(`get a`) `警報` (`alert`) 的資訊。
2.  **工具選擇**：Manus 選擇 `metabase.get_alert` 工具。
3.  **參數對應**：Manus 將 `ID: 12345` 對應到 `alert_id=12345`。
4.  **工具調用**：Manus 執行 `metabase.get_alert(alert_id=12345)`。
5.  **結果呈現**：您會得到一份關於該警報的詳細報告，包括它監控的是哪個「問題」，觸發條件是什麼，以及它會通知誰。

### 場景 3.2：盤點所有生效中的警報

**當您想...**

對團隊目前所有的數據監控點有一個全局的了解時。

**您可以說：**

> 「列出我所有的 Metabase 警報。」

**Manus 如何理解與執行：**

1.  **意圖識別**：Manus 識別出您想要「列出」或「獲取多個」(`get many`) `警報` (`alerts`)。
2.  **工具選擇**：Manus 選擇 `metabase.get_many_alerts` 工具。
3.  **工具調用**：Manus 執行 `metabase.get_many_alerts()`。
4.  **結果呈現**：您會得到一個簡潔的清單，列出您所有警報的名稱和 ID，讓您對現有的監控體系一目了然。
