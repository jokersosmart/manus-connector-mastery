# 「Linear」連結器遞迴分析報告

**作者：Joker**

---

本文使用「4! × 五何法」無限遞迴分析引擎，對 Manus MCP「Linear」連結器的核心功能、目標用戶、應用場景、潛在價值與技術挑戰，進行深度拆解分析。

## 第一層分析：核心功能 (What)

### 1. 這是一個什麼樣的連結器？

「Linear」是一個專為軟體開發團隊設計的專案管理工具。此 MCP 連結器旨在將 Linear 的核心功能整合至 Manus Agent，讓使用者能透過自然語言指令，無縫地在 Manus 環境中操作 Linear 上的議題 (Issues)、專案 (Projects) 與週期 (Cycles)。

### 2. 它的核心功能有哪些？

根據官方 API 文件，其核心功能圍繞著 GraphQL API，主要可歸納為以下幾點：

*   **議題管理 (Issue Tracking):** 建立、查詢、更新、刪除議題，包含指派、狀態變更、評論、附加檔案等。
*   **專案規劃 (Project Management):** 建立與管理專案，將議題歸納至特定專案中，並追蹤專案進度。
*   **週期衝刺 (Cycle Sprints):** 規劃與執行開發週期 (Sprints)，將議題排入特定週期，並監控團隊的開發速率。
*   **團隊協作 (Team Collaboration):** 管理團隊成員、指派任務、並透過評論與提及 (mentions) 進行溝通。
*   **視圖與篩選 (Views & Filtering):** 建立自訂視圖，並使用強大的篩選功能來尋找特定的議題或專案。
*   **Webhook 與自動化:** 透過 Webhook 訂閱事件，實現與其他工具的整合與自動化工作流程。

### 3. 為什麼我們需要這個連結器？ (Why)

在 Manus 的工作流程中，開發者或專案經理時常需要在不同的工具間切換，以完成任務的閉環。例如，在收到一個來自 Slack 的使用者回饋後，需要在 Linear 中建立一個新的議題來追蹤此問題。有了「Linear」連結器，使用者將能直接在 Manus 中下達「在 Linear 為『修復登入頁面樣式』建立一個新議題」這樣的指令，從而省去手動切換應用程式與複製貼上資訊的麻煩，顯著提升工作效率。

### 4. 誰會是主要使用者？ (Who)

*   **軟體工程師:** 快速回報 Bug、更新任務進度、查詢自己被指派的議題。
*   **專案經理 (PM):** 建立新專案、規劃開發週期、追蹤團隊進度、產出報表。
*   **產品設計師:** 將設計稿附加到相關議題、參與功能討論。
*   **技術支援團隊:** 將使用者回報的問題轉化為可追蹤的議題。

### 5. 在哪些場景下會被使用？ (Where & When)

*   **日常開發流程:** 開發者完成一個功能後，更新 Linear 上對應議題的狀態。
*   **每週會議:** 專案經理透過 Manus 快速拉取 Linear 的專案進度，並產生報告。
*   **緊急事件處理:** 收到線上問題警報後，立即在 Linear 建立一個高優先度的議題並指派給相關人員。
*   **跨團隊協作:** 當設計團隊完成 UI Mockup 後，自動在對應的開發議題下留言並附上設計稿連結。
