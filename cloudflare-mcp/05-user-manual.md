# OpenRouter 終端使用者操作手冊

**作者：Joker**

---

## 前言：您的 AI 團隊，隨時待命

歡迎！本手冊將向您展示如何透過簡單的自然語言，指揮一個由全世界最頂尖 AI 模型組成的「專家團隊」。這個團隊的核心，就是 OpenRouter 連結器。

過去，您與 AI 的互動可能像是在和一位全科醫生對話——他什麼都懂一點，但並非樣樣精通。現在，有了 OpenRouter，您擁有了一整本專家名錄。您可以隨時召喚專門的「程式設計師」、「文案寫手」、「數據分析師」或「創意大師」來為您解決特定的問題。

本手冊以任務為導向，將透過具體的範例，向您揭示 Manus 如何理解您的指令，並將其巧妙地轉換為對這些 AI 專家的調用。理解這個過程，將幫助您更精準、更高效地與您的 AI 團隊協作。

---

### **任務一：進行一次普通的對話**

這是最基礎的互動，就像您和任何一位 AI 助理聊天一樣。

*   **您的指令：**
    > 「法國的首都是哪裡？」

*   **Manus in Action (幕後揭秘)：**
    當您沒有指定任何專家時，Manus 會為您選擇一位可靠的「通才型」專家（預設模型）來回答您的問題。這個過程可以被理解為以下的工具調用：

    ```json
    {
      "expert_called": "通才型 AI (例如 google/gemini-1.5-flash)",
      "task": "回答關於法國首都的問題。"
    }
    ```

*   **核心 takeaway：** 即使您什麼都不指定，Manus 也會為您選擇一位合適的 AI 來完成任務。

---

### **任務二：召喚一位「程式設計專家」**

當您有專業問題時，直接點名該領域的專家，會讓您的成果品質大幅提升。

*   **您的指令：**
    > 「**請讓 `openai/gpt-4o` 幫我看看**，這段 Javascript 程式碼有沒有可以優化的地方？」

*   **Manus in Action (幕後揭秘)：**
    Manus 辨識出了關鍵字 `openai/gpt-4o`，這是一位在程式碼領域享有盛譽的專家。於是，它會精準地將您的程式碼和問題，只發送給這位專家。

    ```json
    {
      "expert_called": "openai/gpt-4o",
      "task": "檢查並優化一段 Javascript 程式碼。"
    }
    ```

*   **核心 takeaway：** 直接說出專家的「名字」（模型名稱），是確保專業問題得到最佳解答的最直接方法。

---

### **任務三：讓「創意雙人組」進行腦力激盪**

有時候，您需要的不是單一的答案，而是多元的靈感。這時，您可以讓不同的專家分別提出看法。

*   **您的指令：**
    > 「我想為我的新 podcast 節目取個名字，主題是關於心靈成長。**請 `anthropic/claude-3.5-sonnet` 和 `mistralai/mistral-large-latest` 各給我五個建議。**」

*   **Manus in Action (幕後揭秘)：**
    Manus 會將您的請求分發給兩位在創意寫作領域各具風格的專家，並將他們的結果分別呈現給您，供您比較和挑選。

    *   **調用 1：**
        ```json
        {
          "expert_called": "anthropic/claude-3.5-sonnet",
          "task": "為心靈成長主題的 podcast 提供五個名字建議。"
        }
        ```
    *   **調用 2：**
        ```json
        {
          "expert_called": "mistralai/mistral-large-latest",
          "task": "為心靈成長主題的 podcast 提供五個名字建議。"
        }
        ```

*   **核心 takeaway：** 您可以同時諮詢多位專家，讓他們的「思想碰撞」為您帶來更多靈感。

---

### **任務四：指揮一位專家的「工作風格」**

您不僅可以選擇專家，甚至可以微調他們的工作風格——是嚴謹保守，還是天馬行空。

*   **您的指令：**
    > 「幫我想一個關於時間旅行的電影點子。**我希望點子越瘋狂、越大膽越好。**」

*   **Manus in Action (幕後揭秘)：**
    Manus 偵測到您對「創意程度」的要求（瘋狂、大膽）。它會將這個要求轉換為一個名為 `temperature` 的內部參數。越高的值代表越高的創意和隨機性。

    ```json
    {
      "expert_called": "(一位擅長創意的 AI，例如 mistralai/mistral-large-latest)",
      "task": "想一個關於時間旅行的電影點子。",
      "style_adjustment": {
        "temperature": 1.4
      }
    }
    ```

*   **核心 takeaway：** 透過描述性的詞語（如「嚴謹」、「有創意」、「保守一點」），您可以影響 AI 回答的風格。

---

### **任務五：組建一條「專家流水線」**

這是最高階的技巧：將複雜任務拆解，交給一個由不同專家組成的流水線來協同完成。

*   **您的指令：**
    > 「**首先，請 `google/gemini-1.5-flash` 快速閱讀這份財報（[附上連結]），並總結出最重要的五個財務指標。**」
    > 
    > （在上一位專家完成後）
    > 
    > 「**很好。現在，請 `anthropic/claude-3.5-sonnet` 根據上面的五個指標，為我撰寫一段給股東的摘要報告，語氣要沉穩且專業。**」

*   **Manus in Action (幕後揭秘)：**
    Manus 會忠實地執行您的兩步流程，確保任務的每一步都由最合適的專家處理。

    *   **流水線步驟 1 (數據提取)：**
        ```json
        {
          "expert_called": "google/gemini-1.5-flash",
          "task": "閱讀財報並提取五個關鍵財務指標。"
        }
        ```
    *   **流水線步驟 2 (報告撰寫)：**
        ```json
        {
          "expert_called": "anthropic/claude-3.5-sonnet",
          "task": "基於提供的五個指標，撰寫一段給股東的摘要報告。",
          "style_adjustment": {
            "tone": "沉穩、專業"
          }
        }
        ```

*   **核心 takeaway：** 您是整個 AI 團隊的總指揮。透過將任務拆解，並依序指派給不同的專家，您可以完成極其複雜且對品質要求極高的工作。

掌握 OpenRouter 的關鍵，在於將您的思維從「向 AI 提問」轉變為「向團隊分配任務」。請大膽地在您的日常工作中使用本手冊的指令範例，您會驚訝地發現，您的生產力將會達到一個全新的高度。
