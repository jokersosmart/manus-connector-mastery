# 「Intercom」連接器學習手冊

**作者：Joker**

## 1. 歡迎與導覽

歡迎使用 Intercom 連接器學習手冊！本手冊旨在幫助您在最短的時間內掌握核心技能，並將其應用於您的日常工作中，以顯著提升客戶互動效率。

**學習理念：** 我們相信，最好的學習方式是「帶著目標去實踐」。因此，本手冊將以任務為導向，每個章節都會聚焦於一個具體的、高價值的應用場景。您將透過完成這些任務，自然而然地掌握相關的工具和概念。

**手冊結構：**

*   **學習目標：** 清晰定義您在每個章節結束後應該具備的能力。
*   **核心概念：** 解釋完成任務所需的基礎知識。
*   **實踐練習：** 提供具體的指令和場景，讓您動手操作。
*   **進階思考：** 鼓勵您將所學知識應用到更複雜的場景中。

## 2. 模組一：客戶資料管理員

**學習目標：** 您將能夠在不離開 Manus 的情況下，完成對 Intercom 中聯絡人和公司的建立、查詢、更新等基本操作。

### 核心概念

*   **聯絡人 (Contact):** 您客戶的數位化身，由唯一的 `email` 標識。
*   **公司 (Company):** 您的 B2B 客戶，可以關聯多個聯絡人。
*   **自訂屬性 (Custom Attributes):** 為聯絡人或公司添加的結構化標籤，如 `plan: "premium"`，是實現精準自動化的基礎。

### 實踐練習

**任務 1：建立您的第一個客戶**

*   **場景：** 您剛剛透過電話完成一筆交易，需要將客戶資訊錄入系統。
*   **指令：**
    > 「在 Intercom 中建立一個新使用者，信箱是 `new.customer@example.com`，姓名是『John Smith』，並記錄他的公司 ID 為 `acme-corp-123`。」
*   **內部工具調用：** `create_contact`

**任務 2：更新客戶資訊**

*   **場景：** 客戶 John Smith 升級了他的服務方案。
*   **指令：**
    > 「更新 Intercom 中 `new.customer@example.com` 的資訊，將他的自訂屬性『plan』設置為『enterprise』。」
*   **內部工具調用：** `update_contact`

**任務 3：查詢客戶全貌**

*   **場景：** 在與客戶溝通前，您需要快速了解他的所有背景資訊。
*   **指令：**
    > 「幫我查詢 Intercom 裡 `new.customer@example.com` 的所有資訊。」
*   **內部工具調用：** `get_contact`

### 進階思考

您能否設計一個指令，一次性建立一家公司，並將三位來自該公司的聯絡人與之關聯？

## 3. 模組二：高效對話處理者

**學習目標：** 您將能夠處理日常的客戶對話，包括回覆、添加內部註記和分類標記，將平均處理時間縮短 30%。

### 核心概念

*   **對話 (Conversation):** 所有互動的容器。
*   **公開回覆 (Comment):** 客戶可見的回覆。
*   **內部註記 (Note):** 僅團隊內部可見的溝通與備註，是團隊協作的關鍵。
*   **標籤 (Tag):** 為對話打上的分類標記，如 `billing-issue` 或 `bug-report`，用於後續的分析和流程自動化。

### 實踐練習

**任務 1：快速回覆客戶**

*   **場景：** 一位客戶發來消息：「我的軟體無法登入」。
*   **指令：**
    > 「回覆 Intercom 上的最新對話：『您好，請問您能提供您的帳號信箱嗎？以便我們為您排查問題。』」
*   **內部工具調用：** `reply_to_conversation`

**任務 2：團隊內部協作**

*   **場景：** 您需要工程師的幫助來解決客戶問題。
*   **指令：**
    > 「在最新的對話中加入內部註記：『@engineer-dave，用戶回報無法登入，疑似帳號鎖定，請協助檢查後台。』」
*   **內部工具調用：** `add_note_to_conversation`

**任務 3：分類與追蹤**

*   **場景：** 您判斷這是一個緊急的技術問題。
*   **指令：**
    > 「將當前的對話標記為『technical-issue』和『urgent』。」
*   **內部工具調用：** `tag_conversation`

### 進階思考

您能否設計一個指令，在回覆客戶的同時，自動根據回覆內容（例如包含「帳單」、「發票」等關鍵詞）為對話打上 `billing` 標籤？

## 4. 模組三：自動化流程設計師

**學習目標：** 您將能夠串聯多個操作，設計並執行自動化工作流，處理複雜的多步驟任務。

### 核心概念

*   **工作流 (Workflow):** 將一系列獨立的指令串聯起來，以完成一個更大的目標。這是從「操作者」到「設計師」的關鍵一步。
*   **觸發器 (Trigger):** 啟動工作流的條件（在本手冊中，觸發器是您的自然語言指令）。
*   **組合操作 (Chained Actions):** 一個操作的輸出可以作為下一個操作的輸入，實現流程的自動流轉。

### 實踐練習

**任務：設計一個「新用戶引導」工作流**

*   **場景：** 每當有新用戶註冊時，您都需要發送歡迎郵件、分配客戶經理，並建立一個跟進任務。
*   **指令 (宏指令):**
    > 「啟動『新用戶歡迎』流程，目標用戶 `newbie@example.com`：
    > 1.  在 Intercom 中對該用戶發起對話，內容是：『歡迎加入我們！這是一份快速入門指南...』
    > 2.  將該對話指派給管理員『Alex』。
    > 3.  在對話中加入內部註記：『請 Alex 在 3 天內完成首次跟進聯繫。』」
*   **內部工具調用：** `create_conversation` -> `assign_conversation` -> `add_note_to_conversation`

### 進階思考

您能否設計一個「沉睡客戶喚醒」工作流？例如，自動篩選出超過 30 天未登入的用戶 (`list_contacts` 配合自訂屬性)，並對他們發起一個個人化的關懷訊息 (`create_conversation`)。
