# 「Intercom」連接器終端使用者操作手冊

**作者：Joker**

## 1. 簡介：您的智慧客服助理

歡迎使用 Intercom 連接器！把它想像成您專屬的、精通 Intercom 的智慧助理。您不再需要記住繁瑣的點擊步驟，只需用最自然的語言告訴您的 Manus Agent 您想做什麼，它就能為您完成在 Intercom 平台上的各種操作。

本手冊將以任務為導向，向您展示如何透過簡單的對話，完成從管理客戶資料到處理複雜客戶對話的各種工作。

## 2. 核心任務範例

以下是您在日常工作中可能遇到的場景，以及如何讓 Manus 為您效勞。

### 任務一：管理客戶與公司

**場景 1：快速新增一位潛在客戶**

*   **您說：**
    > 「幫我加一個 Intercom 潛在客戶，信箱是 `potential.lead@startup.com`，名字叫『Alex Chen』。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「建立」一個「潛在客戶 (lead)」。
    2.  **實體提取：** 提取出 `email: "potential.lead@startup.com"` 和 `name: "Alex Chen"`。
    3.  **工具調用：** 執行 `intercom.create_contact` 工具，並傳入以下參數：
        ```json
        {
          "role": "lead",
          "email": "potential.lead@startup.com",
          "name": "Alex Chen"
        }
        ```

**場景 2：查詢客戶的詳細資料**

*   **您說：**
    > 「查一下 Intercom 裡『Alex Chen』的資料，我想看看他的方案是什麼。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「查詢」一個「聯絡人」的資訊。
    2.  **實體提取：** 提取出 `name: "Alex Chen"`。
    3.  **工具調用：** 執行 `intercom.get_contact` 工具，並傳入參數 `{"name": "Alex Chen"}` 或 `{"email": "..."}`（如果之前已知）。
    4.  **結果呈現：** 將返回的客戶資料（包含自訂屬性中的方案資訊）以清晰的格式呈現給您。

**場景 3：將客戶與公司關聯**

*   **您說：**
    > 「把 Intercom 裡的 Alex Chen 加到『Global Tech Inc.』這家公司裡。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「更新」一個「聯絡人」，將其與「公司」關聯。
    2.  **實體提取：** 提取出聯絡人「Alex Chen」和公司「Global Tech Inc.」。
    3.  **多步執行：**
        *   首先，可能需要調用 `intercom.get_company` 找到「Global Tech Inc.」的 ID。
        *   然後，調用 `intercom.update_contact`，在 Alex Chen 的資料中加入公司的關聯 ID。

### 任務二：處理客戶對話

**場景 1：回覆客戶的詢問**

*   **您說：**
    > 「回覆 Intercom 上最新的那則對話：『您好，我們已經收到您的問題，正在為您處理。』」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「回覆」一則「對話」。
    2.  **上下文理解：** 理解「最新」是對話的時間定語。
    3.  **實體提取：** 提取出回覆內容 `"您好，..."`。
    4.  **工具調用：** 執行 `intercom.reply_to_conversation`，將您的回覆內容作為 `body` 參數發送。

**場景 2：在對話中與同事協作**

*   **您說：**
    > 「在當前對話裡加個內部備註，提醒『@lisa』明天要跟進這個客戶。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想添加「內部註記 (note)」。
    2.  **實體提取：** 提取出註記內容 `"@lisa 明天要跟進這個客戶。"`。
    3.  **工具調用：** 執行 `intercom.add_note_to_conversation`，`message_type` 設置為 `note`，確保客戶不可見。

**場景 3：為對話打上分類標籤**

*   **您說：**
    > 「把這個對話標記成『功能建議』。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「標記 (tag)」一則「對話」。
    2.  **實體提取：** 提取出標籤名稱 `"功能建議"`。
    3.  **工具調用：** 執行 `intercom.tag_conversation`。

### 任務三：搜尋知識庫

**場景：快速找到答案以回覆客戶**

*   **您說：**
    > 「在我們的 Intercom 知識庫裡，找一下關於『如何重設密碼』的文章。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到您想「搜尋」「知識庫文章」。
    2.  **實體提取：** 提取出搜尋關鍵詞 `"如何重設密碼"`。
    3.  **工具調用：** 執行 `intercom.search_articles`。
    4.  **結果呈現：** 將搜尋到的文章標題和連結列表呈現給您，供您選擇並發送給客戶。

## 3. 組合指令：您的超能力

當您熟練後，可以嘗試將多個指令組合在一起，實現更高效的工作流。

*   **您說：**
    > 「回覆最新的 Intercom 對話：『您的問題已轉交技術團隊處理。』，然後把這個對話標記為『技術支援』並指派給『工程師小組』。」
*   **Manus 如何理解與執行：**
    1.  **意圖識別：** 識別到一個包含「回覆」、「標記」和「指派」三個動作的複合意圖。
    2.  **順序執行：**
        *   調用 `intercom.reply_to_conversation`。
        *   調用 `intercom.tag_conversation`。
        *   調用 `intercom.assign_conversation`。

## 4. 結語

如果您有任何疑問或發現了新的、高效的使用方式，請隨時與您的團隊分享！
