# HubSpot 從新手到精通：掌握對話式 CRM 分析

本指南旨在為您提供一條清晰的學習路徑，從理解 HubSpot MCP 連接器的基礎概念開始，逐步引導您掌握進階查詢技巧，最終能夠在真實世界的業務場景中，像大師一樣透過自然語言與您的 CRM 數據對話。

作者：Joker

---

### 第一章：基礎入門 (Novice)

#### 1.1 什麼是 HubSpot MCP 連接器？

想像一下，您可以直接用說話或打字的方式，讓 Manus 為您從 HubSpot 中提取數據，而無需點擊無數個篩選器或導航複雜的儀表板。HubSpot MCP 連接器就是實現這一魔法的橋樑。它將您的自然語言指令，轉化為對 HubSpot 數據庫的精確查詢，讓數據探索變得前所未有的直觀和高效。

#### 1.2 核心概念

- **自然語言互動**: 您與 Manus 的對話是核心。您提出問題，Manus 透過連接器找到答案。
- **CRM 物件 (Objects)**: 您的 HubSpot 數據由不同的物件組成，如 `contacts` (聯絡人)、`deals` (交易) 和 `companies` (公司)。了解這些基本單位是查詢的基礎。
- **唯讀存取**: 目前，連接器只會「讀取」您的數據，絕不會修改或刪除任何資訊，確保您的 CRM 資料絕對安全。

#### 1.3 您的第一個查詢

讓我們從一個簡單的任務開始。這相當於程式設計中的 "Hello, World!"。嘗試對 Manus 說：

> 「幫我找一下最新的 5 個聯絡人。」

Manus 會在幕後調用 `search_crm_objects` 工具，設定 `object_types` 為 `["contacts"]`，並將 `sorts` 按 `createdate` 降序排列，`limit` 設為 5，然後將結果呈現給您。恭喜，您已經完成了第一步！

### 第二章：核心功能 (Intermediate)

#### 2.1 精通 `search_crm_objects`

這是您最主要的工具。學會精準地使用篩選 (`filters`) 和排序 (`sorts`) 是進階的關鍵。

**進階篩選：**

您不再僅限於簡單的查詢。嘗試更複雜的條件：

> 「找出所有在『簡報排程』階段，並且金額大於 10,000 美元的交易。」

這裡，Manus 會在 `filters` 中設定兩個條件：一個針對 `dealstage`，另一個針對 `amount`。

#### 2.2 組合多重條件

現實世界的業務問題往往是複雜的。您可以組合 `query`（模糊搜尋）和 `filters`（精確條件）來縮小範圍。

> 「搜尋一下所有在加州，行業為『軟體』的公司，並且員工人數超過 200 人。」

這個指令會將 "加州" 作為 `query` 進行模糊比對，同時用 `filters` 精確匹配 `industry` 和 `numberofemployees`。

#### 2.3 理解關鍵數據屬性

花點時間熟悉 HubSpot 中常用物件的關鍵屬性至關重要。例如：

- **Deals**: `dealname`, `amount`, `dealstage`, `closedate`
- **Contacts**: `firstname`, `lastname`, `email`, `lifecyclestage`
- **Companies**: `name`, `industry`, `city`, `country`

### 第三章：進階技巧 (Advanced)

#### 3.1 鏈式查詢

大師級的技巧在於將多個查詢串聯起來。您可以將一個查詢的結果，作為下一個查詢的輸入。

> 「首先，找出我們最大的那筆交易。然後，告訴我負責該交易的聯絡人是誰。」

Manus 會先執行一次 `search_crm_objects` 找到金額最高的交易，從結果中提取 `associatedcontactid`，然後再調用 `get_object_details` 來顯示該聯絡人的詳細資訊。

#### 3.2 跨物件思考

雖然工具本身是獨立的，但您的「問題」可以跨越物件。您需要引導 Manus 進行關聯查詢。

> 「列出所有由『大型科技公司』來的，且處於『潛在客戶』階段的聯絡人。」

這需要 Manus 先找到所有符合「大型科技公司」條件的公司，獲取它們的 ID，然後再用這些公司 ID 去篩選聯絡人。

### 第四章：實戰應用 (Master)

現在，您已準備好解決真實的業務問題。以下是一些場景，試著用您學到的知識來建構對應的自然語言指令。

#### 4.1 銷售管道分析

- **場景**: 銷售經理想要找出停滯不前的交易。
- **您可以問**: 「找出所有超過一週沒有任何活動更新，並且還在『提案』階段的交易。」

#### 4.2 市場潛在客戶資格鑑定

- **場景**: 市場團隊需要快速識別高價值潛在客戶。
- **您可以問**: 「列出過去 24 小時內，從我們官網註冊，並且公司規模超過 100 人的所有新聯絡人。」

#### 4.3 客戶支援洞察

- **場景**: 客戶成功團隊需要優先處理緊急問題。
- **您可以問**: 「顯示所有建立超過 3 天，優先級為『高』，但狀態仍為『開啟』的支援票證。」

## 結語

從新手到精通的旅程，核心在於思維模式的轉變：從思考「我該用哪個工具」，轉變為思考「我該問什麼問題」。HubSpot MCP 連接器是您強大的盟友，它讓您能專注於業務本身，而將數據提取的複雜性交給 Manus。不斷練習，不斷探索，您將解鎖前所未有的數據洞察力。
