# HubSpot 連接器使用者操作手冊

歡迎使用 HubSpot 連接器！本手冊將引導您如何透過簡單的自然語言指令，與您的 HubSpot CRM 數據進行互動。忘掉複雜的報告和篩選器，從現在開始，您只需「提問」，Manus 就會為您找到答案。

作者：Joker

---

## 它是如何運作的？

整個過程非常簡單，可以分為三個步驟：

1.  **您提問**：您用日常語言向 Manus 提出一個關於 HubSpot 數據的問題。
2.  **Manus 理解與轉換**：Manus 的 AI 大腦會分析您的問題，並將其轉換成一個 HubSpot 連接器可以理解的「工具調用」指令。
3.  **連接器獲取數據**：連接器根據指令，安全地從您的 HubSpot 帳戶中讀取數據，並將結果呈現給您。

這個過程確保您無需學習任何程式碼或 API 知識，就能輕鬆駕馭強大的 CRM 數據。

## 任務導向指令範例

以下是針對不同團隊在日常工作中可能遇到的場景，以及如何使用自然語言指令來解決這些問題。

### 業務團隊

#### 任務：快速概覽我的銷售管道

- **您的指令**：
  > 「顯示我所有正在進行中的交易，按預計關閉日期排序。」

- **Manus 幕後工作 (示意)**：
  Manus 會調用 `search_crm_objects` 工具，尋找所有 `dealstage` 不屬於「已成交」或「已流失」的 `deals`，並按 `closedate` 升序排列。

#### 任務：找出最有價值的潛在客戶

- **您的指令**：
  > 「列出所有金額超過 20,000 美元的交易，並按金額從高到低顯示。」

- **Manus 幕後工作 (示意)**：
  Manus 調用 `search_crm_objects`，設定 `object_types` 為 `["deals"]`，並在 `filters` 中加入 `amount > 20000` 的條件，最後按 `amount` 降序排序。

#### 任務：準備客戶會議

- **您的指令**：
  > 「告訴我關於『陽光科技公司』的所有資訊，包括他們最近的交易和聯絡人。」

- **Manus 幕後工作 (示意)**：
  這是一個鏈式查詢。Manus 會：
  1.  先用 `search_crm_objects` 找到名為「陽光科技公司」的 `companies` 物件。
  2.  提取該公司的 ID。
  3.  再進行兩次查詢：一次是查找與該公司 ID 關聯的 `deals`，另一次是查找關聯的 `contacts`。

### 市場行銷團隊

#### 任務：分析活動成效

- **您的指令**：
  > 「找出過去一週內，透過『夏季特惠』活動新增加的所有聯絡人。」

- **Manus 幕後工作 (示意)**：
  Manus 調用 `search_crm_objects`，篩選 `contacts`。它會在 `filters` 中設定 `createdate` 在過去一週內，並且 `hs_latest_source` 或其他相關屬性為「夏季特惠」。

#### 任務：建立目標客群列表

- **您的指令**：
  > 「給我一份所有居住在紐約，並且對『企業解決方案』感興趣的聯絡人名單。」

- **Manus 幕後工作 (示意)**：
  Manus 調用 `search_crm_objects`，在 `query` 中可能包含 "New York"，並在 `filters` 中設定 `interest = '企業解決方案'`。

### 客戶服務團隊

#### 任務：處理緊急支援請求

- **您的指令**：
  > 「顯示所有優先級為『高』且尚未分配處理人員的支援票證。」

- **Manus 幕後工作 (示意)**：
  Manus 調用 `search_crm_objects`，查詢 `tickets` 物件，並在 `filters` 中設定 `priority = 'HIGH'` 和 `hubspot_owner_id` 為空。

## 提升指令效果的技巧

- **盡量具體**：與其說「找些聯絡人」，不如說「找最近三天內新增的聯絡人」。
- **使用數字**：明確的數字（如「金額大於 5000」）比模糊的詞（如「大額交易」）效果更好。
- **指明排序**：告訴 Manus 您希望結果「按什麼排序」，可以幫助您更快看到重點。
- **一次專注一件事**：雖然 Manus 很聰明，但將複雜問題拆解成幾個簡單問題，通常能得到更精準的答案。
