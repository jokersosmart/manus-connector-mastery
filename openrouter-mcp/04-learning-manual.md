# Airtable 連結器學習手冊

**作者：Joker**

---

## 1. 歡迎！30分鐘內上手 Airtable

本手冊專為希望快速掌握 Airtable 與 Manus 協作能力的你而設計。我們將跳過所有複雜的理論，直接聚焦於最核心、最實用的技能。跟隨本手冊的學習目標和實踐練習，你將在 30 分鐘內建立起操作 Airtable 資料庫的自信。

## 2. 學習目標

完成本手冊後，你將能夠：

1.  **理解核心概念**：能區分 Base、Table、Record 和 Field 的不同。
2.  **掌握資訊查詢**：能用自然語言從 Airtable 中篩選和檢索你需要的資料。
3.  **學會資料維護**：能輕鬆地新增、修改和刪除 Airtable 中的紀錄。
4.  **建立簡單工作流**：能將上述技能組合起來，管理一個簡單的個人專案。

## 3. 關鍵技能一：問問題 (資訊查詢)

你與 Airtable 互動最頻繁的動作就是「問問題」。這對應著 `list_records` 工具，讓你像與助理對話一樣獲取資訊。

- **學習目標**：學會用 `list_records` 檢索和篩選資料。
- **核心指令**：「幫我列出/尋找/篩選...」

**實踐練習**：

假設你有一個名為「內容創作」的 Base，其中有一個「文章點子」的 Table，包含以下欄位：`標題`、`狀態` (點子/撰寫中/已發布)、`主題` (Airtable/Manus/AI)、`預計發布日`。

現在，請嘗試對 Manus 下達以下指令：

1.  「列出『文章點子』表格中的所有紀錄。」 (基本查詢)
2.  「幫我找找看所有狀態是『撰寫中』的文章。」 (單一條件篩選)
3.  「篩選出主題是『AI』而且狀態不是『已發布』的所有點子。」 (多重條件篩選)
4.  「列出所有預計發布日在本週的文章，並按日期排序。」 (日期與排序)

## 4. 關鍵技能二：動動手 (資料維護)

除了查詢，你還需要新增和修改資料。這對應著 `create_record` 和 `update_record` 工具。

- **學習目標**：學會使用 `create_record` 和 `update_record` 來維護資料。
- **核心指令**：「新增一筆...」、「把...改成...」

**實踐練習**：

繼續使用上面的「內容創作」Base。

1.  **新增紀錄**：
    - > 「在『文章點子』表格中新增一筆紀錄，標題是『我的第一個 Airtable 工作流』，狀態是『點子』，主題是『Airtable』。」
2.  **取得紀錄 ID**：新增成功後，Manus 會回傳新紀錄的 ID (例如 `recXXXXXXXXXXXXXX`)，請記下它。
3.  **更新紀錄**：
    - > 「把 ID 是 `recXXXXXXXXXXXXXX` 的紀錄狀態改成『撰寫中』。」
4.  **再次更新**：
    - > 「更新一下 ID `recXXXXXXXXXXXXXX` 的紀錄，把它的預計發布日設為下週一。」

## 5. 綜合實踐：打造你的第一個迷你專案

現在，我們將所有技能結合，建立一個簡單的個人待辦事項清單。

- **學習目標**：整合查詢和維護技能，管理一個完整的工作流程。

**專案設置**：

1.  在 Airtable 中建立一個新的 Base，命名為「我的任務」。
2.  在 Base 中建立一個 Table，命名為「待辦事項」，包含欄位：`任務名稱` (單行文字)、`狀態` (單選：待辦/進行中/已完成)、`優先級` (單選：高/中/低)、`截止日期` (日期)。

**工作流演練**：

1.  **新增任務**：
    - > 「在『待辦事項』裡新增一個任務：『完成 Airtable 學習手冊』，優先級設為『高』，截止日期是今天。」
    - > 「再加一個任務：『整理桌面』，優先級是『低』。」
2.  **檢視任務**：
    - > 「今天有哪些任務要完成？」
    - > 「列出所有優先級是『高』的待辦任務。」
3.  **開始工作**：
    - > (假設「完成 Airtable 學習手冊」的 ID 是 `rec12345678`) 「把任務 `rec12345678` 的狀態改成『進行中』。」
4.  **完成任務**：
    - > 「任務 `rec12345678` 已經搞定，幫我把狀態更新為『已完成』。」
5.  **回顧**：
    - > 「列出本週所有已完成的任務。」
