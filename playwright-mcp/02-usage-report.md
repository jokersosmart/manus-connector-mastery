# 「Close」連結器使用說明報告

作者：Joker

## 簡介

本報告提供 Manus MCP 連結器「Close」的詳細使用說明。Close 是一個專為銷售團隊設計的客戶關係管理 (CRM) 平台，此連結器將其強大的功能整合至 Manus，讓使用者能透過自然語言指令，無縫地管理客戶、追蹤銷售機會並自動化溝通流程。

報告將詳述每個可用工具的功能、參數、使用範例及預期輸出，旨在幫助使用者充分利用此連結器，提升銷售效率。

**參考資料**：
[1] Close API Documentation. (https://developer.close.com/)

## 核心工具 (Core Tools)

核心工具圍繞著 Close CRM 的主要物件，包括潛在客戶 (Leads)、聯絡人 (Contacts) 和銷售機會 (Opportunities)。

### 1. Lead (潛在客戶)

潛在客戶是 Close 中的核心物件，代表一個公司或組織。所有相關的聯絡人、活動和機會都會附加到一個 Lead 上。

#### 1.1 `create_lead`

- **功能說明**：在 Close 中建立一個新的潛在客戶。
- **參數**：
    - `name` (必要)：公司或組織的名稱 (字串)。
    - `status` (可選)：潛在客戶的狀態，例如 `Potential` 或 `Qualified` (字串)。
    - `contacts` (可選)：與此潛在客戶關聯的聯絡人列表 (字典列表)，每個聯絡人字典可包含 `name` 和 `emails`。
    - `custom_fields` (可選)：自訂欄位的鍵值對 (字典)。
- **使用範例**：
  > 「在 Close 新增一個叫做『Innovate Corp』的潛在客戶，狀態是 Potential，聯絡人是『John Doe』，信箱是『john.doe@innovatecorp.com』。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Lead 的詳細資訊，如 ID、名稱、狀態和關聯的聯絡人。

#### 1.2 `get_lead`

- **功能說明**：根據 ID 或名稱搜尋並取得一個或多個潛在客戶的資訊。
- **參數**：
    - `id` (可選)：潛在客戶的唯一 ID (字串)。
    - `name` (可選)：潛在客戶的名稱 (字串)。若提供名稱，可能返回多個結果。
- **使用範例**：
  > 「幫我找一下 Close 裡面叫做『Innovate Corp』的客戶資料。」
- **預期輸出**：一個或一組 JSON 物件，包含符合條件的 Lead 的詳細資訊。

#### 1.3 `update_lead`

- **功能說明**：更新一個已存在的潛在客戶資訊。
- **參數**：
    - `id` (必要)：要更新的潛在客戶的唯一 ID (字串)。
    - `name` (可選)：新的公司名稱 (字串)。
    - `status` (可選)：新的客戶狀態 (字串)。
    - `custom_fields` (可選)：要更新或新增的自訂欄位 (字典)。
- **使用範例**：
  > 「將 ID 是 `lead_xxxxxxxx` 的客戶狀態更新為『Qualified』。」
- **預期輸出**：一個 JSON 物件，包含更新後的 Lead 的完整資訊。

### 2. Contact (聯絡人)

聯絡人是與潛在客戶關聯的個人。

#### 2.1 `create_contact`

- **功能說明**：為一個已存在的潛在客戶新增聯絡人。
- **參數**：
    - `lead_id` (必要)：聯絡人所屬的潛在客戶 ID (字串)。
    - `name` (必要)：聯絡人姓名 (字串)。
    - `emails` (可選)：電子郵件地址列表 (字串列表)。
    - `phones` (可選)：電話號碼列表 (字串列表)。
- **使用範例**：
  > 「在 ID 為 `lead_xxxxxxxx` 的客戶底下，新增一位聯絡人『Jane Smith』，她的信箱是 jane.smith@innovatecorp.com。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Contact 的詳細資訊。

### 3. Opportunity (銷售機會)

銷售機會代表一個潛在的交易，它總是與一個潛在客戶相關聯。

#### 3.1 `create_opportunity`

- **功能說明**：為一個潛在客戶建立一個新的銷售機會。
- **參數**：
    - `lead_id` (必要)：機會所屬的潛在客戶 ID (字串)。
    - `value` (必要)：機會的預估金額 (數字)。
    - `confidence` (可選)：成交的信心指數 (0-100 的整數)。
    - `status` (可選)：機會的當前狀態，例如 `Active`、`Won`、`Lost` (字串)。
- **使用範例**：
  > 「為客戶『Innovate Corp』建立一個新的銷售機會，價值 5000 美元，信心指數 80%。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Opportunity 的詳細資訊。

## 活動工具 (Activity Tools)

活動工具用於記錄與客戶的互動，例如電子郵件、通話和筆記。

### 1. Email

#### 1.1 `log_email`

- **功能說明**：記錄一封已發送或接收的電子郵件到指定的潛在客戶底下。
- **參數**：
    - `lead_id` (必要)：郵件關聯的潛在客戶 ID (字串)。
    - `subject` (必要)：郵件主旨 (字串)。
    - `body` (必要)：郵件內文 (字串)。
    - `direction` (可選)：郵件方向，`incoming` 或 `outgoing` (字串)，預設為 `outgoing`。
- **使用範例**：
  > 「在客戶『Innovate Corp』底下，記錄一封主旨為『關於我們合作的後續步驟』的郵件。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Email Activity 的詳細資訊。

### 2. Call

#### 2.1 `log_call`

- **功能說明**：記錄一通與客戶的通話。
- **參數**：
    - `lead_id` (必要)：通話關聯的潛在客戶 ID (字串)。
    - `duration` (可選)：通話時長（秒）(整數)。
    - `note` (可選)：關於此次通話的筆記 (字串)。
- **使用範例**：
  > 「幫我記錄一通給『Innovate Corp』的電話，我們聊了大概 15 分鐘，討論了他們的預算問題。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Call Activity 的詳細資訊。

### 3. Note

#### 3.1 `create_note`

- **功能說明**：為潛在客戶新增一條筆記。
- **參數**：
    - `lead_id` (必要)：筆記關聯的潛在客戶 ID (字串)。
    - `text` (必要)：筆記內容 (字串)。
- **使用範例**：
  > 「在客戶『Innovate Corp』底下加個備註：他們的主要決策者是 CEO John Doe。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Note Activity 的詳細資訊。

## 自動化與生產力工具 (Automation & Productivity Tools)

這些工具旨在簡化重複性任務，提升銷售團隊的生產力。

### 1. Task (任務)

#### 1.1 `create_task`

- **功能說明**：為自己或團隊成員建立一個待辦任務。
- **參數**：
    - `text` (必要)：任務的描述 (字串)。
    - `lead_id` (可選)：任務關聯的潛在客戶 ID (字串)。
    - `assigned_to` (可選)：被指派者的使用者 ID (字串)，預設為自己。
    - `due_date` (可選)：任務的到期日 (日期字串，例如 `YYYY-MM-DD`)。
- **使用範例**：
  > 「建立一個任務，提醒我明天要跟進『Innovate Corp』的合約進度。」
- **預期輸出**：一個 JSON 物件，包含新建立的 Task 的詳細資訊。

### 2. Bulk Action (批次操作)

#### 2.1 `bulk_update_leads`

- **功能說明**：根據查詢條件，批次更新多個潛在客戶的狀態或自訂欄位。
- **參數**：
    - `query` (必要)：用於篩選目標潛在客戶的查詢語法 (字串)。例如 `status:"Potential"`。
    - `status` (可選)：要更新成的目標狀態 (字串)。
    - `custom_fields` (可選)：要更新的自訂欄位 (字典)。
- **使用範例**：
  > 「將所有狀態是『Trial』的客戶，狀態全部改為『Follow-up』。」
- **預期輸出**：一個 JSON 物件，包含批次操作的狀態，例如已觸發、進行中或已完成。
