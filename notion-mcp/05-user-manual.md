# Supabase 終端使用者操作手冊

作者：Joker

## 歡迎來到您的自然語言後端

本手冊是為您——應用的創造者與管理者——所設計。在這裡，我們不談論程式碼，只專注於**任務**。您將學習如何用最直觀的日常語言，指揮 Manus 和 Supabase 為您完成各種後端工作。忘掉複雜的 API 文件和資料庫語法，從現在開始，您只需要說出您的需求。

## 核心理念：從「如何做」到「做什麼」

傳統的開發模式，您需要學習一套特定的語法（SQL、JavaScript API 等）來告訴電腦「如何」操作數據。Manus Supabase 連結器的核心變革，是讓您只需要專注於「做什麼」。您提出目標，Manus 會負責將其轉化為精確的技術指令。

本手冊將透過一系列常見的任務場景，向您展示這種新工作模式的威力。

## 任務場景導覽

### 任務一：管理您的商品庫存

**場景**：您正在經營一個電子商務網站，需要管理成千上萬的商品。

| 您的自然語言指令 | Manus 如何理解與執行 |
| :--- | :--- |
| 「幫我上架一個新商品，名稱是『超薄筆記型電腦』，價格 1499.99，庫存 50 件。」 | **動作**: 新增紀錄<br>**工具**: `database.insert`<br>**參數**: `into: 'products'`, `values: {name: '...', price: ..., stock: ...}` |
| 「查詢所有庫存少於 10 件的商品。」 | **動作**: 查詢紀錄<br>**工具**: `database.select`<br>**參數**: `from: 'products'`, `filter: {stock: {lt: 10}}` |
| 「把『超薄筆記型電腦』的價格調降為 1399。」 | **動作**: 更新紀錄<br>**工具**: `database.update`<br>**參數**: `table: 'products'`, `filter: {name: '...'}`, `values: {price: 1399}` |
| 「所有『過季』分類的商品都下架。」 | **動作**: 刪除紀錄<br>**工具**: `database.delete`<br>**參數**: `from: 'products'`, `filter: {category: '過季'}` |

### 任務二：營運您的會員系統

**場景**：您正在為您的服務建立一個會員社群，需要管理使用者資料。

| 您的自然語言指令 | Manus 如何理解與執行 |
| :--- | :--- |
| 「手動幫 `new.user@email.com` 註冊一個帳號。」 | **動作**: 建立使用者<br>**工具**: `auth.signUp`<br>**參數**: `email: '...'`, `password: '...'` (可能會提示您設定一個臨時密碼) |
| 「找出所有在過去 30 天內註冊的使用者。」 | **動作**: 查詢紀錄<br>**工具**: `database.select`<br>**參數**: `from: 'users'`, `filter: {created_at: {gt: '...date...'}}` |
| 「將使用者 `id` 為 `usr_abc` 的會員等級提升為 'VIP'。」 | **動作**: 更新紀錄<br>**工具**: `database.update`<br>**參數**: `table: 'users'`, `filter: {id: '...'}`, `values: {level: 'VIP'}` |

### 任務三：打造一個內容發布平台

**場景**：您正在經營一個部落格、新聞網站或任何內容平台。

| 您的自然語言指令 | Manus 如何理解與執行 |
| :--- | :--- |
| 「發布一篇新文章，標題是『AI 的未來』，內容在 `/docs/draft.txt` 裡。」 | **動作**: 讀取檔案 + 新增紀錄<br>**工具**: `file.read` + `database.insert`<br>**流程**: Manus 首先讀取您指定的本地檔案內容，然後將其作為 `content` 欄位的值，插入到 `posts` 資料表中。 |
| 「把文章『AI 的未來』的封面圖片，從本地的 `cover.png` 上傳到 `post-covers` 儲存桶。」 | **動作**: 上傳檔案<br>**工具**: `storage.upload`<br>**參數**: `local_path: '...'`, `bucket: 'post-covers'` |
| 「觸發一個叫做 `notify-subscribers` 的雲端函數，告訴大家有新文章了。」 | **動作**: 調用雲端函數<br>**工具**: `functions.invoke`<br>**參數**: `name: 'notify-subscribers'`, `payload: {post_id: '...'}` |
| 「幫我統計一下，目前總共有幾篇文章？」 | **動作**: 計數查詢<br>**工具**: `database.select` (搭配計數功能)<br>**參數**: `from: 'posts'`, `select: 'count(*)'` |

## 結論：您的想像力是唯一的限制

透過以上範例，您可以看到，Manus Supabase 連結器如何將原本需要多步驟、跨平台、甚至需要編寫程式碼才能完成的任務，簡化為一句話的指令。它不僅僅是一個工具，更是一個賦予您創造力的夥伴。

現在，您已經掌握了與您的後端系統溝通的語言。從管理庫存到營運社群，從發布內容到分析數據，您所需要做的，就是清晰地表達您的目標。剩下的，交給 Manus。
