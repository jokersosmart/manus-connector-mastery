> **核心概念**：學習 Manus for Polygon.io 的關鍵，在於理解它如何將金融市場中，那些稍縱即逝、盤根錯節的數據，轉化為清晰、有序、且可操作的「資訊積木」。這不僅僅是查詢股價，更是一種建立「市場感知系統」的思維。本手冊將引導您，從理解金融數據的基本「原子」——Ticker 和 K 線開始，逐步學會如何將這些原子，組合成「分子」（趨勢分析、比較），最終構建出能夠自動響應市場變化的「複雜結構」（自動化監控與工作流）。

---

## 1. 第一章：金融世界的「身份證」與「心電圖」

### 1.1 Ticker：萬物皆有代碼

在金融市場的汪洋大海中，每一家上市公司、每一種指數、每一對交易的貨幣，都有一個獨一無二的、由字母和符號組成的代碼，這就是 **Ticker**。它是您與市場數據互動的「鑰匙」。

> **核心類比**：將 Ticker 想像成一個人的「身份證號碼」或網站的「域名」。它是明確、無歧義的。當您向 Manus 詢問 `AAPL` 時，它確切地知道您指的是蘋果公司，而不是任何其他水果或公司。

學習使用此連結器的第一步，就是建立起「萬物皆有 Ticker」的認知，並學會使用 `I:` (指數), `C:` (外匯), `X:` (加密貨幣) 等前綴，來精準地定位您想要的資產類別。

### 1.2 K 線 (Aggregates)：市場的「心電圖」

如果說即時報價，是市場在某一秒的「心跳」，那麼 K 線數據 (在 Polygon.io API 中稱為 Aggregates 或 Bars)，就是市場在一段時間內的「心電圖」。它記錄了四個核心資訊：

*   **開盤價 (Open)**: 這段時間開始時的第一筆成交價。
*   **收盤價 (Close)**: 這段時間結束時的最後一筆成交價。
*   **最高價 (High)**: 這段時間內的最高成交價。
*   **最低價 (Low)**: 這段時間內的最低成交價。

> **核心洞察**：單獨一個價格點，是孤立的。但一系列的 K 線，連接起來，就形成了「趨勢」。它們以視覺化的方式，展現了市場在一段時間內的多空力量對比、波動幅度以及價格前進的方向。Manus 的 `get_aggregates` 工具，就是您獲取這些「心電圖」數據的窗口，而其內建的視覺化能力，則是解讀這些心電圖的「顯示器」。

## 2. 第二章：從「單點」到「多維」

### 2.1 數據的維度：超越價格

一個成熟的市場分析，絕不能只看價格。價格的波動，只是結果，其背後，有無數的驅動因素。Manus for Polygon.io 提供了一系列工具，讓您可以從多個維度，來探索一個金融標的：

*   **基本面維度 (`get_financials`)**: 這家公司賺錢嗎？它的利潤率是多少？負債情況如何？這決定了一家公司的內在價值。
*   **消息面維度 (`get_news`)**: 最近有什麼關於這家公司的新聞？是利好還是利空？這影響了市場的短期情緒和預期。
*   **公司面維度 (`get_ticker_details`)**: 這家公司是做什麼的？在哪個行業？有多少員工？這提供了分析的宏觀背景。

> **核心思維**：學習的進階，在於從「盯著價格線」，轉變為「圍繞一個 Ticker，構建一個多維度的資訊儀表板」。當您看到一個異常的價格波動時，您的第一反應，應該是立即調用 `get_news` 來尋找催化劑，或調用 `get_financials` 來評估其基本面是否發生了變化。

### 2.2 比較的藝術：尋找 Alpha 和 Beta

在金融市場中，沒有絕對的好壞，只有「相對」的強弱。一個標的是否值得投資，往往取決於它與「同類」或「大盤」的比較結果。

*   **與大盤比較 (尋找 Beta)**: 將一支個股的走勢，與標普 500 指數的走勢，疊加在一起。如果它漲得比指數多，跌得比指數少，那麼它就具有「超額收益 (Alpha)」的潛力。
*   **與同業比較 (尋找相對強弱)**: 將福特和通用汽車的股價放在一起比較。如果一段時間內，福特的股價持續跑贏通用，這可能意味著市場更看好福特的未來，或者福特的基本面發生了更積極的變化。

> **核心能力**：進階使用者，應熟練掌握，如何讓 Manus 將多個標的的 `get_aggregates` 結果，繪製在同一張圖表上。這種「同框比較」的能力，是發現投資機會和市場異常的關鍵一步。

## 3. 第三章：化被動為主動，讓數據為您工作

### 3.1 監控與警報：從「拉取」到「推送」

新手使用者，是「主動去拉取 (Pull)」數據：我想知道價格 -> 我去問 Manus。
專家使用者，是讓數據「在滿足條件時，主動推送 (Push)」給自己：當價格突破 X 時 -> Manus 來告訴我。

這種從 Pull 到 Push 的轉變，是學習本工具的一個關鍵躍升。它意味著，您將 Manus 從一個「查詢助手」，變成了您 7x24 小時不間斷的「市場監控哨兵」。

### 3.2 工作流：將分析流程「程式化」

在日常工作中，很多分析任務，是重複性的。例如，「每日開盤前，查看主要競爭對手的盤前股價和最新新聞」。

專家級的使用，意味著要識別出這些重複性的流程，並將其「程式化」或「範本化」。您需要學會設計一個包含多個步驟的「工作流」，然後讓 Manus 能夠一鍵執行。

> **工作流的本質**：一個工作流，就是一系列預先定義好的、由 Manus 工具組成的「指令清單」。例如：
> 1.  獲取 Ticker 列表 A, B, C。
> 2.  對於列表中的每一個 Ticker，執行 `get_quote`。
> 3.  對於列表中的每一個 Ticker，執行 `get_news(limit=1)`。
> 4.  將所有結果，格式化為一個表格。
> 5.  將表格，發送到 Slack 的 #market-updates 頻道。

通過本手冊的學習，您應該深刻理解，Manus for Polygon.io 的真正力量，不在於單次查詢的便捷，而在於它能夠將複雜的、多維度的、重複性的市場數據分析任務，轉化為簡單、自動、且能深度融入您日常決策流程的智慧化服務。
